[Solution] 并查集题型整理

[L5] 并查集题型整理

这份笔记作为并查集的主入口，按「问题模型」而不是单纯按模板名字整理。主线是：先用普通并查集维护连通块，再把关系编码成扩展域，再把关系压成带权差值，最后处理队列合并这种特殊位置模型。

一、连通块问题

这一类是并查集的最典型应用，核心在于判断点是否处在同一个连通块内，或者维护连通块数量。

常见变化有三种(暂时整理)：

  1. 离散化约束：变量值域很大，但出现的变量有限，先离散化再合并。
  2. 二分答案 + 连通性检查：答案具有单调性，每次 check 用并查集判断是否形成阻断或连通。
  3. 反向加点 / 加边：正着删点不好维护，倒着恢复点，把分裂问题变成合并问题。

1. P1955 [NOI2015] 程序自动分析

题意

给定若干约束，每条约束形如 xi=xjx_i = x_jxi​=xj​ 或 xi≠xjx_i \neq x_jxi​=xj​，判断这些约束能否同时成立。

约束数量 n≤1e5n \le 1e5n≤1e5，变量编号值域很大，需要离散化。

思路

相等关系具有传递性，所以先把所有 xi=xjx_i = x_jxi​=xj​ 的约束合并起来，得到完整的相等闭包。

变量编号值域很大，但一组数据中真正出现的变量最多只有 2n2n2n 个，所以先用离散化把原变量编号映射成连续下标。

最后再检查所有 xiexjx_i e x_jxi​exj​ 的约束，如果两个变量已经处在同一个连通块里，就说明矛盾。

这题不需要扩展域，因为「不等关系」只用于检查冲突，不需要继续参与传递建模。

2. P1783 海滩防御

题意

给定一个宽度为 nnn 的区域和若干个灯塔。每个灯塔的照亮半径为 rrr，如果灯塔照亮区域从左边界连通到右边界，就会形成阻断。求最小的阻断半径。

思路

答案 rrr 具有单调性：如果半径 rrr 已经能阻断，那么更大的半径也一定能阻断。因此可以二分答案。

对于固定的 rrr，用并查集判断所有灯塔照亮区域之间是否相交。如果两个灯塔距离不超过 2r2r2r，就把它们合并。

额外建立两个虚拟点：左边界 L 和右边界 R；如果某个灯塔距离左边界不超过 rrr，就和 L 合并；距离右边界不超过 rrr，就和 R 合并。

最后如果 L 和 R 连通，说明这个半径已经能形成阻断。

此处左右端点和灯塔之间的距离是 rrr，而两个灯塔之间的距离是 2r2r2r。

3. P1197 [JSOI2008] 星球大战

题意

给定一张无向图，然后依次摧毁若干点。每摧毁一个点后，询问剩余图中有多少个连通块。

思路

本题正着思考，每次删点都会让一个连通块分裂，而并查集不擅长处理分裂。

所以可以考虑反过来做：先把所有被摧毁的点都标记掉，得到最终状态；然后从最后一次摧毁开始，倒着把点一个个恢复回来，这样就转化为了一个合并问题。

恢复一个点时，连通块数量先加一，再枚举它的邻点。如果邻点当前没有被摧毁，就尝试合并两个连通块；每成功合并一次，连通块数量减一。

这样就把动态删点问题变成了动态加点 / 加边问题。

二、扩展域模型

扩展域模型适合处理「同类 / 异类」「敌人 / 朋友」「吃 / 被吃」这类离散关系。

它的核心是：给每个点拆出若干个离散域，然后把关系转化为连通性，若某个点和它不改相同的域连通，就说明存在矛盾。

4. P1892 [BOI2003] 团伙

题意

有 nnn 个人和 mmm 条关系。F p q 表示 p,qp, qp,q 是朋友；E p q 表示 p,qp, qp,q 是敌人。敌人的敌人是朋友，问最后有多少个团伙。

思路

这题不能用二倍域的做法，因为朋友的朋友是朋友，敌人的敌人是朋友，但是敌人的朋友不一定是敌人，这里采用 enemy[] 标记法，只维护题目要求的关系：

  • 如果 p,qp, qp,q 是朋友，直接合并。
  • 如果 p,qp, qp,q 是敌人，那么 ppp 的敌人应该和 qqq 合并，qqq 的敌人应该和 ppp 合并。

因为最后只问团伙数量，所以枚举连通块数目即可。

5. P1525 [NOIP2010] 关押罪犯

题意

有两个监狱，给定若干对罪犯之间的仇恨值。需要安排每个罪犯所在的监狱，使得同一个监狱内冲突值最大的那一对尽可能小。

6. P2024 [NOI2001] 食物链：三倍域写法

题意

有三类动物构成循环捕食关系。给定若干句话：1 x y 表示 x,yx,yx,y 同类，2 x y 表示 xxx 吃 yyy，问有多少句话是假话。

思路

食物链关系不是简单的同类 / 异类，而是一个长度为 333 的循环关系，所以可以把每个动物拆成三个域：

  • id(x, 0)：xxx 自己所在的类。
  • id(x, 1)：xxx 吃的那一类。
  • id(x, 2)：吃 xxx 的那一类。

三、带权并查集

带权并查集适合维护「点与点之间的相对关系」。它不再只是问两个点是否连通，还能在连通时查询 val[x]−val[y]val[x] - val[y]val[x]−val[y]。

常见两种版本：

  1. 取模带权并查集：维护 (val[x]−val[y]) mod mod(val[x] - val[y]) \bmod mod(val[x]−val[y])modmod，适合循环关系，比如食物链。
  2. 势能差带权并查集：维护真实差值 val[x]−val[y]val[x] - val[y]val[x]−val[y]，适合前缀和、距离、区间和一类差分约束。

7. P2024 [NOI2001] 食物链：带权写法

题意

同上一题，判断食物链中有多少句假话。

思路

三倍域是把每个点拆成三个域；带权并查集则是把这三种关系压成一个模 333 意义下的差值。

令 dist(x, y) = (val[x] - val[y]) mod 3，那么：

  • dist(x, y) = 0：x,yx,yx,y 同类。
  • dist(x, y) = 1：xxx 吃 yyy。
  • dist(x, y) = 2：xxx 被 yyy 吃。

所以 opt == 1 就是加入 val[x] - val[y] = 0，opt == 2 就是加入 val[x] - val[y] = 1。

带权并查集的 merge(x, y, w) 本身就可以处理两种情况：如果 x,yx,yx,y 已经连通，就检查原有差值是否等于 www；如果还不连通，就合并两个集合，并维护新根到旧根之间的差值。

从复习角度看，这个写法比三倍域更抽象，但代码更短，也更接近「关系 = 差值约束」这个本质。
