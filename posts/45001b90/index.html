2. 逻辑推导

B - Center Alignment

1. 题意梗概

给定 NNN 个长度为奇数的字符串。求出所有字符串中最长的长度 mmm，并将所有字符串通过在前后填充点号 . 的方式对齐到长度 mmm。

2. 逻辑推导

简单模拟题，找出最大长度 mmm，然后由于是奇数对称的，所以单侧填充长度就是 cnt=(m−len)/2cnt = (m - len) / 2cnt=(m−len)/2。

C - Sugoroku Destination

1. 题意梗概

有 NNN 个单元格，每个格子写着一个数字 AiA_iAi​。棋子从位置 sss 出发，每次移动到当前格子上写的数字对应的格子。求 1010010^{100}10100 次移动后棋子的位置。

  • 1≤N≤5×1051 \le N \le 5\times10^51≤N≤5×105
  • i≤Ai≤N (1≤i≤N)i \le A_i \le N\ (1 \le i \le N)i≤Ai​≤N (1≤i≤N)

2. 逻辑推导

这题有点小巧思，因为要求移动 1010010^{100}10100 次，这就不能手动模拟了。

注意到一个关键约束 Ai≥iA_i \ge iAi​≥i，意味着棋子只能向后移动或留在原处。

所以经过极多次移动后，棋子 xxx 一定会留在某个 Ax=xA_{x} = xAx​=x 的自环上。

DP：由于路径没有回退（无环），我们可以反过来计算 每个点的最终归宿。

  • 若 Ai=iA_{i} = iAi​=i，则终点是它自己；否则它的终点就是 AiA_{i}Ai​ 的终点。

D - Reconstruct Chocolate

1. 题意梗概

给定 NNN 个矩形巧克力碎片的尺寸 (hi,wi)(h_i, w_i)(hi​,wi​)，这些碎片是通过对一个 H×WH \times WH×W 的大矩形进行 N−1N-1N−1 次操作得到的；操作为切下一块放在桌上，手里留一块。求出每个碎片的左上角坐标 (xi,yi)(x_i, y_i)(xi​,yi​)。

Constraints

  • 1≤H≤1091 \leq H \leq 10^91≤H≤109
  • 1≤W≤1091 \leq W \leq 10^91≤W≤109
  • 2≤N≤2×1052 \leq N \leq 2\times 10^52≤N≤2×105
  • 1≤hi≤H1 \leq h_i \leq H1≤hi​≤H
  • 1≤wi≤W1 \leq w_i \leq W1≤wi​≤W

2. 逻辑推导

这个 N≤2e5N \leq 2e5N≤2e5，就没法暴力了。

我们分析这个操作，每一次操作，都会让手上现有的巧克力切成两块，而且是沿着水平或者垂直线切的，这意味这什么？

这意味着切完之后这两块巧克力，要么 hhh 相等，要么 www 相等。

所以我们可以采用贪心模拟的策略，由于从最后一块开始拼装处理起来很麻烦，所以我们考虑从完整的巧克力开始切，考虑剩余的部分可以塞下谁。

维护当前剩余矩形的左上角坐标 (cx, cy) 和剩余尺寸 (ch, cw)。

  1. 若存在碎片高度等于 ch，则该碎片填在当前列，cy 向右移动该碎片的宽度，cw 减少。

  2. 若存在碎片宽度等于 cw，则该碎片填在当前行，cx 向下移动该碎片的高度，ch 减少。

然后就是快速查询的问题了，由于这题的 H,W≤1e9H, W \le 1e9H,W≤1e9，但是 N≤2e5N \leq 2e5N≤2e5，最直接的想法就是离散化，然后二分查询。

因为标准离散化写起来比较麻烦（苍天可鉴，代码复杂度高），我这里就偷懒采用 mapmapmap 作为离散化工具。

因为同一高度或者宽度的巧克力可能有很多，所以我开了一个 map< int, set< int > > 来存储，第一维是具体的值，第二位是符合条件的值的下标。
