[L2] 10_数据结构

[L2] 10_数据结构

核心哲学：数据结构是通过高效数据管理来优化时间复杂度。
要么把数据切成块处理（分治/块），
要么把重叠的信息存起来复用（持久化/动态树）。

1. 知识图谱与演化

  • 核心模型: 递归分治、路径压缩、时空复用、根号平衡。

  • 演化路径:

    1. 基础线性：栈、队列。解决端点或全局问题。

    2. 树形化：树状数组、线段树。利用对数层级结构解决线性扫描。

    3. 动态化/持久化：主席树、LCT。利用历史版本或维护动态拓扑。

    4. 根号平衡：莫队、分块。通过“暴力分组”解决非结合律信息维护。

2. 决策矩阵

  • 信息可差分（和/异或）+ 单点修改：树状数组。

  • 信息满足结合律 + 区间修改：线段树。

  • 查询历史版本 / 区间第 K 大：[[Note] 主席树]。

  • 动态连通性 / 传递性逻辑：并查集。

  • 动态断边/连边 / 树上路径信息：[[Note] LCT]。

  • 无结合律 / 离线区间查询：[[Note] 莫队体系] 或 [[Note] 分块体系]。

  • 信息满足幂等性（Max/Min/GCD）+ 静态海量查询：采用 ST表与倍增。

3. 核心板块索引

3.1 [L3] 11_树形结构

  • ST表与倍增: 幂等性静态覆盖。

  • 树状数组: 轻量的前缀和维护。

  • 线段树: 区间管理的万能工具。

  • [[Note] 线段树合并]：树上信息的合并

  • [[Note] 主席树]：可持久化线段树，跨越时间轴的查询。

  • [[Note] 平衡树]：Splay 与 FHQ-Treap 的动态排序。

  • [[Note] 笛卡尔树]：结合堆与BST性质的结构。

3.2 [L3] [[12_连通性结构]]

  • 并查集：集合合并与动态连通。

  • [[Note] LCT]：(Link-Cut Tree) 动态树问题。

  • [[Note] 可撤销并查集]: 撤销与历史版本。

3.3 [L3] [[13_分块与莫队]]

  • [[Note] 分块体系]: 根号平衡的艺术。

  • [[Note] 莫队体系]: 基于离线排序的区间移动。
