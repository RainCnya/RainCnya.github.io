[L2] 00_基础与战术

[L2] 00_基础与战术

核心哲学：算法竞赛不是死记硬背模板，而是从杂乱的题目描述中剥离出不变的数理规律。
基础算法的本质只有两步：看清哪些是干扰你的废话，然后用最简单的数学逻辑把问题拆散。

1. 知识图谱与演化

  • 核心抓手: 贡献度拆解、不变量构造、单调性观察、信息简化。

  • 演化路径:

    1. 暴力模拟：最原始的逻辑复现，不带任何优化地跟着题目走。

    2. 局部贪心：通过观察局部性质尝试突破全局瓶颈，用小范围的最优推导大范围的解。

    3. 降维优化：引入双指针、前缀和、离散化，将重复的搜索过程转化为简单的线性扫描或查表。

    4. 数学抽象：进入构造与随机化领域，利用数学本质（如二进制、奇偶性）或概率规律寻找答案。

2. 决策矩阵

  • 题目要求“构造一个解”或“判定是否存在”: 优先寻找不变量。观察总和、异或值、或者相对位置在操作中是否保持不变。

  • 答案越大/越小，越容易/越难达标: 典型的单调性，优先考虑二分答案。

  • 性质随区间滑动单调变化，或涉及局部最值: 考虑双指针或单调栈/队列，杀掉回溯扫描。

  • 需要频繁统计或修改某一段区间的总和: 采用前缀和与差分将范围操作变为点位操作。

  • 正面硬推算力不够: 尝试补集转化（求不满足条件的）或者从终点倒着推回来。

3. 核心板块索引

3.1 构造思维

  • [L3] 01 构造思维: 解决“如何构建一个解”的逻辑总纲。

    • [[Note] 奇偶性分析]: 利用对立状态寻找冲突或平衡点。

    • [[Note] 二进制构造]: 利用位独立性拼凑目标。

3.2 搜索战术

  • [L3] 02 搜索战术: 当没有固定算法可跑时，通过聪明地尝试可能性来拿分。

    • [[Note] 折半搜索]: 任务切半，分别搜完再合并，大幅降低运行时间。

    • [[Note] 启发式搜索]: 利用估价函数引导搜索方向，杀掉无效分支。

    • [[Note] 模拟退火]: 针对最优化问题，通过“随机乱跳”撞出最优解。

3.3 交互与随机

  • [L3] 03 交互与随机: 解决需要与评测系统进行实时信息交换的题目。

    • [[Note] 询问策略]: 每次提问都排除掉尽可能多的错误选项。

    • [[Note] 随机化判定]: 通过多次采样或哈希校验得出极速结论。

3.4 降维技巧

  • [L3] 04 降维技巧: 杀掉冗余计算、提升运行效率的核心。

    • [[Note] 双指针]: 绝不回头的逻辑。利用单调性拉直循环。

    • [[Note] 单调栈与队列]: 优胜劣汰模型。实现局部最值的高效维护。

    • [[Note] 离散化]: 只认排名不认数值。挤掉大值域中的空间空隙。

    • [[Note] 前缀和与差分]: 存账本逻辑。实现区间操作的点位化。

    • [[Note] SOS DP]: 维度折叠思想。解决子集求和的终极加速。

3.5 贪心战术

  • [L3] 05 贪心战术: 追求局部最优推导全局最优的博弈艺术。

    • [[Note] 邻项交换排序]: 考虑相邻元素交换后的代价差，推导全局排序标准。

    • [[Note] 反悔贪心]: 建立撤销机制。利用优先队列动态修正之前的贪心选择。

    • [[Note] 中位数贪心]: 绝对值之和最小化模型，杀掉距离代价。

    • [[Note] 权衡贪心]: 解决多约束资源分配，如“免费匹配”与“成本购买”的平衡。

4. 依赖与规划

  • 前置依赖: C++ STL 容器（vector, map）、位运算、基础排序。

  • 学习路径:

    1. P1 (基础)：熟练写出前缀和、差分与双指针，不再在边界条件上纠结。

    2. P2 (中坚)：掌握单调栈优化 DP 以及离散化的标准套路。

    3. P3 (顶尖)：掌握折半搜索与 SOS DP，学会处理复杂的高维搜索。

5. 题目归档
