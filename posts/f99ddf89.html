P5304 [GXOI/GZOI2019] 旅行者

Luogu-P5304 [GXOI/GZOI2019] 旅行者

题目简述

难度：提高+ | 题解格式：V6.3

给定一张 NNN 个点 MMM 条边的有向带权图，以及 KKK 个关键城市。求这 KKK 个关键城市中，任意两个不同城市之间最短路距离的最小值。

  • 数据范围：N≤105,M≤5×105,K≤NN \le 10^5, M \le 5 \times 10^5, K \le NN≤105,M≤5×105,K≤N。

解题思路

1. 初步认知

  • 直观想法：对每一个关键城市跑一遍单源 Dijkstra，统计到其他关键城市的距离。
  • 瓶颈分析：这样做的时间复杂度为 O(K⋅Mlog⁡N)O(K \cdot M \log N)O(K⋅MlogN)。当 KKK 接近 NNN 时，复杂度退化为 O(N⋅Mlog⁡N)O(N \cdot M \log N)O(N⋅MlogN)，显然会超时。我们需要一种与 KKK 无关（或者对数级）的算法。

2. 主题渗透 (正解)

算法：染色法 + 多源 Dijkstra (O(Mlog⁡N)O(M \log N)O(MlogN))

  • 基本概念：利用多源最短路将图中的点根据“离谁最近”划分为不同的势力范围（染色）。
  • 状态定义：
    • 正向染色：建立虚拟超级源点连接所有关键城市（或直接将所有关键城市入队，距离设为0）。跑一遍 Dijkstra。
      • d1[u]：从最近的某个关键城市到 uuu 的距离。
      • c1[u]：该关键城市的编号（颜色）。
    • 反向染色：建立反图，同样对所有关键城市跑多源 Dijkstra。
      • d2[u]：从 uuu 到最近的某个关键城市的距离（原图视角）。
      • c2[u]：该关键城市的编号（颜色）。
  • 转移逻辑：
    答案对应的最短路一定是由某个关键点 AAA 出发，经过一系列点，最终到达另一个关键点 BBB。
    这条路径上一定存在一条边 u→vu \to vu→v，使得 uuu 属于 AAA 的势力范围（c1[u]==A），而 vvv 属于 BBB 的势力范围（c2[v]==B）。
    因此，只需枚举每条边 u→wvu \xrightarrow{w} vuw​v，若 c1[u] != c2[v]，则用 d1[u] + w + d2[v] 更新答案。

#include<bits/stdc++.h>using namespace std;using ll = long long;using PII = pair<int, int>;using PLI = pair<ll, int>;const int maxn = 1e5 + 50;const ll inf = 1e18;// adj 存原图，radj 存反图vector<PII> adj[maxn];vector<PII> radj[maxn];int ks[maxn];int n, m, k;// d1, c1: 正向的距离和颜色// d2, c2: 反向的距离和颜色ll d1[maxn], d2[maxn];int c1[maxn], c2[maxn];// 通用多源 Dijkstra 函数// type = 0: 正向图处理 d1, c1// type = 1: 反向图处理 d2, c2void dijkstra( int type ){    ll *dist; int *color; vector<PII> *graph;    // 使用指针动态绑定数组和图，复用逻辑    if( type == 0 ) dist = d1, color = c1, graph = adj;    else dist = d2, color = c2, graph = radj;    for( int i = 1; i <= n; ++ i )    {        dist[i] = inf;        color[i] = 0;    }     priority_queue< PLI, vector<PLI>, greater<PLI> > pq;    // 多源 BFS/Dijkstra 初始化：将所有关键点同时入队    for( int i = 1; i <= k; ++ i )    {        int u = ks[i]; // 修正变量名，避免与全局 k 混淆        dist[u] = 0;        color[u] = u; // 初始颜色就是自己        pq.push({ 0, u });    }    while( !pq.empty( ) )    {        auto [d, u] = pq.top( );        pq.pop( );        if( d > dist[u] ) continue;        // 遍历绑定的图（原图或反图）        for( auto [v, w] : (*graph)[u] ) // 修正拼写 gragh -> graph        {            if( dist[u] + w < dist[v] )            {                dist[v] = dist[u] + w;                color[v] = color[u]; // 关键点：颜色的传递（扩散）                pq.push({ dist[v], v });            }        }    }}void solve( ){    cin >> n >> m >> k;    for( int i = 1; i <= n; ++ i )    {        adj[i].clear( );        radj[i].clear( );    }    for( int i = 1; i <= m; ++ i )    {        int u, v, w;        cin >> u >> v >> w;        if( u != v ) // 自环对最短路无贡献，可忽略        {            adj[u].push_back({ v, w });            radj[v].push_back({ u, w });        }    }    for( int i = 1; i <= k; ++ i )    {        cin >> ks[i];    }    dijkstra( 0 ); // 正向染色    dijkstra( 1 ); // 反向染色    ll ans = inf;    // 枚举每一条边 u -> v    for( int u = 1; u <= n; ++ u )    {        for( auto& [v, w] : adj[u] )        {            // 核心逻辑：            // 1. u 必须能从某个关键点到达 (c1[u] != 0)            // 2. v 必须能到达某个关键点 (c2[v] != 0)            // 3. 来源的关键点和去往的关键点必须不同 (c1[u] != c2[v])            if( c1[u] && c2[v] && c1[u] != c2[v] )                ans = min( ans, d1[u] + w + d2[v] );        }    }    cout << ans << '\n';}int main( ){    ios::sync_with_stdio(0);    cin.tie(0); cout.tie(0);    int _t;    cin >> _t;    while( _t -- )    {        solve( );    }    return 0;}

四、 结语

本题采用了处理“关键点对最短路”问题的经典降维打击思路：染色法。

通过两次多源 Dijkstra（一次正向、一次反向），我们将 O(K2)O(K^2)O(K2) 或 O(K⋅N)O(K \cdot N)O(K⋅N) 的问题转化为 O(Mlog⁡N)O(M \log N)O(MlogN)。

这种“中间相遇”的思想（枚举边界边连接两个不同区域）在图论中非常通用。
