TZ9549 - Signal Transmission

1. 核心逻辑

  • 模型抽象: 信号从节点 uuu 向根节点传输，若当前信号 S>apS > a_pS>ap​ 则 SSS 变为 apa_pap​ 且压缩次数 +1+1+1。求各节点总压缩次数。

    • 中译中: 找根到节点路径上，“前缀最小值”的变化次数。这本质上是在树上跑 LIS (最长下降子序列)。

  • 逻辑支点 (树上 LIS 贪心):

    1. 贪心策略: 维护一个从根出发的单调递减栈 stk。

    2. 二分定位: 采用 O(Nlog⁡N)O(N \log N)O(NlogN) 的 LIS 核心思想。每到一个新节点 uuu，在栈中二分找到第一个 ≥au\ge a_u≥au​ 的位置并替换它。

    3. 状态转移: dp[u]=dp[stk[pos−1]]+1dp[u] = dp[stk[pos - 1]] + 1dp[u]=dp[stk[pos−1]]+1。

      • 这里 pos 是 uuu 在单调栈中的插入位置。

      • stk[pos-1] 即为 uuu 上方最近的一个能够触发压缩的祖先。

2. 逻辑推导

  • 初始化:

    • 设一个虚拟节点 0，其带宽 a[0]=0a[0] = 0a[0]=0，作为所有压缩的逻辑终点。

    • 令 dp[0] = -1，确保根节点（1号点）出发时的压缩次数初始化为 000。

  • 回溯:

    • 由于是全局数组模拟栈，DFS 返回前必须还原 stk[pos]，防止污染其他分支。

4. 复盘

  • 复杂度: O(Nlog⁡N)O(N \log N)O(NlogN)。

  • 认知补丁:

    • 逻辑穿透: 不要被“信号传输”带歪。看到路径上的“变化次数”，第一反应应该是单调性维护。

    • 初始化技巧: dp[0] = -1 配合虚拟节点 0 是处理这种“根节点作为终点不计数”边界情况的常用手段。
