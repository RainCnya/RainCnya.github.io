TZOJ-8937

Week3 - 8937: 序列平衡

本题是 前缀和 的经典应用。

一. 题意分析

题目大意

题目给定一个长度为 nnn 的序列 an{a_n}an​。我们需要找到一个 “分割点” 1≤k<n1 \leq k < n1≤k<n, 将序列分成左右两个部分。

  • 左半部分：a1,a2,...,aka_1, a_2, ..., a_ka1​,a2​,...,ak​

  • 右半部分：ak+1,ak+2,...,ana_{k+1}, a_{k+2}, ... , a_nak+1​,ak+2​,...,an​

目标是让 左半部分 和 右半部分 之和的差的绝对值最小。

用数学语言描述，就是找到一个 kkk，使得 ∣(a1+...+ak)−(ak+1+...+an)∣|(a_1 + ... + a_k) - (a_{k+1} + ... + a_n)|∣(a1​+...+ak​)−(ak+1​+...+an​)∣ 最小。

二. 思路推进

1. 暴力（最直观的想法）

最直观的想法是：枚举所有可能的分割点。

分割点可以从第 111 个元素后，一直到 n−1n-1n−1 个元素后。

  • 当 k=1k=1k=1 时，分割点在 a1a_1a1​ 和 a2a_2a2​ 之间。

  • 当 k=n−1k=n-1k=n−1 时，分割点在 an−1a_{n-1}an−1​ 和 ana_nan​ 之间。

对于每个分割点 kkk，我们都可以通过两个循环分别计算左右部分的和，然后计算差值，更新全局最小值即可。

暴力

  • 复杂度分析：外层循环 kkk 从 111 到 n−1n-1n−1，复杂度为 $O(n)。内层计算 left 和 right 的总复杂度也是 O(n)O(n)O(n)。因此，总时间复杂度为 O(n2)O(n^2)O(n2)。

  • 结果：对于这题 n≤1e5n\leq1e5n≤1e5 来说，最终时间为 1e101e101e10 远远超过了 1s1s1s 时间限制的 1e81e81e8 的数据处理量。

1e5=1051e5=10^51e5=105 科学技术法 方便书写

一般来说，计算机 1s1s1s 能执行 1e81e81e8 的数据处理。

所以对于输入数据的规模，我们需要找到不同时间复杂度的算法来解决问题。

这就是算法的应用，你理解了吗？

因此，我们需要一个更加优秀的算法解决这个问题。

2. 前缀和（不难想到的优化）

根据之前对题目的分析，我们发现，暴力算法的瓶颈在于 重复计算。

当 kkk 从 222 变成 333 的时候，我们只需要在 k=2k=2k=2 的 left 基础上加上 a3a_3a3​，以及 right 基础上减去 a3a_3a3​ 即可。

但暴力算法缺从 a1a_1a1​ 重新加了一遍。

这种 快速求区间和 的需求，正是 前缀和 的应用场景。

2.1 什么是前缀和？

前缀和数组 sumsumsum 是对原数组 aaa 的预处理。

sum[i]sum[i]sum[i] 定义为原数组 aaa 的 前 iii 项和。

即 sum[i]=a1+a2+...+aisum[i] = a_1 + a_2 + ... + a_isum[i]=a1​+a2​+...+ai​

接下来就可以用 $O(n)的时间递推地预处理出 sumsumsum 数组：

sum[0]=0sum[0] = 0 sum[0]=0

sum[i]=sum[i−1]+a[i]sum[i] =sum[i-1] + a[i] sum[i]=sum[i−1]+a[i]

2.2 如何使用前缀和？

如何使用前缀和？预处理之后，我们可以在 O(1)O(1)O(1) 的时间内得到任意区间 [l,r][l, r][l,r] 的和：

Sum(l,r)=al+...+ar=sum[r]−sum[l−1]\text{Sum}(l, r) = a_l + ... + a_r = sum[r] - sum[l-1] Sum(l,r)=al​+...+ar​=sum[r]−sum[l−1]

2.3 回归本题

回到本题，对于任意分割点 kkk：左半部分 [1,k][1, k][1,k] 的和：sum[k]−sum[0]=sum[k]sum[k] - sum[0] = sum[k]sum[k]−sum[0]=sum[k]

右半部分 [k+1,n][k+1, n][k+1,n] 的和：sum[n]−sum[k]sum[n] - sum[k]sum[n]−sum[k]

这样，我们就可以在 O(1)O(1)O(1) 的时间内计算出差值了！

三. 还有陷阱

还有一个隐藏陷阱，注意到 a≤1e5a \leq 1e5a≤1e5，并且n≤1e5n \leq 1e5n≤1e5。

所以，极端情况下，我们会处理到 1e51e51e5 个数值为 1e51e51e5 的区间，此时区间和最大为 n∗ai=1e10n * a_i = 1e10n∗ai​=1e10。

而在 C++标准中我们的 intintint 整数类型只能存 231−12^{31}-1231−1 也就是差不多 1e91e91e9 的数字。

假如计算过程中超过了 intintint 的范围，就会导致 整数溢出，从而导致 WA。

所以，我们还需要使用 longlonglong longlonglong 来存储数据。
