算法学习笔记：最长公共子序列 (LCS)

一、 算法图谱

  • 命名： 最长公共子序列 (Longest Common Subsequence)
  • 领域： 线性动态规划 (Linear DP) / 双串模型
  • 定义： 给定两个序列，找出它们之间不要求连续的、最长的共同子序列。
  • 典型设问： “求两个字符串的最长公共子序列长度。”

二、 核心推演

1. 基本思路

  • 状态定义： dp[i][j] 表示字符串 AAA 的前 iii 个字符和字符串 BBB 的前 jjj 个字符的 LCS 长度。
  • 状态转移：
    • Case 1 (相等)： 若 A[i]==B[j]A[i] == B[j]A[i]==B[j]，则该字符一定在 LCS 中。
      • dp[i][j]=dp[i−1][j−1]+1dp[i][j] = dp[i-1][j-1] + 1dp[i][j]=dp[i−1][j−1]+1
    • Case 2 (不等)： 若 A[i]≠B[j]A[i] \neq B[j]A[i]=B[j]，则两个字符至少有一个不在 LCS 结尾。
      • dp[i][j]=max⁡(dp[i−1][j],dp[i][j−1])dp[i][j] = \max(dp[i-1][j], dp[i][j-1])dp[i][j]=max(dp[i−1][j],dp[i][j−1])

2. 优化思路

  • 滚动数组 (Space Optimization)：
    • 观察转移方程，dp[i][...] 只依赖于 dp[i-1][...]（上一行）。
    • 方案： 使用 dp[2][m]，通过 i & 1 (当前行) 和 (i-1) & 1 (上一行) 交替使用，将空间从 O(NM)O(NM)O(NM) 降至 O(M)O(M)O(M)。

3. 核心思想

  • 将大问题分解为“最后一个字符是否匹配”的子问题，利用二维 DP 表递推求解。

三、 实战部署

1. 代码模板 (朴素解法)

{% fold info @LCS (O(NM) 空间) %}

// 假设 a[n], b[m] (1-based)
// dp[n][m] 初始化为 0

for( int i = 1; i <= n; ++ i )
{
    for( int j = 1; j <= m; ++ j )
    {
        if( a[i] == b[j] )
        {
            dp[i][j] = dp[i-1][j-1] + 1;
        }
        else
        {
            dp[i][j] = max( dp[i-1][j], dp[i][j-1] );
        }
    }
}
// 答案: dp[n][m]

{% endfold %}

2. 代码模板 (滚动数组优化)

{% fold info @LCS (O(M) 空间 - 推荐) %}

// dp[2][m] 初始化为 0

for( int i = 1; i <= n; ++ i )
{
    int curr = i & 1;
    int prev = (i - 1) & 1;

    for( int j = 1; j <= m; ++ j )
    {
        if( a[i] == b[j] )
        {
            dp[curr][j] = dp[prev][j-1] + 1;
        }
        else
        {
            dp[curr][j] = max( dp[prev][j], dp[curr][j-1] );
        }
    }
}
// 答案: dp[n & 1][m]

{% endfold %}

3. 部署细节

  • 1-based 索引： 必须使用。dp[0][...] 表示空串，长度为 0，是天然的边界条件。
  • 子序列 vs 子串： * LCS (子序列)：else 分支取 max（继承）。
    • 子串 (连续)：else 分支必须置为 0（断开）。

四、 维度概览

维度 复杂度 说明
时间 O(NM)O(NM)O(NM) 必须遍历整个二维表。
空间 O(M)O(M)O(M) 滚动数组优化后。
特性 双串匹配 经典的二维线性 DP。
