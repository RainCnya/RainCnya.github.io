C - Sake or Water

1. 题意梗概（翻译）

给 NNN 杯水，其中有 KKK 杯有酒。 求至少喝掉 XXX 的酒，至少需要选择多少个杯子。

  • 1≤K≤N≤3×105,1≤Ai≤1091 \leq K \leq N \leq 3\times 10^5,1 \leq A_i \leq 10^91≤K≤N≤3×105,1≤Ai​≤109
  • 1≤X≤3×10141 \leq X \leq 3\times 10^{14}1≤X≤3×1014

2. 逻辑推导

这题的关键在构造最坏情况，即贪心思想的应用。

为了保证在最坏情况也能喝到 XXX 毫升的酒，我们就需要让我们选出的杯子中，酒的最少可能总量 “大于等于” XXX。

假设我们选了 mmm 个杯子，在所有 NNN 个杯子中，有 KKK 杯是酒，N−KN-KN−K 杯是水。为了让我们喝到的酒最少，那 N−KN - KN−K 杯水就需要占据我们选出的杯子中容量较大的那部分。

  • 若 m≤N−Km \leq N - Km≤N−K ，就说明此时喝不到酒，k=0k = 0k=0。
  • 若 m≥N−Km \geq N - Km≥N−K ，此时酒的杯子数： k=m−(N−K)k = m - (N - K)k=m−(N−K)。
  • 为了让这 kkk 杯酒的容量尽可能少，所以贪心选择这 mmm 被中容量最小的即可。

明确了这些之后，我们先进行一次排序，将 AAA 降序（从大到小）排序。

如果我们选前 mmm 大的杯子。那么这 mmm 个杯子中，容量最小的 kkk 个分别为：Am−k+1′,Am−k+2′,…,Am′A'_{m-k+1}, A'_{m-k+2},\dots,A'_{m}Am−k+1′​,Am−k+2′​,…,Am′​。

代入 k=m−(N−K)k = m - (N - K)k=m−(N−K) ，得到酒的初始下标为 m−(m−(N−K))+1=N−K+1m - (m - (N - K)) + 1 = N - K + 1m−(m−(N−K))+1=N−K+1，即把问题转换为，从 N−K+1N - K + 1N−K+1 开始，一直加到第 mmm 大的杯子的总和。

D - Paid Walk

1. 题意梗概（翻译）

给一个有向图： N≤2e5N \leq 2e5N≤2e5 个点，M≤2e5M \leq 2e5M≤2e5 条边，每个顶点的出度最多为 444。

找出所有满足条件的点——存在一条路径 1→v1 \to v1→v，恰好遍历了 LLL 条边，遍历 CostCostCost 总和 S≤Cost≤TS \le Cost \le TS≤Cost≤T。

  • 1≤L≤10,1≤S≤T≤1e9,1≤Ci≤1e81 \leq L \leq 10,1 \leq S ≤ T \leq 1e9,1 \leq C_i \leq 1e81≤L≤10,1≤S≤T≤1e9,1≤Ci​≤1e8

2. 逻辑推导

好像有点无从下手，我最开始想能不能先跑 Dijkstra，但是发现跟这题相性很差。再接着看题发现，L≤10L \leq 10L≤10，每个点的出度最多为 444，也就是说，每个点的后续情况最多只有 444 种。

这意味着，如果直接dfs 的话，时间复杂度为 4L≤4104^L \leq 4^{10}4L≤410，绰绰有余！！！

OK，那题目就变得很简单了，我们只需要把所有路线都搜出来，最终满足条件的点记录，最后输出即可。

E - A > B substring

1. 题目梗概

给定一个长度为 N≤5e5N \leq 5e5N≤5e5 的字符串 SSS，它由 A,B,C 组成，求所有子串中 A 的数量大于 B 的个数。

2. 逻辑推导

问有多少个区间内 A 的数量大于 B，那我们不妨令 A=1,B=-1。若某一段区间和大于000，不就说明 A 的数量大于 B 了吗？

既然要用区间和，不妨用前缀和优化。

那么问题就转化为了，找到 i<ji < ji<j 使得 s[j]−s[i−1]>0  ⟺  s[i−1]<s[j]s[j] - s[i-1] > 0 \iff s[i-1] < s[j]s[j]−s[i−1]>0⟺s[i−1]<s[j]，咦，看起来这就是 "顺序对" 啊。

那么这里有两种方案了。"树状数组" / "归并排序"。

我才用树状数组来统计答案，应该是比较经典的写法。

F - Must Buy

1. 题面梗概（翻译）

给定 N≤1000N \leq 1000N≤1000 个物品，第 iii 个物品的价格为 PiP_iPi​，价值为 ViV_iVi​，每个物品只有一个。同时给定 M≤5e4M \leq 5e4M≤5e4 的预算，处理每个物品的类别。

分类要求，为了让所选问题的总价值最大化：必选 A，必不选 C，其他 B。

  • 1≤Pi≤M,1≤Vi≤1e91 \leq P_i \leq M, 1 \leq V_i \leq 1e91≤Pi​≤M,1≤Vi​≤1e9

2. 逻辑推导

乍一看，这不背包问题吗？只有一个？直接 0/1 背包启动！！！

仔细品了品，发现问题不简单，简单的跑一遍背包只能算出结果，不能算出每个物品的分类。

既然如此，我们就需要通过别的方式来完成分类。注意到我们只需要考虑 "必选" 和 "必不选" 两种情况即可，不妨用前后缀 DP 来解决。

令 Pre[i][j] 表示 1→i1 \to i1→i 中总价格为 jjj 的最大价值。Post[i][j] 表示从 n→in \to in→i 中总价格为 jjj 的最大价值。

那么我们就只需要讨论当前这个物品 "选或不选" 会不会影响最终决策，当然最终的最大值方案就是 Pre[n][m]。

假设当前在考虑第 iii 个物品，那么不选它的最大价值为：

Without=max1m(pre[i−1][j]+post[i−1][m−j])Without = max_{1}^{m}(pre[i-1][j] + post[i-1][m-j]) Without=max1m​(pre[i−1][j]+post[i−1][m−j])

若 Without<Pre[n][m]Without < Pre[n][m]Without<Pre[n][m] 则说明这个物品不能不选，反之就是必选。

同理，如果这个物品必选，那么它的最大价值为：

With=max1m−p[i](pre[i−1][j]+post[i−1][m−p[i]−j])+v[i]With = max_{1}^{m-p[i]}(pre[i-1][j] + post[i-1][m-p[i]-j]) + v[i] With=max1m−p[i]​(pre[i−1][j]+post[i−1][m−p[i]−j])+v[i]

若 With<Pre[n][m]With < Pre[n][m]With<Pre[n][m] 则说明这个物品不能必选，反之就是必不选。
