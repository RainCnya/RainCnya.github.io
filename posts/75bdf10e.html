算法学习笔记：最长上升子序列 (LIS)

算法学习笔记：最长上升子序列 (LIS)

一、 初步认知

  • 命名： 最长上升子序列 (Longest Increasing Subsequence)
  • 领域： 线性动态规划 (Linear DP) / 单串模型
  • 定义： 给定一个序列，找出其中严格单调递增且长度最长的子序列。
  • 典型设问： “求最少剔除多少个元素，能使序列变为单调递增序列？” (N−LISN - LISN−LIS)

二、 主题渗透

1. 基本思路 (O(N2)O(N^2)O(N2) DP)

  • 状态定义： dp[i] 表示以第 iii 个元素结尾的 LIS 长度。
  • 决策： 遍历 iii 之前的所有 jjj (j<ij < ij<i)。如果 a[j]<a[i]a[j] < a[i]a[j]<a[i]，说明 a[i]a[i]a[i] 可以接在 a[j]a[j]a[j] 后面。
  • 转移方程： dp[i]=max⁡j<i,a[j]<a[i](dp[j])+1dp[i] = \max_{j < i, a[j] < a[i]} (dp[j]) + 1dp[i]=maxj<i,a[j]<a[i]​(dp[j])+1。

2. 优化思路 (O(Nlog⁡N)O(N \log N)O(NlogN) 贪心 + 二分)

  • 瓶颈： 之前的思路，每次寻找最优 jjj 需要遍历，太慢了。
  • 贪心策略： 同样长度的上升子序列，结尾越小，潜力越大（越容易接上后面的数）。
  • 状态重定义： f[len] 表示长度为 len 的 LIS 的最小结尾元素。
  • 性质： f 数组是单调递增的（反证法可证）。
  • 操作： 对于每个新元素 a[i]a[i]a[i]：
    • 如果 a[i]>f[max_len]a[i] > f[max\_len]a[i]>f[max_len]，直接延长 →\to→ f[++len] = a[i]。
    • 否则，在 f 数组中二分查找第一个 ≥a[i]\ge a[i]≥a[i] 的位置并替换之（让该长度的结尾变小，增加潜力）。

3. 核心思想

  • DP 解法利用“最优子结构”；贪心解法利用“结尾越小越优”的性质结合二分查找。

4. 演示

▶
模拟 (贪心 + 二分)
  • 输入： [3, 1, 4, 1, 5, 9, 2, 6]
  • 过程：
    1. 3: f=[3], len=1
    2. 1: 1 < 3, 替换 3 →\to→ f=[1], len=1 (长度为1的LIS结尾从3变为更优的1)
    3. 4: 4 > 1, 延长 →\to→ f=[1, 4], len=2
    4. 1: 1 <= 1, 替换 1 →\to→ f=[1, 4], len=2 (不变)
    5. 5: 5 > 4, 延长 →\to→ f=[1, 4, 5], len=3
    6. 9: 9 > 5, 延长 →\to→ f=[1, 4, 5, 9], len=4
    7. 2: 2 替换 4 →\to→ f=[1, 2, 5, 9], len=4 (潜力增加)
    8. 6: 6 替换 9 →\to→ f=[1, 2, 5, 6], len=4
  • 最终答案： len=4

3. 部署细节

  • 答案位置：

    • DP 解法：答案是 max⁡(dp[i])\max(dp[i])max(dp[i])，不是 dp[n]dp[n]dp[n]。
    • 贪心解法：答案是 len，不是 f[len] 的值。

  • fff 数组： f 数组不是 LIS 本身！！！！！它只是一个辅助状态数组。

    • 因此，这个贪心写法也很难找出最长序列，只能统计长度。

  • 严格上升 vs 不下降：

    • 严格上升 (<<<): 二分找 ≥a[i]\ge a[i]≥a[i] 的位置替换。
    • 不下降 (≤\le≤): 二分找 >a[i]> a[i]>a[i] 的位置替换（upper_bound）。

五、 进阶实战与变体

基础的 LIS 模型在实际题目中常与其他知识点结合，或作为转化目标：

  • 模型转化： 当求解两个排列的 LCS 时，可转化为 LIS 问题。
    • 题解：Luogu-P1439 两个排列的 LCS
  • 状态重定义： 当序列的连接条件由“大小”变为“位运算”时，状态需定义在位域上。
    • 题解：Luogu-P4310 绝世好题
  • 专题总结： 关于 LIS 与 LCS 的模型转化与维度优化对比。
    • 专题：突破 O(N²)：LIS 与 LCS 的模型转化与维度优化
