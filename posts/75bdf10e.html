算法学习笔记：最长上升子序列 (LIS)

算法学习笔记：最长上升子序列 (LIS) -V3.2

一、 初步认知 (Macro View)

1. 叫什么 (Identity)

  • 中文名： 最长上升子序列
  • 英文名： Longest Increasing Subsequence (简称 LIS)

2. 什么地位 (Status)

  • 所属领域： 线性动态规划 (Linear DP)
  • 核心地位： 线性 DP 中最经典、最核心的模型之一。

3. 有什么用处 (Utility)

  • 解决的核心问题： 在一个序列中，找到一个严格单调递增且长度最长的子序列。
  • 典型应用场景：
    • 股票问题（最长上涨天数）。
    • 拦截导弹系统 (即 LIS 和 LDS)。
    • 许多复杂 DP 问题的子模型。

二、 主题渗透 (Theme Osmosis)

1. 核心原理 (The Eureka Moment)

解法一：O(N2)O(N^2)O(N2) 动态规划 (DP)

  • 底层思想： 动态规划 (DP)。
  • 关键推导：
      - 状态定义： dp[i]dp[i]dp[i] 表示以第 iii 个元素 a[i]a[i]a[i] 结尾的最长上升子序列的长度。
      - 状态转移： 我们需要遍历 iii 之前的所有 jjj ( 1≤j<i1 \le j < i1≤j<i )。
        - 如果 a[j]<a[i]a[j] < a[i]a[j]<a[i]，说明 a[i]a[i]a[i] 可以接在 a[j]a[j]a[j] 结尾的上升子序列后面，形成一个更长的序列。
        - 此时， dp[i]dp[i]dp[i] 可以被 dp[j]+1dp[j] + 1dp[j]+1 更新。
      - 转移方程： dp[i]=max⁡(dp[j]+1)dp[i] = \max(dp[j] + 1)dp[i]=max(dp[j]+1)，对于所有 1≤j<i1 \le j < i1≤j<i 且 a[j]<a[i]a[j] < a[i]a[j]<a[i]。
      - 初始化： dp[i]=1dp[i] = 1dp[i]=1 (每个元素自身至少构成长度为 1 的子序列)。

解法二：O(Nlog⁡N)O(N \log N)O(NlogN) 贪心 + 二分

  • 底层思想： 贪心 + 二分查找。
  • 关键推导：
    • 状态定义 (贪心)： 引入辅助数组 f。f[len]f[len]f[len] 存储的是，所有长度为 len 的上升子序列中，结尾元素最小的那一个值。
    • 贪心思想： 对于相同长度的 LIS，结尾元素越小，它在未来就越有可能被延长，因此它“更优”。我们只保留这个“最优”的结尾。
    • 关键性质： f 数组（ f[1…len]f[1 \dots len]f[1…len] ）一定是单调递增的。（因为 f[len]<f[len+1]f[len] < f[len+1]f[len]<f[len+1]）。
    • 核心推论： 既然 f 数组是有序的，我们就可以在它上面进行二分查找来优化状态转移，这就是 log⁡N\log NlogN 的来源。

3. 详细实现过程 (Step-by-Step)

O(N2)O(N^2)O(N2) 解法

  1. 初始化 (Initialization)：
        - for(int i = 1; i <= n; ++i) dp[i] = 1;
  2. 核心迭代 (Iteration)：
        - 外层循环 i 从 1 到 nnn。
        - 内层循环 j 从 1 到 i−1i-1i−1。
  3. 状态转移 (Transition)：
        - if (a[j] < a[i]) { dp[i] = max(dp[i], dp[j] + 1); }
  4. 获取答案 (Get Answer)：
        - ans = max(ans, dp[i])。

O(Nlog⁡N)O(N \log N)O(NlogN) 解法

  1. 初始化 (Initialization)：
        - len = 0。
        - f[0] = -INF (设置一个极小的哨兵)。
  2. 核心迭代 (Iteration)：
        - 循环 i 从 1 到 nnn，处理 a[i]。
  3. 状态转移 (Transition)：
        - Case 1 (延长)： 如果 a[i] > f[len]：
          - len++
          - f[len] = a[i]
        - Case 2 (替换)： 如果 a[i] <= f[len]：
          - 在 f[1 \dots len] 中二分查找第一个 ≥a[i]\ge a[i]≥a[i] 的位置 pos。
          - f[pos] = a[i]
  4. 获取答案 (Get Answer)：
        - 循环结束后，len 的值即为 LIS 的长度。

三、 总结分析 (Analysis & Comparison)

1. 复杂度分析 (Complexity)

维度 复杂度 说明
O(N2)O(N^2)O(N2) 解法 O(N2)O(N^2)O(N2) 基础 DP，两层 for 循环。
O(Nlog⁡N)O(N \log N)O(NlogN) 解法 O(Nlog⁡N)O(N \log N)O(NlogN) 贪心 + 二分。遍历 NNN 个元素，每次二分 O(log⁡N)O(\log N)O(logN)。
空间复杂度 O(N)O(N)O(N) 两种解法均需要 O(N)O(N)O(N) 的辅助数组。

2. 核心注意点 (Key Pitfalls)

  1. 状态定义： O(N2)O(N^2)O(N2) 解法的核心在于 dp[i]dp[i]dp[i] 必须定义为“以 a[i]a[i]a[i] 结尾”，如果定义为“前 iii 个元素”的 LIS，状态将无法转移。
  2. 初始化： dp[i]dp[i]dp[i] 至少为 1（它本身），必须初始化为 1。
  3. O(Nlog⁡N)O(N \log N)O(NlogN) 解法：该解法中的 f 数组（或 d 数组）的含义完全不同，f[len] 表示“长度为 len 的 LIS 的最小结尾元素”，但它不能用于还原 LIS 的具体路径。
  4. 最终答案：
    • 第一种 O(N2)O(N^2)O(N2) 的算法，最大值是 max⁡(dp[i])(1≤i≤n)\max(dp[i]) (1 \le i \le n)max(dp[i])(1≤i≤n)。
    • 而第二种 O(Nlog⁡N)O(N \log N)O(NlogN) 的算法，最大值就是 len 本身。
  5. fff 数组含义： O(Nlog⁡N)O(N \log N)O(NlogN) 解法中的 f 数组不是 LIS 本身。它只是一个贪心维护的辅助数组，f[3] = 5 只代表“长度为 3 的 LIS 最小结尾是 5”，但这个 LIS 可能是 [1, 4, 5] 或 [3, 4, 5]，数组 f 本身 [1, 2, 5, 6] 甚至不是一个合法的子序列。

所以我们能发现，时间复杂度越大的算法，能够实现的功能越强大。

而很多时候，算法竞赛中遇到的问题往往都是子集中的特殊方向优化。
