TZOJ-7670

7670：大门

本题是 差分 的经典运用。如果你还不了解差分，别担心，本文会从零开始讲解。

一. 题意分析

本题的描述有点绕，我来梳理一下：

  • 物品：有 mmm 把钥匙，编号从 111 到 mmm。
  • 条件：有 nnn 个矿洞（大门），编号从 1 到 nnn。
  • 关系： 第 iii 个矿洞给出了一个钥匙区间 [li,ri][l_i, r_i][li​,ri​]，表示钥匙 lil_ili​ 到 rir_iri​ 都能打开这个门。
  • 目标： 统计有多少把钥匙，它能打开的门的数量 ≥k\ge k≥k。

换句话说，我们要对每一把钥匙 jjj ( 1≤j≤m1 \le j \le m1≤j≤m ) 统计它能打开多少个门，我们称这个数量为 key_count[j]。

最后，我们要找出有多少个 jjj 满足 key_count[j] >= k。

二. 思路推进

1. 暴力解法

最直观的想法是：

  1. 创建一个数组 key[m+1]，全部初始化为 0。这个数组用来记录每把钥匙能开的门数。
  2. 遍历 nnn 个矿洞。对于第 iii 个矿洞，遍历它对应的钥匙区间 [li,ri][l_i, r_i][li​,ri​]。
  3. 对于区间内的每一把钥匙 jjj，都执行 key[j]++。
  4. 所有矿洞都处理完后，再遍历一次 key 数组（从 1 到 mmm），统计 key[i]>=kkey[i] >= kkey[i]>=k 的个数。

2. 差分优化

我们发现，暴力算法的瓶颈在于对每一个区间 [l,r][l,r][l,r] 上的所有元素执行 +1+1+1 操作。我们对 nnn 个区间执行了这个操作，时间开销巨大。

而这种“频繁的区间修改，最后统一查询”的情况，就是 差分 算法的用武之地。

2.1 差分是什么

可以简单理解为 是 前缀和 的逆运算。

  • 前缀和：快速求 区间和。
  • 差分：快速执行 _区间修改。

我们定义一个原数组 aaa，以及另一个差分数组 difdifdif。

dif[i]=a[i]−a[i−1]dif[i] = a[i] - a[i-1] dif[i]=a[i]−a[i−1]

这么看，aaa 数组就是 difdifdif 数组的前缀和。

a[i]=dif[1]+dif[2]+...+dif[i]a[i] = dif[1] + dif[2] + ... + dif[i] a[i]=dif[1]+dif[2]+...+dif[i]

2.3 回归本题

根据之前对题目的分析，我们发现，其实我们的瓶颈是对 [l,r][l,r][l,r] 区间的修改。

不难想到，这里我们可以采取差分数组进行优化。

  1. 原数组 keykeykey 初始为 000。差分数组 difdifdif 也初始为 000。
  2. 遍历 nnn 个矿洞。对于第 iii 个矿洞给出的区间 [l.r][l.r][l.r]，我们进行 O(1)O(1)O(1) 的差分操作：
    • dif[l]+1dif[l]+1dif[l]+1
    • dif[r+1]−1dif[r+1]-1dif[r+1]−1
  3. 在处理完 nnn 个区间后，我们想知道原数组 keykeykey 的值。所以我们采用 前缀和 来“还原”它。
  4. 我们可以在还原过程中进行统计，这样的话就不需要 sumsumsum 数组了。
