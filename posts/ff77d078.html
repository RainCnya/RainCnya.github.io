TZOJ-7670

7670：大门

本题是 差分 的经典运用。如果你还不了解差分，别担心，本文会从零开始讲解。

一. 题意分析

本题的描述有点绕，我来梳理一下：

  • 物品：有 m 把钥匙，编号从 1 到 m。
  • 条件：有 n 个矿洞（大门），编号从 1 到 n。
  • 关系： 第 i 个矿洞给出了一个钥匙区间 [li, ri]，表示钥匙 li 到 ri 都能打开这个门。
  • 目标： 统计有多少把钥匙，它能打开的门的数量  ≥ k。

换句话说，我们要对每一把钥匙 j ( 1 ≤ j ≤ m ) 统计它能打开多少个门，我们称这个数量为 key_count[j]。

最后，我们要找出有多少个 j 满足 key_count[j] >= k。

二. 思路推进

1. 暴力解法

最直观的想法是：

  1. 创建一个数组 key[m+1]，全部初始化为 0。这个数组用来记录每把钥匙能开的门数。
  2. 遍历 n 个矿洞。对于第 i 个矿洞，遍历它对应的钥匙区间 [li, ri]。
  3. 对于区间内的每一把钥匙 j，都执行 key[j]++。
  4. 所有矿洞都处理完后，再遍历一次 key 数组（从 1 到 m），统计 key[i] >  = k 的个数。

暴力
  • 复杂度分析：
    • 步骤 2 和 3 是核心。外层循环 n 次。内层循环最坏情况下（所有区间都是 [1, m]）要 m 次。
    • 总时间复杂度为 O(n × m)。
  • 问题：题目给的 n, m ≤ 1e5，所以最坏情况 O(m × n) 算法的计算量到达 1e10 的数量级，远远超过 1e8 的限制，导致 TLE。

2. 差分优化

我们发现，暴力算法的瓶颈在于对每一个区间 [l, r] 上的所有元素执行 +1 操作。我们对 n 个区间执行了这个操作，时间开销巨大。

而这种“频繁的区间修改，最后统一查询”的情况，就是 差分 算法的用武之地。

2.1 差分是什么

可以简单理解为 是 前缀和 的逆运算。

  • 前缀和：快速求 区间和。
  • 差分：快速执行 _区间修改。

我们定义一个原数组 a，以及另一个差分数组 dif。

dif[i] = a[i] − a[i − 1]

这么看，a 数组就是 dif 数组的前缀和。

a[i] = dif[1] + dif[2] + ... + dif[i]

2.2 差分的用法

  • O(1) 的区间修改 假设我们要对 a 数组的 [L, R] 区间全部 +v0，看看 dif 数组会发生什么：

这里列个表来更容易理解。

a 为 原始数列 ，dif 为 差分数列。 | —- | 1 | 2 | 3 | 4 | 5 | | —- | - | - | - | - | - | | a | 1 | 3 | 2 | 5 | 7 | | dif1 | 1 | 2 | -1 | 3 | 2 |

现在假设我要对 [2, 4] 区间上每一个数加上 2

—- 1 2 3 4 5
a 1 3 2 5 7
dif1 0 2 0 0 -2
b 1 5 4 7 7
dif2 1 4 -1 3 0

发现了吗？对于 dif 数组来说，我们只修改了 dif[2] 以及 dif[5]。

结论： 对 a 数组的 [L, R] 区间 +1，等价于只对 dif 数组进行两个单点修改：

dif[l] + + dif[r + 1] − −

这个时间复杂度只有 O(1)。

2.3 回归本题

根据之前对题目的分析，我们发现，其实我们的瓶颈是对 [l, r] 区间的修改。

不难想到，这里我们可以采取差分数组进行优化。

  1. 原数组 key 初始为 0。差分数组 dif 也初始为 0。
  2. 遍历 n 个矿洞。对于第 i 个矿洞给出的区间 [l.r]，我们进行 O(1) 的差分操作：
    • dif[l] + 1
    • dif[r + 1] − 1
  3. 在处理完 n 个区间后，我们想知道原数组 key 的值。所以我们采用 前缀和 来“还原”它。
  4. 我们可以在还原过程中进行统计，这样的话就不需要 sum 数组了。

三. 注意事项 & 复杂度分析

  • 这题需要开 long long 吗？
    • 不用，因为假设区间都 +1，一共 n 个区间。
    • 所以 current 最大值也就是 n = 1e5。
    • 因此用 int 完全足以满足题目条件。
  • 复杂度分析
    • 读入 n 个区间，并修改差分数组：O(n * 2) = O(n)
    • 遍历 m 个钥匙，通过前缀和还原：O(m)
    • 总时间复杂度 O(m + n)。
