算法学习笔记：最长公共子序列 (LCS) -V3.2

一、 初步认知 (Macro View)

1. 叫什么 (Identity)

  • 中文名： 最长公共子序列
  • 英文名： Longest Common Subsequence (简称 LCS)

2. 什么地位 (Status)

  • 所属领域： 线性动态规划 (Linear DP)
  • 核心地位： 线性 DP 中最经典、最核心的模型之一。

3. 有什么用处 (Utility)

  • 解决的核心问题： 找出两个序列中，不要求连续的、最长的共同子序列。
  • 典型应用场景：
    • 文本差异比对（diff 命令的算法原型）。
    • 生物信息学（DNA 序列对齐）。

二、 主题渗透 (Theme Osmosis)

1. 核心原理 (The Eureka Moment)

解法一：O(NM)O(NM)O(NM) 空间 动态规划

  • 底层思想： 动态规划 (DP)。
  • 关键推导：
    • 状态定义： dp[i][j] 表示字符串 aaa 的前 iii 个字符 和 字符串 bbb 的前 jjj 个字符的最长公共子序列的长度。
    • 状态转移： 核心是看 a[i] 和 b[j] 是否相等。
      • Case 1： if (a[i] == b[j])
        • dp[i][j] = dp[i-1][j-1] + 1
      • Case 2： if (a[i] != b[j])
        • dp[i][j] = max(dp[i-1][j], dp[i][j-1])

解法二：O(M)O(M)O(M) 空间 动态规划 (滚动数组)

  • 底层思想： 状态压缩 / 滚动数组。
  • 关键推导：
    • dp[i][j] 的计算只依赖于 i 和 i-1 这两行。
    • 状态定义 (压缩)： 使用 dp[0][...] 和 dp[1][...] 分别代表“上一行”和“当前行”。
    • 使用 i & 1： 用 i & 1 (奇偶性) 来区分两行。
      • curr = i & 1
      • prev = (i - 1) & 1
    • 转移方程：
      • Case 1： if (a[i] == b[j])
        • dp[curr][j] = dp[prev][j-1] + 1
      • Case 2： if (a[i] != b[j])
        • dp[curr][j] = max(dp[prev][j], dp[curr][j-1])

3. 详细实现过程 (Step-by-Step)

O(NM)O(NM)O(NM) 空间 解法

  1. 初始化：
    • 声明 dp[maxn][maxn]数组，表的第 0 行和第 0 列自动为 0。
  2. 循环迭代：
    • 外层循环 i 1→n1 \to n1→n（遍历 a）。
    • 内层循环 j 1→m1 \to m1→m（遍历 b）。
  3. 状态转移：
    • Case 1： if (a[i] == b[j]) { dp[i][j] = dp[i-1][j-1] + 1; }
    • Case 2： else { dp[i][j] = max(dp[i-1][j], dp[i][j-1]); }
  4. 获取答案：
    • 最终答案为 dp[n][m]。

O(M)O(M)O(M) 空间 (滚动数组) 解法

  1. 初始化：
    • 声明 dp[2][maxn] 数组，并初始化为 0。
  2. 循环迭代：
    • 外层循环 i 1→n1 \to n1→n。
    • int curr = i & 1; (例如：1)
    • int prev = (i - 1) & 1; (例如：0)
    • 内层循环 j 从 1→m1 \to m1→m。
  3. 状态转移：
    • Case 1： if (a[i] == b[j]) { dp[curr][j] = dp[prev][j-1] + 1; }
    • Case 2： else { dp[curr][j] = max(dp[prev][j], dp[curr][j-1]); }
  4. 获取答案：
    • 循环结束后，最终答案为 dp[n & 1][m] (即 nnn 所在行的 mmm 位置)。

三、 总结分析 (Analysis & Comparison)

1. 复杂度分析 (Complexity)

维度 复杂度 说明
O(NM)O(NM)O(NM) 空间解法 时间 O(NM)O(NM)O(NM)
空间 O(NM)O(NM)O(NM)		 基础 DP，二维数组。NNN 为 a 长度，MMM 为 b 长度。
O(M)O(M)O(M) 空间解法 时间 O(NM)O(NM)O(NM)
空间 O(2M)=O(M)O(2M) = O(M)O(2M)=O(M)		 滚动数组优化，空间复杂度与较短的字符串长度相关。

2. 核心注意点 (Key Pitfalls)

  1. 1-based 索引： 个人习惯使用 1-based 索引 (a[i] 对应 dp[i])，dp[0][0] 等边界自然为 0，从而极大简化边界判断。
  2. 子序列 vs 子串： 如果是子序列 (subsequence)，不要求连续。如果是子串 (Substring)，else 分支的转移应为 dp[i][j] = 0。
  3. 答案查询： dp[n][m]。

四、 代码模板 (Code Template)

{% fold info @O(NM) 空间 DP %}

for( int i = 1; i <= n; ++ i )
{
    for( int j = 1; j <= m; ++ j )
    {
        if( a[i] == b[j] )
        {
            dp[i][j] = dp[i-1][j-1] + 1;
        }
        else
        {
            dp[i][j] = max( dp[i-1][j], dp[i][j-1] );
        }
    }
}
// 最终答案为 dp[n][m]

{% endfold %}

{% fold info @O(M) 空间 DP_滚动数组 %}

for( int i = 1; i <= n; ++ i )
{
    int curr = i & 1;     // 0 或 1
    int prev = (i - 1) & 1; // 1 或 0

    for( int j = 1; j <= m; ++ j )
    {
        if( a[i] == b[j] )
        {
            // 依赖 [上一行][j-1]
            dp[curr][j] = dp[prev][j-1] + 1;
        }
        else
        {
            // 依赖 [上一行][j] 和 [当前行][j-1]
            dp[curr][j] = max(dp[prev][j], dp[curr][j-1]);
        }
    }
}
// 最终答案为 dp[n & 1][m]
