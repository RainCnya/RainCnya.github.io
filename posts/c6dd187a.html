算法学习笔记：冒泡排序 (Bubble Sort)

算法学习笔记：冒泡排序 (Bubble Sort)

一、 初步认知 (Macro View)

1. 叫什么 (Identity)

  • 中文名： 冒泡排序
  • 英文名： Bubble Sort

2. 什么地位 (Status)

  • 所属领域： 排序 / 基础算法
  • 核心地位： 最简单、最直观的排序算法之一。

3. 有什么用处 (Utility)

  • 解决的核心问题： 将一个杂乱无章的序列（数组）按照从小到大（或从大到小）的顺序排列。

  • 典型应用场景：

    • 算法教学，用于建立排序的基本概念。
    • 作为引子，用于深入理解排序的本质——消除“逆序对”。

二、 主题渗透 (Theme Osmosis)

1. 核心原理 (The Eureka Moment)

  • 底层思想： 比较交换 + 消除逆序对。

  • 关键推导：

    • 逆序对 (Inversion Pair)：

      • 当 i < j 且 A[i] > A[j] 时，称 (A[i], A[j]) 为一个逆序对。
      • “逆序对”的数量是衡量数组“有多乱”的指标。

    • 冒泡的本质：

      • 冒泡排序的操作非常纯粹——从头到尾依次比较相邻的两个数，如果 A[j] > A[j + 1]（构成逆序对），就交换它们。

    • 核心结论：

      • 每执行一次交换，都只会消除 (A[j], A[j + 1]) 这一个相邻的逆序对，且不会产生任何新的逆序对。因此，冒泡排序的“交换次数”精确地等于数组的“逆序对总数”。

2. 详细实现过程 (Step-by-Step)

  1. 外层循环 (Rounds)： 总共需要进行 n − 1 轮冒泡。第 i 轮的目标是把当前未排序部分的最大值“冒泡”到它最终应该在的位置（即 n − i + 1 的位置）。

  2. 内层循环 (Comparisons)： 在第 i 轮中，从头开始（j = 1），比较到未排序部分的末尾（j ≤ n − i）。

  3. 交换 (Swap)： 比较 a[j] 和 a[j + 1]。如果 a[j] > a[j + 1]，则交换两者。

3. 模拟演示 (Visualization)

▶
模拟

让我们用 [5, 4, 3, 2, 1] 这个例子来看具体操作。

  • 第一轮： (目标是把最大的数“冒泡”到最后)

    • 对比第 1 和第 2 个数：5 > 4，交换！ -> [4, 5, 3, 2, 1]
    • 对比第 2 和第 3 个数：5 > 3，交换！ -> [4, 3, 5, 2, 1]
    • 对比第 3 和第 4 个数：5 > 2，交换！ -> [4, 3, 2, 5, 1]
    • 对比第 4 和第 5 个数：5 > 1，交换！ -> [4, 3, 2, 1, 5]
    • 第一轮后，最大的 5 已经到了正确位置 14。

  • 第二轮： (目标是把第二大的数冒泡到倒数第二)

    • 我们对 [4, 3, 2, 1] 部分重复操作。
    • … 交换后 … -> [3, 2, 1, 4, 5]

  • 第三轮：

    • 我们对 [3, 2, 1] 部分重复操作。
    • … 交换后 … -> [2, 1, 3, 4, 5]。

  • 第四轮：

    • 我们对 [2, 1] 部分重复操作。
    • … 交换后 … -> [1, 2, 3, 4, 5]。

现在，整个数组完全有序了。由此我们可以得出下面的结论：

对于一个有 n 个数字的数组，我们总共需要进行 n − 1 轮这样的冒泡。

三、 总结分析 (Analysis & Comparison)

1. 复杂度分析 (Complexity)

维度 复杂度 说明
时间复杂度 O(N2) 需要 O(N) 轮，每轮最多 O(N) 次比较。总比较次数为 $\frac{N(N-1)}{2}$
空间复杂度 O(1) 原地排序，仅需常数级别的交换空间

2. 核心注意点 (Key Pitfalls)

  • 稳定性： 冒泡排序是稳定的。

    • 因为只有 A[j] > A[j + 1] 时才交换，相等的元素不会改变其相对顺序。

  • 效率低下： O(N2) 的复杂度使其在 N > 104 时基本不可用，一般仅用于教学。

  • 核心价值： 不过它的价值不在于效率，而在于它清晰地体现了“交换次数 = 逆序对数量”这一深刻联系。
