动态规划笔记

DP 的 核心是 “状态暂存”，将一个状态的统计结果传递到另一个状态。

一、概念

1. 状态

动态规划的状态可以笼统地解释为 “问题当前的局面”。

凑够 www 的金额最少需要多少张纸币；
从第 xxx 行第 yyy 个数字走到最下层的最大数字和。

某状态、某局面、某状态表示的答案（的值）、某状态代表的方案（集合）、某状态代表的有向无环图上的点，在本质上相通，有时会用一个字母（如 fif_{i}fi​ ）来表示。

可以看到，状态具有明确良好的定义，一般可以写成：（用变量表示的）当前局面的（最优）答案，或者满足某种性质的方案数。

凑够 www 的金额 -> 关于 www 的局面；从第 xxx 行第 yyy 个数字走到最下层 -> 关于 (x,y)(x, y)(x,y) 的局面。
最少需要多少张纸币 / 最大数字和 -> 这个问题所对应的最优答案。

状态的答案应该只依赖状态定义的局面和一些状态以外的常量（如纸币的面额相对于状态来说是常量）。

也就是说，在一般情况下，状态变化并不会影响状态以外的量，该状态之后的演变不受这个状态之前决策的影响。

因此，如果要用动态规划来解决一道题，应把所有可能的变动加入状态的定义中去。

2. 转移

因为要求解最终状态的答案，所以状态之间需要存在某些关系。一般把这样的计算关系称作 转移，计算式称为（状态）转移方程。

可以看到，只有依据正确的状态定义和状态转移方程，才可以有理有据地做对一道题。

状态转移方程一般具有它自己的意义，体现的是从一个状态到另一个状态的变化。

从 f(w−ai)f(w - a_{i})f(w−ai​) 转移到 f(w)f(w)f(w) 的变化是 “多取用了一张面值为 aia_{i}ai​ 的纸币”，对应于答案的变化是 “多用了一张纸币”，故需要将答案 +1+1+1。

对于 最优化问题 来说，状态转移方程的目的就是找出最优的前继情况的答案。在无特殊性质的情况下，转移方程需要找出所有的前继情况并取出最优答案。

注意，若前继状态的最优答案不能转移成为当前状态的最优答案，则该问题（至少目前这个定义的状态）不满足动态规划解决的要求。前继状态最优解与当前状态之间最优解的关联性被称为动态规划的 最优子结构 性质。

对于 计数问题 来说，转移方程的目的就是不遗漏不重复地统计局面的所有方案数。一般来说，转移方程需要找出所有前继情况并进行恰当地组合计数。

值得一提的是，转移存在两种方式，一种是 "发送型转移"，另一种是 “接受型转移”。前者枚举状态的后继，并计算对后继的贡献；后者枚举状态的前继，当即计算出状态的答案。

两种转移方式的选取主要取决于枚举前后继状态的难度和转移的复杂程度。不同的题选用恰当的转移方式，代码编写难度会显著下降。

有向无环图 DAG

接下来，借助 DAG 的概念理解 DP 为何如此高效。DAG 是一种可以表示 “依赖关系” 的图，在 DP 中，依赖关系体现为 “一个局面可以从哪些局面转移过来”。这里不容许
“相互依赖” 的情形发生：一旦两个局面可以互相抵达，那么这个问题（至少是目前这个定义的状态）不具备 DP 解决的性质。

对于纸币问题来说，所有纸币的面值都是正整数，不存在多拿了一张纸币，面值还减小了（转移到了已经计算好的状态），即状态之间不可能相互抵达。

这个状态之间不能相互到达的性质以及未来与过去无关的性质统称为 DP 的 无后效性。

把 DP 的状态和转移理解与 DAG 的点和边建立对应关系。由于 DAG 具有严格的依赖性质，这就要求在处理每个状态时，必须先处理完它的前继状态。那些没有前继（或者前继相对不完全）的状态称为 边界状态，例如纸币问题中的 f(0)f(0)f(0)。

因为转移顺序的需要，有时题意中的 DAG 和 DP 建模时的 DAG 的边恰好是相反的，即所谓的 “自底向上”。

按照拓扑排序遍历图就能保证当前的状态的前继状态全部被遍历过，恰好解决了这个问题。当然对于绝大多数 DP 问题来说，拓扑排序是明显的，有时还可以是分层的。有时，把这种明显的层次称为 阶段。

f(w)f(w)f(w) 从小到大被称为阶段，即纸币金额；g(x,y)g(x, y)g(x,y) 从下到上也被称为阶段，即数字金字塔的层。

阶段的含义呼之欲出，它体现了一种步骤，只要按照步骤执行，整个问题就可以有条不紊地（按照拓扑排序）被解决。同时阶段也和状态的定义紧密相关。只要找到了正确地阶段，就不需要刻意建图寻找拓扑排序，只要按照某种相对简单的顺序遍历整个状态数组即可。

二、流程

1. 理解题意

阅读题目信息并理解题意。

2. 寻找阶段/性质

对简单题来说要在题目中寻找可能可以成为阶段的量。

对较复杂的题则要寻找一些性质来支撑转移或状态设计。

3. 构造状态

用之前找到的阶段或者性质来构造状态。定义状态的通常形式为：

【（用变量表示的）所在局面】 的 【（最优）答案 / 满足某种性质的方案数】。

注意：这里的状态要满足 重叠子结构 性质。

4. 设计转移

依靠定义的状态以及题意来写出状态之间的转移关系。

对最优化题来说就是最优解的转移关系；对计数题来说是组合计数关系。

5. 确定边界条件

边界点通常有两类：

  • 一是先验答案点，即答案由题意确定的点，一般是作为整张 DAG 的起点；
  • 二是不完全转移点，即对于一般的状态转移方程可能会出现越界的情况的点。

这两类点的处理需要注意规避错误，特别是后一种比较难查错。

6. 优化

包括但不限于：状态优化、转移优化、数据结构加速、单调性优化（斜率优化、四边形不等式）、FFT 加速、反演计数、根号分治、CDQ 分治等。

对于计数问题和最优化问题通常有着不同的优化思路。
