最短路体系 (Shortest Path System)

1. 生态位

  • 定义: 在加权图中，寻找从源点到目标点总权值最小的路径。
  • 核心机制: 松弛 (Relaxation)。
    • 通过中间节点 uuu 尝试优化 vvv 的距离：if( dist[u] + w < dist[v] ) dist[v] = dist[u] + w。
  • 算法选型:
算法 核心逻辑 适用场景 复杂度 局限
Dijkstra 贪心 + 优先队列 非负权单源最短路 (SSSP) O(Mlog⁡N)O(M \log N)O(MlogN) 无法处理负权边
SPFA 队列 + 反复入队 负权边/判负环 Avg O(kM)O(kM)O(kM) 最坏 O(NM)O(NM)O(NM) (易被 Hack)
Floyd DP + 阶段划分 多源最短路 (APSP) / 传递闭包 O(N3)O(N^3)O(N3) N≤500N \le 500N≤500

2. 逻辑支点

  • Dijkstra 的支点: "确定的贪心"

    • 将节点分为“已确定”和“未确定”集合。每次从“未确定”中选出距离最小的点 uuu，由于边权非负，uuu 的最短路不可能再被后续节点优化，故 uuu 变为“已确定”。
    • 隐喻: 水流蔓延，优先填满低洼处。

  • SPFA 的支点: "纠错的代价"

    • 允许节点多次入队。如果 uuu 的距离被更新（纠错），那么 uuu 的邻居可能也需要随之更新。
    • 判定: 只有“刚被更新过”的节点才由资格去更新别人。

  • Floyd 的支点: "阶段性中转"

    • 状态定义: dp[k][i][j]dp[k][i][j]dp[k][i][j] 表示“仅允许经过节点 1…k1 \dots k1…k 作为中转”的最短路。
    • 转移: i→ji \to ji→j 的路径要么不经过 kkk (继承 k−1k-1k−1 阶段)，要么经过 kkk (i→k→ji \to k \to ji→k→j)。

4. 高阶建模 (Advanced Modeling)

4.1 最短路图 (Shortest Path DAG)

  • 定义: 原图中所有满足 dist(S,u)+w+dist(v,T)==dist(S,T)dist(S, u) + w + dist(v, T) == dist(S, T)dist(S,u)+w+dist(v,T)==dist(S,T) 的边 (u,v)(u, v)(u,v) 构成的子图。
  • 性质: 必定是有向无环图 (DAG)（若无 0 环）。可在其上进行 DP、拓扑排序或计数。
  • 应用:
    • 最长公共子路径: 求两对起点终点的最短路重合部分。
    • 最短路计数: 在 DAG 上做路径计数 DP。

4.2 拆点与分层图 (State Expansion)

当单纯的“位置”不足以描述状态时，将状态扩展为二维 dist[u][state]。

  • K 短路模型:
    • 定义: dist[u][0]dist[u][0]dist[u][0] 最短路，dist[u][1]dist[u][1]dist[u][1] 次短路。
    • 转移: 用新的距离 ddd 分别尝试更新最短和次短，并发生“挤出效应”（原最短变成次短）。
  • 分层图最短路:
    • 场景: 有 kkk 次免费机会、或者 kkk 次反转边的机会。
    • 建模: 将图复制为 k+1k+1k+1 层，层间连边表示消耗一次机会。

4.3 多源最短路与染色 (Coloring)

  • 场景: 求 KKK 个关键点中，最近的一对关键点的距离。
  • 策略:
    • 建立超级源点连接所有关键点（或所有关键点同时入队）。
    • 染色: 记录 color[u]color[u]color[u] 表示 uuu 属于哪个关键点的“势力范围”。
    • 相遇: 枚举边 (u,v)(u, v)(u,v)，若 color[u]≠color[v]color[u] \neq color[v]color[u]=color[v]，则说明连接了两个不同的关键点域，更新答案 dist[u]+w+dist[v]dist[u] + w + dist[v]dist[u]+w+dist[v]。
