算法学习笔记：线性动态规划 (Linear DP)

算法学习笔记：线性动态规划 (Linear DP)

一、 初步认知 (Macro View)

1. 叫什么 (Identity)

  • 中文名： 线性动态规划
  • 英文名： Linear Dynamic Programming (简称 线性 DP)

2. 什么地位 (Status)

  • 所属领域： 动态规划

  • 核心地位： 动态规划最基础、最入门的模型。

    • 它是 DP 思想的基石。

3. 有什么用处 (Utility)

  • 解决的核心问题： 在一个线性序列（如数组、字符串）上进行最优决策的问题。

  • 典型应用场景：

    • 最长上升子序列 (LIS)： 找出序列中单调递增的最长子序列。
    • 最长公共子序列 (LCS)： 找出两个序列中共有的最长子序列。
    • 最大子数组和： 找出序列中连续一段和最大的子数组。

二、 主题渗透 (Theme Osmosis)

1. 核心原理 (The Eureka Moment)

  • 底层思想： 最优子结构。

  • 关键推导：

    • 线性 DP 的状态通常是一维 dp[i] 或二维 dp[i][j]。
    • 状态 dp[i] 的含义通常只与序列的前 i 个元素相关。
    • 递推性： 状态沿着序列 单向（从左到右）发展。
      • 计算 dp[i] 时，我们所需要的所有子问题,都必须已经计算完毕
      • （如 dp[i − 1], dp[j] (其中 j < i)）。
    • dp[i] = f(dp[j])，其中 j < i。

3. 详细实现过程 (Step-by-Step)

  1. 定义状态 (Define State)： 明确 dp[i] 的含义。这是最关键的一步。

    • 易混淆点： dp[i] 是表示 “前 i 个元素的最优解”，还是表示 “以第 i 个元素结尾的最优解” ？

  2. 初始化 (Initialization)： 确定 dp 数组的初始值。

    • LIS 中，for(i=1; i<=n) dp[i] = 1;

  3. 核心迭代 (Iteration)：

    • O(N2) 模型：设置外循环 for(i = 1; i <= n; ++i)，内循环 for(j = 1; j < i; ++j) 来寻找 j 对 i 的贡献。
    • O(N) 模型：设置单循环 for(i = 1; i <= n; ++i)，此时 dp[i] 通常只依赖 dp[i − 1] 或 dp[i − 2]。

  4. 状态转移 (Transition)： 在循环体内编写转移方程，例如 if (a[j] < a[i]) dp[i] = max(dp[i], dp[j] + 1);。

  5. 获取答案 (Get Answer)：

    • 如果 dp[i] 定义为“前 i 个”的最优解，答案通常是 dp[n]。
    • 如果 dp[i] 定义为“以 i 结尾”的最优解，答案通常是 max(dp[i]) ( 1 ≤ i ≤ n)。

三、 总结分析 (Analysis & Comparison)

1. 复杂度分析 (Complexity)

维度 复杂度 说明
时间复杂度 O(N2) 或 O(N) O(N2) 是经典 LIS/LCS 的复杂度 (依赖 j < i)。
O(N) 是最大子数组和、爬楼梯的复杂度 (依赖 i − 1)。
空间复杂度 O(N) 或 O(N2) O(N) 用于一维 DP (LIS)。
O(N2) 用于二维 DP (LCS)。

2. 核心注意点 (Key Pitfalls)

  • 1. 状态定义： 必须严格区分 dp[i] 是代表“前 i 个” 还是“以 i 结尾” 。错误的定义会导致转移方程全错。
  • 2. 答案统计： 承接上一点，如果定义是“以 i 结尾”（如 LIS），最终答案不是 dp[n]，而必须遍历 dp 数组取最大值。
  • 3. 初始化： 边界 dp[1] 必须正确设置。求最小值时，非法状态应初始化为 INT_MAX。
  • 4. 1-based 索引： 使用 1-based 时，处理 dp[i − 1] 要注意 i = 1 时的边界；
    • LCS 中 dp[i][j] 对应 s1[i] 和 s2[j]，但转移可能依赖 dp[i − 1][j − 1]。
