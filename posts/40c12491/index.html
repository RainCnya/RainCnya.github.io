[Leaf] [ABC446F] Reachable Set 2

f---reachable-set-2">F - Reachable Set 2

1. 题面梗概

中译中：给一个有向图。对于每一个 k∈[1,N]k \in [1, N]k∈[1,N]，问最少删掉多少个点，能保证从 111 号点出发，无论怎么走都走不出 {1,…,k}\{1, \dots, k\}{1,…,k} 。

N,M≤3×105N, M \leq 3 \times 10^5N,M≤3×105

2. 逻辑推导

给定 NNN 次查询，总不能每次都鲨鲨去搜一遍吧，那样复杂度肯定爆。

2.1 预处理

为了快速判断一个点 vvv 能否在 “仅经过编号 ≤k\leq k≤k 的点” 的情况下被 111 号点到达，我们定义 f[v]f[v]f[v]：

  • 从节点 111 到节点 vvv 的所有路径中，路径上最大节点的编号的最小值。

如果 f[v]≤kf[v] \leq kf[v]≤k，意味着从 1→v1\to v1→v 上存在一条全部节点 ≤k\leq k≤k 的路径。

如果 f[v]>kf[v] > kf[v]>k，则意味着要到达 vvv，必须经过至少一个编号大于 kkk 的点。

2.2 类Dijkstra

计算 f[v]f[v]f[v] 本质上是一个变体的最短路问题：

  • 初始化：f[1]=1f[1] = 1f[1]=1（起点的瓶颈值设为 1），其余 f[v]=∞f[v] = \inftyf[v]=∞。

  • 转移：对于边 u→vu \to vu→v，新的潜在可能值为 max⁡(f[u],u)\max(f[u], u)max(f[u],u)。

  • 松弛：f[v]=min⁡(f[v],max⁡(f[u],u))f[v] = \min( f[v], \max( f[u], u ) )f[v]=min(f[v],max(f[u],u))。

我们利用 priority_queue 维护 fff 值，即可在 O(Mlog⁡N)O(M \log N)O(MlogN) 内完成预处理。

2.3 判定

合法性判定：对于一个确定的 kkk，怎么判断能不能只留下 1…k1 \dots k1…k ？很简单，只需要满足以下条件即可。

max⁡1≤i≤k{f[i]}≤k\max_{1 \leq i \leq k}\{f[i]\} \leq k 1≤i≤kmax​{f[i]}≤k

删除点数量：接着下一个问题，如果 kkk 合法，我们需要删掉多少个点。

我们必须删掉 v>kv > kv>k 且 f[v]≤kf[v] \leq kf[v]≤k 的点，也就是 f[v]≤k<vf[v] \leq k < vf[v]≤k<v。

这就意味着，对于每个顶点 vvv，它会对 k∈[f[v],v−1]k \in [f[v], v-1]k∈[f[v],v−1] 这个区间上都贡献一次删除。

那么这种区间修改就可以用差分来快速解决了。

2.4 结论

Dijkstra 变体 + 差分思想。
