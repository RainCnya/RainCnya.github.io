B - Pepper Addiction

1. 题意梗概

有 MMM 种货物，第 jjj 种重量为 CjC_jCj​ 克。有 NNN 个背包，第 iii 个背包只能放第 AiA_iAi​ 种货物，且该货物最多能放 BiB_iBi​ 克。

求在货物总量和每个背包上限的限制下，最多能放多少货物。

N,M≤1000N, M \leq 1000N,M≤1000

2. 逻辑推导

贪心。由于同一个货物可以随意分配给需要它的不同背包，我们可以把所有需要同一种货物的背包的上限累加起来。

统计出所有背包对每个货物的总需求量：

cnt[j]=∑Ai=jBicnt[j] = \sum_{A_{i} = j} B_{i} cnt[j]=Ai​=j∑​Bi​

对于第 jjj 个货物，最多能使用的量就是 总储备 和 总需求 的较小值，贪心即可。

C - Except and Min

1. 题意梗概

袋子里有 NNN 个球，有 QQQ 次询问，每次询问临时拿出 KKK 个指定的球，求袋子里剩下的球中数字的最小值。询问结束后把球放回。

N≤3×105;Q≤2×105;K≤5N \leq 3 \times 10^5; Q \leq 2 \times 10^5; K \leq 5N≤3×105;Q≤2×105;K≤5

2. 逻辑推导

1. 切入点

这题的突破点在于 k≤5k \leq 5k≤5 哈。最小值的可能性是固定的。

2. 思路分析

注意到 k≤5k \leq 5k≤5，意味着每次查询的答案数据规模是固定的，最坏的情况是拿出了五个最小的球，答案就是第六小的球，由此发现虽然 N≤3e5N \leq 3e5N≤3e5 但实际能用的数据只有前 666 个最小的。

那么提前把这 666 个提取出来，然后每次用取出的球 和 前六小的球 进行匹配，这样就解决了这个查询问题。

D - Integer-duplicated Path

1. 题意梗概

给一棵树，每个节点有一个点权，然后输出从 111 到 kkk 的路径上是否有重复的点。

输出 1 到每个点的查询，k∈[1,n]k \in [1, n]k∈[1,n]。

N≤2×105,Ai≤109N \leq 2 \times 10^5, A_{i} \leq 10^9N≤2×105,Ai​≤109

2. 逻辑推导

2.1 切入点

回溯型的 DFS。

2.2 思路分析

这个数据规模暗示我们需要采用最大 O(Nlog⁡N)O( N \log N )O(NlogN) 的写法，毕竟 O(N2)O( N^2 )O(N2) 就爆了。

然后我们可以想到，从根节点开始往下 DFS 搜索，同时记录当前找到了哪些数，如果新搜索到的那个点权之前有找到过，那么就把答案记录下来。

为什么不直接输出？因为 DFS 序不一定就是 1,2,…,n1, 2, \dots, n1,2,…,n 的输出顺序。

然后接着分析，如何记录当前找到了哪些数？显然可以用数组存 cnt[k] = m 表示值为 kkk 的节点出现了 mmm 次。但是这样会出现一个问题，A≤109A \leq 10^9A≤109 没法直接开这么大规模的数组。

发现值域大但是稀疏，所以我们可以考虑离散化，这里我用 map 偷懒了。

最后还有在 DFS 过程中记录：之前的节点中是否已经有重复的，如果有，那么即使当前值不重复，它的这条路径上也是存在重复的。

最后在理一下复杂度，DFS 一遍整个树 O(N)O( N )O(N)（ 每个点搜一遍 ），map / 离散化 O(Nlog⁡N)O( N \log N )O(NlogN)，map 则是插入查询均为 O(log⁡N)O(\log N)O(logN)，离散化是排序 O(Nlog⁡N)O( N \log N )O(NlogN)，查询是 O(1)O(1)O(1)。

E - Simple Division

前置芝士：基础数论 | 矩阵快速幂

1. 题意梗概

给定整数 NNN 的游程编码表示（ 由 KKK 段组成，第 iii 段是连续 lil_ili​ 个数字 cic_ici​ ）和一个整数 MMM。 求 ⌊N/M⌋(mod10007)\lfloor N / M \rfloor \pmod{10007}⌊N/M⌋(mod10007) 的值。

M≤104;K≤105;li≤109M \leq 10^4; K \leq 10^5; l_i \leq 10^9M≤104;K≤105;li​≤109

2. 逻辑推导

看不懂题面，看一下样例：

所以 N=316227766N = 316227766N=316227766，对比一下应该是能看得懂的吧。

这里插一道类似的题：P3216 [HNOI2011] 数学作业 - 洛谷 大体思路差不多。

2.1 切入点

这里我简单推一下公式，因为这个公式比较抽象：

⌊NM⌋=N−N mod MM\left\lfloor \frac{N}{M} \right\rfloor = \frac{N - N \bmod M}{M} ⌊MN​⌋=MN−NmodM​

然后我们要求：

⌊NM⌋(modP)=N−N mod MM(modP)\left\lfloor \frac{N}{M} \right\rfloor \pmod P = \frac{N - N \bmod M}{M} \pmod P ⌊MN​⌋(modP)=MN−NmodM​(modP)

因为模意义下的除法很麻烦，而且这里如果 MMM 和 PPP 不互质的话，就没法求逆元。

所以这里我采用一个类似换元法的方法接着推：设 R=N(modPM)R = N \pmod{PM}R=N(modPM)，那么 N=k(PM)+RN = k(PM) + RN=k(PM)+R，代入式子得：

⌊NM⌋=⌊k(PM)+RM⌋=⌊kP+RM⌋\left\lfloor \frac{N}{M} \right\rfloor = \left\lfloor \frac{k(PM)+R}{M} \right\rfloor = \left\lfloor kP + \frac{R}{M} \right\rfloor ⌊MN​⌋=⌊Mk(PM)+R​⌋=⌊kP+MR​⌋

最后对其取模 PPP，可得：

⌊kP+RM⌋(modP)=⌊RM⌋(modP)\left\lfloor kP + \frac{R}{M} \right\rfloor \pmod P = \left\lfloor \frac{R}{M} \right\rfloor \pmod P ⌊kP+MR​⌋(modP)=⌊MR​⌋(modP)

好，至此，我们得到了一个能用的算子，问题就转化为了求出大数 NNN 在模 (PM)(PM)(PM) 意义下的值，然后除以 MMM 即可。

C 语言中的除法自然向下取整哈

2.2 思路分析

接着我们捋一捋这个大数 NNN 怎么推，很简单的思路就是 X=X⋅10+c0X = X \cdot 10 + c_{0}X=X⋅10+c0​，简单的递推就行。

但是！！！ li≤109l_{i} \leq 10^9li​≤109，如果线性递推的话，直接就 TLE 了，接着考虑优化。

首先这是个递推关系，然后复杂度太高了，所以我们可以考虑矩阵快速幂加速。

下面简要写一下这个矩阵是怎么构造的，具体线代的知识咱不在这里展开了:

根据递推式 X=X⋅10+1⋅cX = X \cdot 10 + 1 \cdot cX=X⋅10+1⋅c，我们发现我们其实需要维护两个状态 XXX 和 111，因此答案矩阵为：

[X1]\begin{bmatrix} X \\ 1 \\ \end{bmatrix} [X1​]

答案矩阵一般都是单列的，需要什么放什么，注意是必要的状态。

然后构造转移矩阵：

[10c001]\begin{bmatrix} 10 & c_{0} \\ 0 & 1 \\ \end{bmatrix} [100​c0​1​]

怎么来的？我可以待定系数法解一遍：

[abcd]×[X1]=[aX+bcX+d]\begin{bmatrix} a & b \\ c & d \\ \end{bmatrix} \times \begin{bmatrix} X \\ 1 \\ \end{bmatrix} = \begin{bmatrix} aX + b \\ cX + d \\ \end{bmatrix} [ac​bd​]×[X1​]=[aX+bcX+d​]

然后对照递推式，解得 a=10,b=c0,c=0,d=1a = 10, b = c_{0}, c = 0, d = 1a=10,b=c0​,c=0,d=1，是这么来的。

这块内容也没有涉及到很高深的线性代数，它的使用只需要了解基本的矩阵乘法概念，然后就可以很简单地写出转移了，作用就是把 O(N)O(N)O(N) 的线性递推，用快速幂思想加速。

这样就很轻松能在 O(log⁡N)O( \log N )O(logN) 的时间内求出这个大数 NNN 了，这个代码前半部分不写注释了，你完全可以当做是一个模板，只需要你掌握待定系数法，你就能写。
