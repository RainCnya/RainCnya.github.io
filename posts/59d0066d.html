Luogu-P5851 / TZOJ-1704 Greedy Pie Eaters P

题目简述

难度：提高+(9403) | 题解格式：V6.2

有 NNN 个派和 MMM 头奶牛。每头奶牛有一个体重 wiw_iwi​ 和想吃的区间 [li,ri][l_i, r_i][li​,ri​]。

你需要安排一个奶牛的进食序列。轮到某头奶牛吃时，它会吃掉区间 [li,ri][l_i, r_i][li​,ri​] 内所有剩余的派。

若一头奶牛能吃到至少一个派，则获得 wiw_iwi​ 的权重。求最大总权重。

  • 数据范围：N≤300,M≤N(N−1)2N \le 300, M \le \frac{N(N-1)}{2}N≤300,M≤2N(N−1)​。

解题思路

1. 初步思考

通过题目中所给的状态我们可以想到使用区间DP来进行求解。

  • 正向思考：如果枚举“谁第一个吃”，状态会变得非常复杂，因为剩下的派是不连续的，很难用简单的区间表示。

  • 那逆向呢？：考虑区间 [i,j][i, j][i,j]。我们想把它划分成更小的子问题。

    • 我们可以枚举区间内的某一个派 kkk (i≤k≤ji \le k \le ji≤k≤j)。

    • 假设派 kkk 被某头奶牛 CCC 吃掉了。

    • 如果我们假设 CCC 是最后一个吃掉派 kkk 的牛（在所有覆盖 kkk 且被选中的牛中），那么在 CCC 进食之前，区间 [i,k−1][i, k-1][i,k−1] 和 [k+1,j][k+1, j][k+1,j] 必须已经被处理完毕（被其他牛吃过了，或者留给 CCC 吃）。

    • 这样，问题就分裂成了 [i,k−1][i, k-1][i,k−1] 和 [k+1,j][k+1, j][k+1,j] 两个独立的子问题，再加上奶牛 CCC 的贡献 wCw_CwC​。

2. 核心转化 (区间 DP)

状态定义：

  • dp[i][j]dp[i][j]dp[i][j]：区间 [i,j][i, j][i,j] 内能获得的最大总权重。

转移方程：

对于区间 [i,j][i, j][i,j]，我们枚举分割点 i≤k≤ji \le k \le ji≤k≤j。

dp[i][j]=max⁡k=ij{dp[i][k−1]+dp[k+1][j]+val(i,j,k)}dp[i][j] = \max_{k=i}^j \{ dp[i][k-1] + dp[k+1][j] + \text{val}(i, j, k) \} dp[i][j]=k=imaxj​{dp[i][k−1]+dp[k+1][j]+val(i,j,k)}

其中 val(i,j,k)\text{val}(i, j, k)val(i,j,k) 表示：在区间 [i,j][i, j][i,j] 范围内，能覆盖点 kkk 的最大权重的奶牛。

  • 为什么是覆盖 kkk？因为 kkk 是我们的分割点，它必须被吃掉，这连接了左右两边。选取覆盖 kkk 的最优奶牛，相当于让它“负责”吃掉 kkk（以及它范围内其他剩余的派）。

辅助 DP (预处理 val)：

为了快速求出 val(i,j,k)\text{val}(i, j, k)val(i,j,k)，我们定义辅助数组 g[i][j][k]g[i][j][k]g[i][j][k]。

  • 定义：g[i][j][k]g[i][j][k]g[i][j][k] 表示满足 i≤l≤k≤r≤ji \le l \le k \le r \le ji≤l≤k≤r≤j 的所有奶牛中，体重的最大值。

  • 转移：这是一个从小区间向大区间扩展的过程。

    g[i][j][k]=max⁡(g[i+1][j][k],g[i][j−1][k],cow[i][j])g[i][j][k] = \max(g[i+1][j][k], g[i][j-1][k], \text{cow}[i][j]) g[i][j][k]=max(g[i+1][j][k],g[i][j−1][k],cow[i][j])

    • 这里的 cow[i][j]\text{cow}[i][j]cow[i][j] 指的是区间恰好为 [i,j][i, j][i,j] 的奶牛体重。

  • 空间优化：注意到计算 g[i][j][...]g[i][j][...]g[i][j][...] 只依赖于长度更小的区间（i+1i+1i+1 或 j−1j-1j−1），因此可以使用滚动数组优化长度，将 O(N3)O(N^3)O(N3) 空间降至 O(N2)O(N^2)O(N2)。

复杂度：

  • 时间：O(N3)O(N^3)O(N3)。枚举区间长度、起点、分割点 kkk。

  • 空间：O(N2)O(N^2)O(N2)（滚动后）。

四、 结语

这道题是区间 DP 的一个变式，难点在于状态转移的设计。

常规的区间 DP 往往是 dp[i][j]=max⁡(dp[i][k]+dp[k+1][j])dp[i][j] = \max(dp[i][k] + dp[k+1][j])dp[i][j]=max(dp[i][k]+dp[k+1][j])（简单的合并）。

而本题引入了“区间内覆盖某一点的最大权值”这一子问题，并需要用另一个 DP ( ggg 数组) 来维护它。

其实通过区间 DP 的一般化想法，很容易能想到这么构造，主要在于拆分子区间求解。

关于空间优化：

如果不优化，g[300][300][300]g[300][300][300]g[300][300][300] 的 long long 数组大约需要 200MB 内存，在这题 128MB 空间限制下就爆了。所以需要采取滚动数组优化将其降至 O(N2)O(N^2)O(N2)。
