C - Drop Blocks

题意

有两种操作，一种是给第 xxx 个格子 +1+1+1，一种是查询有多少个格子 ≥y\ge y≥y，如果所有的格子都 ≥1\ge 1≥1，那么所有的格子都减去 111。

N,Q≤3e5,x≤N,y≤3e5N, Q \le 3e5, x \le N, y \le 3e5N,Q≤3e5,x≤N,y≤3e5

思路

如果暴力模拟每次全局减一的操作，复杂度最坏是 O(NQ)\mathcal{O}(NQ)O(NQ)，显然不可取。

瓶颈在于，当 “所有格子都 ≥1\ge 1≥1” 这个条件被触发时，我们不能真的去遍历整个数组减一。为此可以维护一个全局删除次数 del，记录到目前为止已经执行过多少轮全体减一。

令 CiC_iCi​ 表示第 iii 个格子被 +1+1+1 的总次数，那么实际上该格子当前的方块数是：Di=Ci−min⁡(C)D_i = C_i - \min(C)Di​=Ci​−min(C) 其中 min⁡(C)\min(C)min(C) 就是全局删除次数 del。

对于查询 Di≥yD_i \ge yDi​≥y，等价于： Ci≥y+min⁡(C)=y+delC_i \ge y + \min(C) = y + delCi​≥y+min(C)=y+del。因此，我们只需要动态维护每个 CiC_iCi​ 值的出现情况，然后在查询时快速统计有多少个 Ci≥y+delC_i \ge y + delCi​≥y+del。

为了回答查询，我们需要高效的数据结构来维护每个 CiC_{i}Ci​ 值出现的频次，于是我就想到了权值树状数组，But，这是 C 题啊，怎么就树状数组上了呢？

重新回到题目设问，考虑 CiC_{i}Ci​ 变化时对答案的影响，当 CxC_{x}Cx​ 从 v−1v-1v−1 变成 vvv 的时候，对于所有的统计情况有什么影响呢？发现在这之前满足 ≥1,≥2,…,≥v−1\ge 1, \ge 2, \dots, \ge v - 1≥1,≥2,…,≥v−1，现在依然满足，而同时产生的新贡献则是 ≥v\ge v≥v 的格子多了一个。

所以我们可以只维护一个计数数组 cnt[v]，表示 Ci≥vC_i \ge vCi​≥v 的格子个数。然后就好操作了，只需要将该格子的累计次数 C[x] ++，然后 cnt[C[x]] ++ 就行了，最后检查，如果 cnt[del + 1] == n，就说明所有的格子都至少为 del + 1 了，此时 del ++。

最后注意数据规模即可，这种写法数组得开两倍，因为当 del 很大的时候 y + del 会超过 3e53e53e5。

D - Adjacent Distinct String

题意

给定一个字符串 SSS，判断能否通过重排字符，使得任意两个相邻字符都不相同，若可以则输出任意一种方案。

T≤3e5,∣S∣≤1e6T \le 3e5, |S| \le 1e6T≤3e5,∣S∣≤1e6

思路

大胆猜测贪心构造，搓了搓样例，有一个贪心策略是，把所有字母出现的次数都记录下来，然后每次都拿出数量最多的字符拼在当前答案后面，然后保证相邻两个字符不同即可。

如果最后构造完发现字符串长度不够，就说明这是不可能构造的，我不会证明，但是感觉应该挺对的。最后选择使用 优先队列 来处理，特殊处理堆顶出队后入队还是堆顶的情况就行。

复杂度：每个测试点 O(∣S∣log⁡26)=O(∣S∣)O(|S|\log 26) = O(|S|)O(∣S∣log26)=O(∣S∣)，总复杂度 O(∑∣S∣)O(\sum |S|)O(∑∣S∣)，感觉比 C 简单多了 ......

E - Select from Subtrees

题意

给定一棵以 111 为根的有根树，共 NNN 个结点。结点 iii 有 CiC_iCi​ 个糖果，所有糖果互不相同。有 NNN 只松鼠，第 iii 只松鼠需要从以 iii 为根的子树中选取 DiD_iDi​ 个糖果，且不同松鼠不能选取同一个糖果。求所有松鼠成功选取糖果的方案数。

N≤2e5,C≤1e9,∑D≤1e6N \le 2e5, C \le 1e9, \sum D \le 1e6N≤2e5,C≤1e9,∑D≤1e6

思路

这种方案数的问题大概率是考察 DP，看看数据的形式 —— 是树，大概率是树上 DP，一般来说，我们的思路是从叶子节点往根节点倒推。

对于以 uuu 为根节点的子树，设该子树的糖果总数为 SumCSumCSumC，松鼠糖果总需求 SumDSumDSumD，有一个很显然的结论，如果 SumC<SumDSumC < SumDSumC<SumD，那么一定是供不应求的。

对于 uuu 的每一个子节点 vvv ，其子树的松鼠挑选完糖果后，还剩下 SumCv−SumDvSumC_{v} - SumD_{v}SumCv​−SumDv​ 颗糖果，也就是说，uuu 这一点的糖果总数为 cntu=Cu+∑v:u(SumCv−SumDv)=SumCu−SumDu+Ducnt_{u} = C_{u} + \sum_{v : u} (SumC_{v} - SumD_{v} ) = SumC_{u} - SumD_{u} + D_{u}cntu​=Cu​+∑v:u​(SumCv​−SumDv​)=SumCu​−SumDu​+Du​。

关键的性质在于这句话，“即使最终选择的糖果组是相同的，如果选择它们的松鼠不同，它们也会被算作不同的方式”。也就说所有的糖果对于节点 uuu 以及它的祖先来说都是等价的，那么总方案数就是：

(cntuDu)=(SumCu−SumDu+DuDu)\binom{cnt_{u}}{D_{u}} = \binom{SumC_{u} - SumD_{u} + D_{u}}{D_{u}} (Du​cntu​​)=(Du​SumCu​−SumDu​+Du​​)

注：因为是等价的，所以只需要算组合数就行了，不需要计算排列数。

最后把答案再运用乘法法则把答案乘起来就得到了结果：

∏i=1n(SumCi−SumDi+DiDi)(mod998244353)\prod_{i=1}^{n} \binom{SumC_{i} - SumD_{i} + D_{i}}{D_{i}} \pmod{998244353} i=1∏n​(Di​SumCi​−SumDi​+Di​​)(mod998244353)

SumCSumCSumC 可能非常大（可达 10910^9109 及以上），但 Du≤106D_u \le 10^6Du​≤106，所以不能使用简单的阶乘预处理解决。因此可以回归组合数公式本身：

(nr)=n⋅(n−1)⋯(n−r+1)r!\binom{n}{r} = \frac{n \cdot (n-1) \cdots (n-r+1)}{r!} (rn​)=r!n⋅(n−1)⋯(n−r+1)​

只需要计算分子，最后乘以分母的逆元，分母的逆元范围很小，可以提前预处理。最后写一个 DFS，从叶子结点开始，逐个倒推合并结果即可得到答案。
