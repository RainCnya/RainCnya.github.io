算法学习笔记：单调队列 (Monotonic Queue)

算法学习笔记：单调队列 (Monotonic Queue)

一、 初步认知 (Macro View)

1. 叫什么 (Identity)

  • 中文名： 单调队列
  • 英文名： Monotonic Queue

2. 什么地位 (Status)

  • 所属领域： 数据结构 / 基础算法
  • 核心地位： 一种用于优化特定问题（尤其是动态规划和滑动窗口）的高效辅助数据结构。

3. 有什么用处 (Utility)

  • 解决的核心问题： 在移动的区间（滑动窗口）内，快速（O(1)O(1)O(1)） 查询最值（最大值或最小值）。
  • 典型应用场景：
    • 求解固定长度的滑动窗口最大/最小值问题。（例如：洛谷 P1886）
    • 优化某些 DP 方程的转移（例如，将 O(N2)O(N^2)O(N2) 优化到 O(N)O(N)O(N)）。

二、 主题渗透 (Theme Osmosis)

1. 核心原理 (The Eureka Moment)

  • 底层思想： 空间换时间 + 过滤冗余。
  • 关键推导：
    • 队列只存储 “有潜力” 成为最值的元素。
    • （以维护单调递减队列为例，求窗口最大值）：
    • 1. 淘汰（队尾）： 当一个新元素 a[i]a[i]a[i] 准备入队时，如果它比队尾元素 a[q.back()]a[q.back( )]a[q.back()] 大，说明队尾元素 “既不如此时新，也不如此时强”。
      • 它在 iii 之后不可能再成为最大值（因为 a[i]a[i]a[i] 更优且活得更久），因此队尾元素 a[q.back()]a[q.back( )]a[q.back()] 必须出队（q.pop_back( )）。
    • 2. 淘汰（队首）： 当队首元素 q.front()q.front( )q.front() 的索引已经滑出当前窗口 [i−k+1,i][i-k+1, i][i−k+1,i] 时，它（即使再大）也已“过期”，必须出队（q.pop_front( )）。
    • 核心： 经过上述淘汰后，队首 q.front( ) 始终是当前窗口 [i−k+1,i][i-k+1, i][i−k+1,i] 内的最大（或最小）值的索引。

3. 详细实现过程 (Step-by-Step)

  1. 初始化 (Initialization)： 定义数组 a[] 存储元素，数组 q[] 模拟队列（存储索引）。初始化队首 hh = 0，队尾 tt = -1 (表示空队列)。
  2. 核心迭代 (Iteration)： 循环遍历 i 从 1 到 nnn：
    • a. (检查队首) 淘汰过期元素： while (hh <= tt && q[hh] <= i - k) { hh++ }。
    • b. (维护单调性) 淘汰冗余元素： （以求最大值为例）while (hh <= tt && a[q[tt]] <= a[i]) { tt-- }。
    • c. (入队) 插入当前元素： q[++tt] = i;
    • d. (获取答案) ： 当 i >= k 时（即第一个完整窗口形成时），窗口形成，此时的队首 q[hh] 即为当前窗口 [i−k+1,i][i-k+1, i][i−k+1,i] 的最值索引，答案是 a[q[hh]]。
  3. 终止条件 (Termination)： 循环 i 结束。

三、 总结分析 (Analysis & Comparison)

1. 复杂度分析 (Complexity)

维度 复杂度 说明
时间复杂度 O(N)O(N)O(N) NNN 为元素总数。每个元素最多入队一次，出队一次。因此总操作次数是线性的 O(2N)O(2N)O(2N)，均摊复杂度为 O(1)O(1)O(1)。
空间复杂度 O(N)O(N)O(N) 在最坏情况下（例如数组完全单调递减时），队列 q 需要存储所有 NNN 个元素的索引。

2. 核心注意点 (Key Pitfalls)

  • 易错点 1 (存储内容)： 队列 q 中存储的必须是索引 (index)，而不是元素的值 (value)。因为我们必须通过索引来判断队首元素是否“过期”。
  • 易错点 2 (淘汰逻辑)：
    • 队首 (hh) 淘汰：看索引是否过期（q[hh] <= i - k）。
    • 队尾 (tt) 淘汰：看值是否破坏单调性（a[q[tt]] <= a[i]）。
  • 易错点 3 (边界检查)：
    • 检查队列是否为空 ( hh <= tt ) 必须在所有 q[hh] 或 q[tt] 访问之前。
  • 易错点 4 (1-based 索引)： 使用 1-based 索引时，窗口的左边界是 i−k+1i-k+1i−k+1。因此判断队首 q[hh] 是否过期的条件是 q[hh] < i - k + 1，即 q[hh] <= i - k。
