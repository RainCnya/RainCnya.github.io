C - AtCoder Riko

1. 题意梗概

给定一个包含 NNN 个正整数的序列 AAA。这些数字是由若干根长度为 LLL 的零食经过摇晃后产生的。每根原始长度为 LLL 的零食要么保持完整，要么断成两截（长度和为 LLL）。

求所有可能的原始长度 LLL。

2. 逻辑推导

这其实是一个构造问题，注意到每个零食只有两种情况，所以我们可以把目光转移到最大值上。

  • 情况一：AmaxA_{max}Amax​ 是一根完整的零食。那么 L=AmaxL = A_{max}L=Amax​。此时需要验证剩下的所有元素能否两两配对成 AmaxA_{max}Amax​。

  • 情况二：AmaxA_{max}Amax​ 也是断裂后的碎片之一。这意味着没有任何一根零食是完整的。此时，AmaxA_{max}Amax​ 必须与序列中最小的元素 AminA_{min}Amin​ 配对。即 L=Amin+AmaxL = A_{min} + A_{max}L=Amin​+Amax​。

因此，可能的 LLL 只有两个可能得情况。我们只需对这两个值分别进行 O(N)O(N)O(N) 的双指针配对检验即可。

E - Sparse Range

1. 题意梗概

给定一个长度为 NNN 的数组 AAA 和一个正整数 DDD。

求有多少子区间 (L,R)(L, R)(L,R) 满足：区间内任意两个元素之差的绝对值都 ≥D\ge D≥D。

2. 逻辑推导

其实看到这个 子区间，计数，范围约束。很容易会想到滑动窗口（双指针）的写法。

起手就可以固定右端点 RRR，尝试寻找最小的合法左端点 LLL。

如果区间 [L,R][L, R][L,R] 合法，那么它的所有子区间 [L+1,R],…,[R,R][L+1, R], \dots, [R, R][L+1,R],…,[R,R] 也都合法。

当加入新元素 ARA_RAR​ 时，我们需要检查它是否与当前窗口 [L,R−1][L, R-1][L,R−1] 内的元素冲突。

即 [L,R−1][L,R-1][L,R−1] 内的元素是否存在一个值在区间 (AR−D,AR+D)(A_R - D, A_R + D)(AR​−D,AR​+D)。

前面都很简单，都很好想，但问题就在这里，这个判断怎么写呢？

首先排除暴力，因为一旦暴力复杂度就是 N2N^2N2，原地爆炸，双指针优化了个寂寞
（bushi。

我们需要一种能够快速进行范围查询的数据结构。

2.1 辅助数据结构

  1. 权值树状数组

不难想到可以用权值树状数组来维护这个查询，但我们回到数据范围 N≤4×105N \leq 4 \times 10^5N≤4×105，A1≤109A_1 \leq 10^9A1​≤109，D≤109D \leq 10^9D≤109。

这个值域可不好维护啊，注意到 N≤4×105N \leq 4 \times 10^5N≤4×105，好像可以离散化处理，这样就是可以接受范围的权值树状数组了。

Anyway，这里就会出现新的问题，因为每次加入的右端点之后，新的限制值域 (AR−D,AR+D)(A_R - D, A_R + D)(AR​−D,AR​+D) 这两个边界的值没有离散化，所以这样离散化很麻烦，很难处理（理论上应该是可行的，不推荐）。

  1. 平衡树

利用红黑树的特性，我们可以在 O(log⁡N)O(\log N)O(logN) 的时间内完成这个动态维护和查找。

只需要 lower_bound( A[r] - D + 1 ) 找到窗口内第一个 ≥AR−D+1\geq A_R - D + 1≥AR​−D+1 的数。

然后再判断这个数是否冲突，即是否 <AR+D< A_R + D<AR​+D，如果冲突就删掉，收缩窗口。

2.2 set or multiset

直观上，如果数组中存在重复元素，应该使用 multiset。但结合题意分析：如果数组中存在两个相同的数，则它们的差值为 000，而题目给定的 D≥1D \geq 1D≥1，这显然是矛盾的。

这就意味着，合法区间内是不允许出现重复元素的，因此这里使用 set 也是能 AC 的，但逻辑上可能会有点奇怪。

不过这里我采用 multiset 维护窗口中的元素（赛场上没想到上面这个点）。
