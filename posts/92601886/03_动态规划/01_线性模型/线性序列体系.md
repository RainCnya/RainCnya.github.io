## 1. 生态位

* **定义**: 状态仅与序列的**位置 (Index)** 有关，转移通常依赖前缀 $i-1$ 或 $1 \dots i-1$。
* **解决痛点**:
    * **LIS (最长上升子序列)**: 选出最长的严格单调递增子序列。
    * **LCS (最长公共子序列)**: 两个序列中最长的公共部分。
* **竞品对比**:
    * **$O(N^2)$ LIS**: 记录具体方案时使用。
    * **$O(N \log N)$ LIS**: 仅求长度或利用贪心性质时使用。

## 2. 逻辑支点

* **LIS 的贪心二分优化**:
    * **核心直觉**: 同样长度的上升子序列，**结尾元素越小**，它后面能接纳的元素就越多，潜力越大。
    * **状态重定义**: $g[len]$ 表示长度为 $len$ 的上升子序列的**最小结尾元素**。
    * **不变量**: $g$ 数组始终保持**单调递增** (Strictly Increasing)，这是可以使用 `lower_bound` 的前提。
* **LCS 的排列转化**:
    * **场景**: 当序列 $A, B$ 均为 $1 \sim N$ 的**排列**时。
    * **策略**: 将 $A$ 映射为基准 $1, 2, \dots, N$，将 $B$ 按映射规则转换。此时 $A, B$ 的公共子序列等价于 $B'$ 的上升子序列。
    * **复杂度**: $O(N^2) \to O(N \log N)$。

## 3. 实战部署

### 3.1 LIS ($O(N \log N)$)

{% fold info @Code: LIS Binary Search %}
```cpp
// 求解最长上升子序列长度
// g[k]: 长度为 k 的 LIS 的最小结尾
vector< int > g;

void solve( ) 
{
    for( int x : a ) 
    {
        // 若当前元素大于 g 末尾，直接延长 (Strictly Increasing)
        if( g.empty() || x > g.back() ) 
        {
            g.push_back( x );
        }
        else 
        {
            // 否则，找到第一个 >= x 的位置替换，保留“更小结尾”的潜能
            // airy style: strictly increasing -> lower_bound
            auto it = lower_bound( g.begin(), g.end(), x );
            *it = x;
        }
    }
    // ans = g.size()
}
```

{% endfold %}

### 3.2 LCS (Standard $O(N^2)$)

{% fold info @Code: LCS %}

```C++
// dp[i][j]: A 前 i 个与 B 前 j 个的 LCS
for( int i = 1; i <= n; ++ i )
{
    for( int j = 1; j <= m; ++ j )
    {
        if( a[i] == b[j] ) 
        {
            dp[i][j] = dp[i-1][j-1] + 1;
        }
        else 
        {
            dp[i][j] = max( dp[i-1][j], dp[i][j-1] );
        }
    }
}
```

{% endfold %}

## 4. 知识粘附

- **Dilworth 定理**:
    
    - **内容**: 偏序集的最长链长度等于最小反链覆盖数。
        
    - **应用**: 最长上升子序列长度 = 最少(非严格)下降子序列覆盖数。
        
    - **例子**: 一套导弹拦截系统拦截高度递减，要拦截所有导弹最少需要几套系统？ $\iff$ 求原序列的最长上升子序列。
