## 1. 生态位

* **定义**: 状态定义在**连续区间 $[i, j]$** 上，通过枚举**分割点 $k$** 将大区间分裂为子区间的决策模型。
* **解决痛点**:
    * **合并类问题**: 石子合并、能量项链 (局部最优无法推导全局最优，必须枚举分割)。
    * **递归结构**: 括号匹配、树的遍历序 (需结合递归性质)。

## 2. 逻辑支点

* **区间合并**:
    * $dp[i][j] = \max/\min \{ dp[i][k] + dp[k+1][j] \} + Cost(i, j)$。
    * **枚举顺序**: 先枚举**区间长度 (len)**，再枚举**左端点 (i)**。
* **断环成链 (The Broken Ring)**:
    * 策略: 将长度为 $N$ 的环复制一遍接在末尾，形成长度 $2N$ 的链。
    * 答案: 在 $dp[i][i+N-1]$ 中取最优值 ($1 \le i \le N$)。

## 3. 实战部署

### 3.1 石子合并 (区间 DP)

{% fold info @Code: Stone Merge %}
```cpp
// sum[i] 为前缀和
// dp[i][j] 表示合并 i..j 的最小代价
for( int len = 2; len <= n; ++ len ) // 阶段：长度
{
    for( int i = 1; i + len - 1 <= n; ++ i ) // 状态：左端点
    {
        int j = i + len - 1;
        dp[i][j] = INF;
        for( int k = i; k < j; ++ k ) // 决策：分割点
        {
            dp[i][j] = min( dp[i][j], 
                            dp[i][k] + dp[k+1][j] + sum[j] - sum[i-1] );
        }
    }
}
```

{% endfold %}

### 3.2 递归结构 (CF149D 括号涂色)

- **逻辑分支**: 处理区间 $[l, r]$ 时，需区分它是**嵌套结构** `(...)` 还是**拼接结构** `AB`。
    

{% fold info @Code: Recursive Interval %}

```C++
// match[l] == r: Nested ( ... )
// match[l] < r : Concatenated A B
ll dfs( int l, int r, int cl, int cr ) 
{
    if( l + 1 == r ) return check(cl, cr); // Base case
    if( memo[l][r][cl][cr] != -1 ) return memo[l][r][cl][cr];
    
    ll ans = 0;
    if( match[l] == r ) // Nested
    {
        if( ! check_valid(cl, cr) ) return 0;
        for( int ncl = 0; ncl < 3; ++ ncl )
            for( int ncr = 0; ncr < 3; ++ ncr )
                 if( ! conflict(cl, ncl) && ! conflict(cr, ncr) )
                     ans += dfs( l + 1, r - 1, ncl, ncr );
    }
    else // Concatenated: split at match[l]
    {
        int k = match[l];
        for( int mcl = 0; mcl < 3; ++ mcl )
            for( int mcr = 0; mcr < 3; ++ mcr )
                if( ! conflict(mcl, mcr) )
                    ans += dfs( l, k, cl, mcl ) * dfs( k + 1, r, mcr, cr );
    }
    return memo[l][r][cl][cr] = ans % MOD;
}
```

{% endfold %}
