## 1. 生态位

* **定义**: 通过分析题目特定的**物理约束**或**逻辑顺序**，对 DP 的状态空间或转移路径进行剪枝和重构。
* **解决痛点**:
    * **无效时间**: 时间轴过长但事件稀疏 (P5017 摆渡车)。
    * **耦合困境**: 正向操作导致状态相互缠绕，无法拆分 (P5851 压路机)。

## 2. 逻辑支点

* **Path Compression (路径压缩)**:
    * **场景**: P5017 摆渡车。
    * **策略**: 若两批人的到达间隔 $> 2m$，中间的等待是无效的。强制将时间轴压缩。
* **Reverse Logic (逆向区间)**:
    * **场景**: P5851. 寻找一段路，使得覆盖它的线段权重最大。
    * **转化**: 正向思考很难决定“当前线段是否保留”。**逆向**思考：枚举断点 $k$，将区间 $[i, j]$ 分裂，利用辅助数组 $g[i][j]$ 记录覆盖全区间的最大权重。

## 3. 实战部署

### 3.1 摆渡车 (时间压缩)

{% fold info @Code: Time Compression %}
```cpp
// 1. Compress Time
if( a[i] - a[i-1] > 2 * m ) 
    offset += (a[i] - a[i-1] - 2 * m);
real_time[i] = a[i] - offset;

// 2. DP Transfer
// dp[i] = min cost to depart at time i
for( int i = 0; i <= max_time; ++ i ) 
{
    // Pruning: if no one waits in (i-m, i], just inherit i-m
    if( i >= m && cnt[i] == cnt[i-m] ) 
    {
        dp[i] = dp[i-m]; continue;
    }
    // ... transfer ...
}
```
