## 1. 生态位

* **定义**: 将 DP 转移方程转化为**滑动窗口最值**模型，利用单调队列 (Deque) 将转移复杂度从 $O(K)$ 降至 $O(1)$。
* **解决痛点**:
    * **多重背包**: 物品数量 $C$ 很大，二进制拆分 ($O(V \log C)$) 依然超时，需优化至 $O(V)$。
    * **定长区间最值**: $dp[i] = \max_{j \in [i-L, i-R]} \{ dp[j] \} + val(i)$。

## 2. 逻辑支点

* **核心公式**: $dp[i] = \max_{j \in [L, R]} \{ dp[j] + \text{val}(j) \} + \text{const}(i)$
    * 关键在于将依赖 $j$ 的项和依赖 $i$ 的项彻底分离。
    * 依赖 $j$ 的部分 `val(j)` 放入队列维护单调性。
* **同余分组 (Congruence Splitting)**:
    * **背包优化**: 容量 $j$ 只能由 $j-w, j-2w \dots$ 转移而来。
    * **策略**: 将 $dp$ 数组按 $\% w$ 分为 $w$ 条独立的链，每条链单独跑单调队列。

## 3. 实战部署

### 3.1 多重背包单调队列优化 (P1776)

{% fold info @Code: Monotonic Knapsack %}
```cpp
// 枚举余数 r (0 ~ w-1)
for( int r = 0; r < w; ++ r ) 
{
    deque< pair<int, int> > q; // {val, index}
    // 遍历链: j = r, r+w, r+2w...
    for( int k = 0; r + k * w <= V; ++ k ) 
    {
        int j = r + k * w;
        // calc(j): dp[j] - k * v (去除 k 个 v 的相对值)
        int val = old_dp[j] - k * v;
        
        while( ! q.empty() && q.back().first <= val ) q.pop_back();
        q.push_back( {val, k} );
        
        // Window size limit: count (物品数量)
        if( q.front().second < k - count ) q.pop_front();
        
        // Restore real value: relative_max + k * v
        dp[j] = max( dp[j], q.front().first + k * v );
    }
}
```
