## 1. 生态位

* **定义**: 在贪心或 DP 转移中，利用**新产生元素天然有序**的特性，使用**多路单调队列 (Multi-Queue)** 替代 **优先队列 (Priority Queue)**。
* **解决痛点**:
    * **复杂度降维**: 将 $O(M \log N)$ (堆) 优化至 $O(M)$ (队列)。

## 2. 逻辑支点

* **核心直觉**: **"The Output is Ordered" (产出即有序)**。
    * 如果先被处理的元素 $A$，其产出的子元素 $A'$ 一定大于后被处理元素 $B$ 的产出 $B'$，则无需用堆维护。
* **蚯蚓模型**:
    * 取最大的一个 $\to$ 产生两个更小的。若 $L_1 > L_2$，则 $\lfloor pL_1 \rfloor \ge \lfloor pL_2 \rfloor$，满足单调性。

## 3. 实战部署

### 3.1 P2827 蚯蚓 (3路队列)

* **全局增长处理**: 维护一个全局偏移量 `delta`。
    * 取出时: `val + delta`
    * 存入时: `real_val - delta - q` (因为这只新蚯蚓没参与这一秒的增长，相当于提前减去)

{% fold info @Code: Earthworm %}

```cpp
// 3 queues: q[0] original, q[1] left_part, q[2] right_part
int get_max() 
{
    int res = -INF, idx = -1;
    for( int i = 0; i < 3; ++ i ) 
    {
        if( ! q[i].empty() && q[i].front() > res ) 
        {
            res = q[i].front();
            idx = i;
        }
    }
    if( idx != -1 ) q[idx].pop_front();
    return res;
}
```
