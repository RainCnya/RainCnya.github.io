## 1. 生态位

* **定义**: 在**有限容量** (Capacity) 限制下，通过**组合选择**物品以达成最大价值 (Value) 的模型。
* **本质**: **维度限制下的组合优化**。
* **解决痛点**: 资源分配、子集和问题、策略博弈。

## 2. 逻辑支点

* **核心直觉**: **循环方向决定复用性**。
    * **倒序 ($V \to w$)**: 隔离当前层影响 $\implies$ 物品仅被选 **1次** (0/1 背包/分组背包)。
    * **正序 ($w \to V$)**: 利用当前层影响 $\implies$ 物品可选 **$\infty$次** (完全背包)。
* **模型坍缩 (Collapse)**:
    * 所有背包问题最终都可归纳为 **分组背包** (互斥组) 或 **0/1 背包** (独立项)。
    * **多重背包**: $K$ 个物品 $\to$ 二进制拆分为 $\log K$ 个 0/1 物品。
    * **依赖背包**: 主件+附件 $\to$ 互斥的组合策略 $\to$ 分组背包。

## 3. 实战部署

### 3.1 分组背包 (P1757)

{% fold info @Code: Group Knapsack %}
```cpp
// 组 k -> 容量 j (倒序) -> 组内物品 i
// 保证每组最多选一个: j 的循环必须在 i 之前
for( int k = 1; k <= max_group; ++ k )
{
    for( int j = V; j >= 0; -- j )
    {
        for( int idx : group[k] ) // 遍历该组所有物品
        {
            if( j >= w[idx] )
            {
                dp[j] = max( dp[j], dp[j - w[idx]] + v[idx] );
            }
        }
    }
}
```

{% endfold %}

### 3.2 多重背包 (二进制拆分)

{% fold info @Code: Binary Splitting %}

```C++
// 将 "count 个 v, w" 拆分为 0/1 物品列表
void add_item( int v, int w, int count )
{
    for( int k = 1; k <= count; k <<= 1 )
    {
        W.push_back( k * w );
        V.push_back( k * v );
        count -= k;
    }
    if( count > 0 ) // 处理剩余部分
    {
        W.push_back( count * w );
        V.push_back( count * v );
    }
}
// 之后对 W, V 数组跑标准 0/1 背包
```

{% endfold %}
