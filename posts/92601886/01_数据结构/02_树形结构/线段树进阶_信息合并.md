## 1. 生态位

* **定义**: 探讨线段树在处理 **"非简单可加性信息"** 时的合并策略。从标量 (Scalar) 的最大子段和，进化到集合 (Set) 的动态连通性。
* **核心矛盾**: 
    * 简单线段树: `sum = L.sum + R.sum` (独立性)。
    * 复杂线段树: 左右子区间不仅有独立贡献，还存在 **"跨界 (Cross-Boundary)"** 的交互。
* **演进路线**:
    * **Level 1 (GSS)**: 维护 `lmax`, `rmax` 解决数值跨界问题。
    * **Level 2 (Connectivity)**: 维护 `L-Port`, `R-Port` 与 DSU 解决拓扑连接问题。

## 2. Level 1: 数值跨界 (GSS 模型)

### 2.1 逻辑支点
在 **最大子段和 (Maximum Subarray Sum)** 问题中，区间最优解只有三种可能：
1.  完全在左儿子 (`L.dat`)。
2.  完全在右儿子 (`R.dat`)。
3.  **跨越中点**: 左儿子的最大后缀 + 右儿子的最大前缀 (`L.rmax + R.lmax`)。

这迫使我们在节点中额外维护 `lmax` 和 `rmax`。

### 2.2 实战部署 (GSS3)

{% fold info @Code: GSS Push Up %}
```cpp
struct Node { int sum, lmax, rmax, dat; };

Node merge( Node L, Node R ) 
{
    Node res;
    res.sum = L.sum + R.sum;
    // 前缀可能贯穿左边，延伸到右边
    res.lmax = max( L.lmax, L.sum + R.lmax ); 
    res.rmax = max( R.rmax, R.sum + L.rmax );
    // 最大子段和的三种情况
    res.dat = max( max( L.dat, R.dat ), L.rmax + R.lmax );
    return res;
}
```

{% endfold %}

## 3. Level 2: 拓扑跨界 (动态连通性)

### 3.1 场景描述 (ZJOI2006 / TZOJ 3277)

- **问题**: 网格图，支持修改边/点权，查询两点连通性或连通块个数。
    
- **难点**: 左右区间合并时，中间的分界线 (`mid` 与 `mid+1`) 上可能有多个连通的接口。
    

### 3.2 逻辑支点

- **端口映射**: 维护区间左右边界的连通性编号。
    
- **并查集缝合**: 在 `push_up` 时，将左区间的右边界与右区间的左边界进行 `unite` 操作。
    

{% fold info @Code: Connectivity Merge %}

```C++
// 伪代码逻辑：缝合中轴线
void push_up( int u ) 
{
    int ls = u << 1, rs = u << 1 | 1;
    int mid = ( tr[ls].l + tr[u].r ) >> 1;

    // 1. 继承基础计数
    tr[u].cnt = tr[ls].cnt + tr[rs].cnt; 

    // 2. 缝合交界面 (Stitching Interface)
    for( int i = 1; i <= n; ++ i ) 
    {
        // 检查中轴线 (mid, mid+1) 的颜色/连通性是否一致
        if( G[i][mid] == G[i][mid+1] ) 
        {
            // 获取交界面两侧的连通分量 ID
            int root_L = find( tr[ls].right_port[i] ); 
            int root_R = find( tr[rs].left_port[i] );

            if( root_L != root_R ) 
            {
                fa[root_L] = root_R; // 合并
                tr[u].cnt --; // 连通块减少
            }
        }
    }
    // 3. 更新父节点端口信息 (Mapping)...
}
```
