## 1. 生态位

* **定义**: 内部元素保持严格单调（递增或递减）的线性数据结构。本质是**"存储有潜力的未来"**。
* **前置依赖**: [队列/栈基础], [滑动窗口]
* **解决痛点**:
    * **单调栈**: $O(N)$ 寻找每个元素左/右侧**第一个**比它大/小的元素 (NGE 问题)。
    * **单调队列**: $O(N)$ 解决**滑动窗口**内的最值查询 (RMQ)。
* **竞品对比**:

| 算法 | 核心优势 | 劣势/局限 | 适用场景 |
| :--- | :--- | :--- | :--- |
| **单调队列** | $O(N)$ 线性极速 | 仅限固定/变长窗口 | 滑动窗口、DP 优化 |
| **ST 表** | $O(1)$ 查询任意区间 | 不支持修改、建表慢 | 静态 RMQ |
| **线段树** | 支持修改 | 常数大、写起来烦 | 动态 RMQ |

## 2. 逻辑支点

* **淘汰哲学**: **"既不如此时新，也不如此时强，那便毫无意义。"**
    * 若新元素 $X$ 比老元素 $Y$ 更优且更晚过期，则 $Y$ 永久删除。
* **存储内容**: 务必存储 **下标 (Index)**。
    * 下标包含“位置/过期时间”信息。

## 3. 实战部署

### 3.1 单调栈模板 (Next Greater Element)

* **策略**: 找右边第一个比我大的 $\to$ 维护**单调递减**栈。

{% fold info @Code: Monotonic Stack %}
```cpp
// ans[i] 存储 i 右边第一个比 a[i] 大的元素下标
stack<int> stk; 

for( int i = 1; i <= n; ++ i ) 
{
    // 破坏单调性 -> 结算时刻
    while( !stk.empty() && a[i] > a[stk.top()] ) 
    {
        ans[stk.top()] = i; 
        stk.pop(); 
    }
    stk.push( i ); 
}
```

{% endfold %}

### 3.2 单调队列模板 (Sliding Window Max)

- **策略**: 求最大值 $\to$ 维护**单调递减**队列。
    

{% fold info @Code: Monotonic Queue %}

```cpp
deque<int> q; // 存下标

for( int i = 1; i <= n; ++ i ) 
{
    // 1. 检查队头过期
    if( !q.empty() && q.front() <= i - K ) 
        q.pop_front();
        
    // 2. 淘汰弱者
    while( !q.empty() && a[i] >= a[q.back()] ) 
        q.pop_back();
        
    // 3. 入队 & 取值
    q.push_back( i );
    if( i >= K ) printf("%d ", a[q.front()]);
}
```

{% endfold %}

### ⚠️ 边界与 Hack

- **[坑点 1]**: 单调栈处理**直方图最大矩形**时，需在数组末尾加 `0` 强制弹栈结算。
    
- **[坑点 2]**: 单调队列优化 DP 时，初始状态（如 `dp[0]`）需提前入队。
    

## 4. 知识粘附

- **变体模型**:
    
    - **二维滑动窗口**: (P2216) 先对每一行做单调队列，得到 `row_max[i][j]`；再对该结果的每一列做单调队列。
        
    - **DP 优化**: (P3800) 形如 $dp[i][j] = \max_{k \in [j-T, j+T]} (dp[i-1][k]) + val[i][j]$，内层 max 可用单调队列 $O(1)$ 转移。
        
    - **计数单调栈**: (P1950) 统计全 `.` 子矩形数量。维护每个点作为“悬线”底部的左右边界 `L[i], R[i]`，贡献为 $h[i] \times (i-L[i]) \times (R[i]-i)$。
