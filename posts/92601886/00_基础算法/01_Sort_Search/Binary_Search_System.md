## 1. 生态位

* **定义**: 在**单调**定义域上，通过不断折半缩小搜索区间，以 $O(\log N)$ 复杂度锁定目标值的算法。
* **前置依赖**: [排序], [单调性证明]
* **解决痛点**: 
    * **二分查找**: 在有序数组中快速定位元素（替代 $O(N)$ 遍历）。
    * **二分答案**: 将“求解问题”转化为“判定问题”。
        * *原问题*: 求最大的 $X$。
        * *转化后*: 给定 $X$，判定 `check(X)` 是否合法？
* **竞品对比**:

| 算法 | 核心优势 | 劣势/局限 | 适用场景 |
| :--- | :--- | :--- | :--- |
| **二分查找** | $O(\log N)$ 极速定位 | 必须依赖单调性/有序 | 静态查找、求最值 |
| **倍增法** | 适合处理“跳跃”和“区间” | 状态需满足结合律 | LCA、RMQ |


## 2. 逻辑支点

* **核心直觉**: **非黑即白的分界线**。
    * 问题的解空间必须具备单调性（例如：$X$ 合法 $\implies X-1$ 一定合法）。二分的本质是寻找合法与不合法的**边界 (Boundary)**。
* **通用模型**:
    * **定义域**: $[L, R]$。
    * **中点**: $mid = L + (R - L) / 2$。
    * **决策**: 根据 `check(mid)` 的结果，舍弃一半区间。
* **两类模板**:
    * **寻找第一个 $\ge K$ 的数**: `L` 收敛于目标。
    * **寻找最后一个 $\le K$ 的数** (或二分答案最大值): `mid` 计算需向上取整或使用 `ans` 记录法。

## 3. 实战部署

### 3.1 整数二分模板

* **策略**: 推荐使用 `L <= R` + `ans` 记录法。这种写法逻辑最清晰，不易死循环。

{% fold info @Code: Integer BS %}
```cpp
int solve( int L, int R ) 
{
    int ans = -1;
    while( L <= R ) 
    {
        int mid = L + ( (R - L) >> 1 );
        if( check( mid ) ) 
        {
            ans = mid;
            L = mid + 1; 
        }
        else 
        {
            R = mid - 1; 
        }
    }
    return ans;
}
```

{% endfold %}

### 3.2 实数二分模板

- **策略**: 建议固定循环次数 (如 100 次)，避免精度死循环，同时精度优于 `eps` 法。

{% fold info @Code: Real BS %}

```cpp
double solve( double L, double R ) 
{
    // 循环 100 次，精度可达 1e-30 级别，足够所有竞赛题目
    for( int i = 0; i < 100; ++ i ) 
    {
        double mid = ( L + R ) / 2.0;
        if( check( mid ) ) L = mid; // 或 R = mid
        else R = mid;
    }
    return L;
}
```

{% endfold %}

### 3.3 Check 函数的构造艺术

二分答案的难点永远不在二分本身，而在于 **`check()`** 函数的设计。以下是几种经典结合：

- **BS + 差分/前缀和**:
    
    - **P1083 借教室**: 判定前 $k$ 个订单是否合法 $\to$ 差分数组区间修改，判定是否有负值。
        
    - **P1314 聪明的质监员**: 判定参数 $W$ $\to$ 前缀和快速计算区间校验值。
        
- **BS + 二维前缀和/贪心**:
    
    - **P3017 Brownie Slicing**: 判定最小面积 $X$ $\to$ 贪心切割，利用二维前缀和 $O(1)$ 算面积。
        
- **BS + 单调队列/01分数规划**:
    
    - **P1419 寻找段落**: 判定平均值 $\ge X$ $\to$ 转化为 $\sum (a_i - X) \ge 0$ $\to$ 单调队列求最大子段和。

### ⚠️ 边界与 Hack

- **[坑点 1] 死循环**: 若写成 `L < R` 且 `mid = (L+R)/2`，当 `L=R-1` 时易死循环。
    
- **[坑点 2] 单调性**: 并非所有问题都具备全局单调性，单峰函数需用三分法。
