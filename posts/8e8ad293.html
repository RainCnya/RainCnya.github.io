算法学习笔记：棋盘动态规划

算法学习笔记：棋盘动态规划

一、 初步认知

  • 命名： 棋盘动态规划 (Grid Dynamic Programming)

  • 领域： 动态规划 (DP) / 二维 DP 模型

  • 定义： 状态定义依赖于二维坐标 (i,j)(i, j)(i,j)，通常表示到达 (i,j)(i, j)(i,j) 的最优解或方案数。其转移依赖于相邻格子 (i−1,j)(i-1, j)(i−1,j) 或 (i,j−1)(i, j-1)(i,j−1) 的状态。

  • 典型设问： “从 (1,1)(1, 1)(1,1) 走到 (N,M)(N, M)(N,M)，只能向右或向下，求最大价值或路径总数。”（例如：数字三角形、摘花生）。

二、 主题渗透

1. 基础模型：路径与最大/最小价值

  • 状态定义： dp[i][j]dp[i][j]dp[i][j] 表示从起点到格子 (i,j)(i, j)(i,j) 时的最大价值（或最小成本、或路径总数）。

  • 核心转移： 只能从上方 (i−1,j)(i-1, j)(i−1,j) 或左方 (i,j−1)(i, j-1)(i,j−1) 转移而来。

    • 求最大价值：

      dp[i][j]=max⁡(dp[i−1][j], dp[i][j−1])+Value(i,j)dp[i][j] = \max(dp[i-1][j], \ dp[i][j-1]) + \text{Value}(i, j) dp[i][j]=max(dp[i−1][j], dp[i][j−1])+Value(i,j)

    • 求路径总数：

      dp[i][j]=dp[i−1][j]+dp[i][j−1]dp[i][j] = dp[i-1][j] + dp[i][j-1] dp[i][j]=dp[i−1][j]+dp[i][j−1]

2. 进阶模型：多重访问与多维度

  • 多重访问：

    • 如果题目允许路径多次经过同一格子（如“数字三角形”变种），我们需要将状态扩展到访问次数，或者改变转移顺序。

    • 经典应用： “方格取数”问题（求两条路径最大价值），需要定义四维状态 dp[i1][j1][i2][j2]dp[i_1][j_1][i_2][j_2]dp[i1​][j1​][i2​][j2​]，并利用两条路径同时向右/向下的同步 DP 来简化状态。

      • 同步 DP 状态简化： 由于两条路径步数相同，我们可以用总步数 k=i+jk = i+jk=i+j 来降维。

      • dp[k][i1][i2]dp[k][i_1][i_2]dp[k][i1​][i2​]：第 kkk 步时，第一条路径在 (i1,k−i1)(i_1, k-i_1)(i1​,k−i1​)，第二条路径在 (i2,k−i2)(i_2, k-i_2)(i2​,k−i2​) 的最大价值。

  • 状态压缩 DP (轮廓线/插头 DP)：

    • 当棋盘较小（如 N≤10N \le 10N≤10）但需要覆盖整个棋盘时，或者涉及到复杂的连通性限制（如“汉密尔顿回路”）时。

    • 核心思想： dp[i][j][state]dp[i][j][\text{state}]dp[i][j][state]，state\text{state}state 是一个二进制数，压缩了当前格子 (i,j)(i, j)(i,j) 及其上方/左方的格子的连接或访问状态。

3. 核心思想

  • 空间换时间： 将路径搜索转化为递推计算。

  • 无后效性： 路径只能向右/向下，确保计算 dp[i][j]dp[i][j]dp[i][j] 时，其子问题 dp[i−1][j]dp[i-1][j]dp[i−1][j] 和 dp[i][j−1]dp[i][j-1]dp[i][j−1] 已经是最优解。

2. 部署细节

  • 边界处理： 必须单独处理第 1 行和第 1 列的初始化，因为它们只有一个来源方向。

  • 障碍物： 如果格子 (i,j)(i, j)(i,j) 是障碍物，则设置 dp[i][j]=0dp[i][j] = 0dp[i][j]=0（计数问题）或 dp[i][j]=−∞dp[i][j] = -\inftydp[i][j]=−∞（最值问题，确保它不作为有效转移的来源）。

  • 滚动数组： 如果 NNN 和 MMM 非常大，可以采用滚动数组将 O(NM)O(NM)O(NM) 的空间优化为 O(min⁡(N,M))O(\min(N, M))O(min(N,M))，即 dp[i][j]dp[i][j]dp[i][j] 只依赖于 dp[i−1][… ]dp[i-1][\dots]dp[i−1][…] 和 dp[i][j−1]dp[i][j-1]dp[i][j−1]。
