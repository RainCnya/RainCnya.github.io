03 动态规划优化 核心指南

核心哲学：动态规划本质上是在一堆决策里找最优路径。
所谓“优化”，就是利用数学直觉发现：
有些决策注定是垃圾，我们可以提前扔掉；
有些计算是重复的，我们可以只算一次。

1. 知识图谱与演化

  • 核心抓手: 状态精简、决策单调性、贡献度复用、维度拆解。

  • 演化路径:

    1. 基础模型：线性、背包、区间、树形。解决“怎么定义状态”和“怎么转”的问题。

    2. 状态压缩：当状态很碎但范围很小时，把一堆开关状态塞进一个二进制数里。

    3. 决策优化：利用单调队列、斜率、四边形不等式，让原本需要扫一遍的决策过程，变成一眼看穿。

    4. 结构结合：让 DP 跑在自动机（SAM/AC）或数据结构上，处理更复杂的限制条件。

2. 决策矩阵

  • 状态看起来像“开关”且范围 N≤20N \le 20N≤20: 优先考虑状压 DP。

  • 转移方程形如 dp[i]=min⁡(dp[j]+cost(j,i))dp[i] = \min(dp[j] + \text{cost}(j, i))dp[i]=min(dp[j]+cost(j,i))，且 cost\text{cost}cost 满足决策单调性:

    • 如果 cost\text{cost}cost 是简单的窗口最值，用单调队列。

    • 如果 cost\text{cost}cost 带坐标乘积项（如 a[i]×b[j]a[i] \times b[j]a[i]×b[j]），考虑斜率优化。

  • 题目涉及字符串匹配且需要统计方案数: 结合自动机（如 AC 自动机、SAM）进行状态转移。

  • 树形 DP 统计子树信息且复杂度过高: 考虑轻重链剖分（长链剖分）或 DSU on Tree 优化。

3. 核心板块索引

3.1 经典模型

  • [L3] 01 经典模型: DP 的底座。解决线性、背包、区间和树上的逻辑闭环。

  • [Note] 树形 DP: 在递归回溯中维护子树的最优状态。

  • [Note] 背包体系: 解决“选或不选”的资源分配问题。

3.2 状态压缩

  • [L3] 02 状态压缩: 解决“小规模但多维度”的状态表示。

  • [Note] 状压 DP: 利用二进制位运算，将 NNN 个开关状态压缩为一个整数。

  • [Note] 插头 DP: 处理网格图连通性的极致压缩手段。

3.3 决策优化

  • [L3] 03 决策优化: 杀掉转移复杂度 O(N)O(N)O(N) 的利器。

  • [Note] 单调队列优化: 丢弃那些“又老又差”的决策，保持决策窗口的单调性。

  • [Note] 斜率优化: 将转移看成寻找凸壳上的点，通过维护凸壳来快速定位最优决策点。

  • [Note] 四边形不等式: 利用决策点的单调移动，将 O(N3)O(N^3)O(N3) 压制到 O(N2)O(N^2)O(N2)。

3.4 自动机 DP

  • [L3] 04 自动机 DP: 字符串与 DP 的跨界组合。

  • [Note] 数位 DP: 解决“统计区间 [L,R][L, R][L,R] 内满足条件的数字个数”，核心是按位填写。

  • [Note] AC 自动机 DP: 在多模式匹配图上进行路径计数或最值求解。

4. 依赖与规划

  • 前置依赖: 必须熟练掌握递归、记忆化搜索、以及基础的线性与区间 DP 推导。

  • 学习路径:

    1. P1 (基础)：能独立推导出基础 DP 方程，不再因为“状态定义不清楚”而卡壳。

    2. P2 (中坚)：攻克状压 DP 和单调队列优化。这是 CF 1900 左右最常出现的逻辑卡点。

    3. P3 (顶尖)：掌握斜率优化、插头 DP 和复杂的自动机结合模型。
