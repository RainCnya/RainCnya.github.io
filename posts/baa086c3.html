背包DP

算法学习路线图：背包动态规划 (Knapsack DP Roadmap)

一、 初步认知 (Macro View)

1. 叫什么 (Identity)

  • 中文名： 背包动态规划
  • 英文名： Knapsack Problem (简称 KP 或 Knapsack DP)

2. 什么地位 (Status)

  • 所属领域： 动态规划
  • 核心地位： 它是 DP 状态定义、循环优化和逻辑建模的经典入门模型。

3. 有什么用处 (Utility)

  • 解决的核心问题： “选择与权衡”的最优解问题。
  • 典型应用场景：
    • 给定一组物品（各自有重量/成本和价值）和一个容量（预算）。
    • 如何选择物品，才能在不超过容量的前提下，使总价值最大。

二、 主题渗透 (Theme Osmosis)

1. 核心原理 (The Eureka Moment)

  • 底层思想： 动态规划 (DP) + 决策 (选或不选)。

  • 关键推导：

    • 背包问题的核心是对第 iii 个物品做决策。

    • 状态定义： dp[j]dp[j]dp[j] 通常表示在容量（或成本）恰好为 jjj 时的最大价值。

    • 状态转移： 当我们考虑第 iii 个物品（重量 w[i]w[i]w[i], 价值 v[i]v[i]v[i]）时，对于容量 jjj：

    • 决策 1 (不选 iii)： 价值不变，dp[j]=dp[j]dp[j] = dp[j]dp[j]=dp[j] (继承上一轮 i−1i-1i−1 的结果)。

    • 决策 2 (选 iii)： 必须能放下（j>=w[i]j >= w[i]j>=w[i]），价值从“未放 iii 时的状态”转移而来。

      • dp[j]=dp[j−w[i]]+v[i]dp[j] = dp[j - w[i]] + v[i]dp[j]=dp[j−w[i]]+v[i]

    • 转移方程： dp[j]=max(dp[j],dp[j−w[i]]+v[i])dp[j] = max(dp[j], dp[j - w[i]] + v[i])dp[j]=max(dp[j],dp[j−w[i]]+v[i])

2. 模拟演示 (Visualization)

背包 DP 学习路线图

阶段一：0/1 背包 (The Foundation)

  • 问题： NNN 个物品，每个物品只有 1 件。
  • 核心： for i = 1 to N (物品)   ⟹  \implies⟹ for j = V down to w[i] (容量)
  • 关键点： 容量 jjj 必须倒序循环。
  • 原因： 保证 dp[j - w[i]] 是上一轮 i−1i-1i−1 的状态。如果正序，dp[j - w[i]] 会被本轮 iii 的状态污染，导致一个物品被重复选取。

阶段二：完全背包 (Unbounded Knapsack)

  • 问题： NNN 个物品，每个物品有无限件。
  • 核心： for i = 1 to N (物品)   ⟹  \implies⟹ for j = w[i] to V (容量)
  • 关键点： 容量 jjj 必须正序循环。
  • 原因： 我们故意要让 dp[j - w[i]] 是本轮 iii 的状态，这等价于 dp[j] = max(dp[j], dp[j - w[i]*k] + v[i]*k)，即允许第 iii 个物品被重复选取。

阶段三：多重背包 (Bounded Knapsack)

  • 问题： NNN 个物品，第 iii 个物品有 c[i]c[i]c[i] 件。

  • 核心： 它是 0/1 背包和完全背包的结合体。

  • 解法 1 (朴素)： 将 c[i]c[i]c[i] 件物品拆分为 c[i]c[i]c[i] 个 0/1 物品。复杂度 O(V∑ci)O(V \sum c_i)O(V∑ci​)，会超时。

  • 解法 2 (二进制拆分)：

    • 思想：将 c[i]c[i]c[i] 件拆分为 log⁡ci\log c_ilogci​ 组（如 1, 2, 4, …）。

    • 核心：将多重背包转化为 0/1 背包。复杂度 O(NVlog⁡C)O(NV \log C)O(NVlogC)。

  • 解法 3 (单调队列优化)：

    • 思想：将转移方程 dp[j]=max(dp[j−k∗w]+k×v)dp[j] = max(dp[j - k*w] + k \times v)dp[j]=max(dp[j−k∗w]+k×v) 视为一个滑动窗口最值问题。
    • 核心：使用单调队列优化。复杂度 O(NV)O(NV)O(NV)。

阶段四：分组背包 (Grouped Knapsack)

  • 问题： NNN 个物品，分为 KKK 组，每组至多选 1 件。
  • 核心： 三重循环。
    • for k = 1 to K (组)
    • for j = V down to 0 (容量, 必须倒序)
    • for i in group[k] (组内物品)
    • dp[j] = max(dp[j], dp[j - w[i]] + v[i])

3. 详细实现过程 (Step-by-Step)

这是解决任何背包问题的通用思考流程：

  1. 识别模型 (Identify)：

    • 是 0/1（选 1 次）、完全（选 ∞\infty∞ 次）、多重（选 CCC 次）还是分组（KKK 组选 1）？

  2. 定义状态 (Define State)：

    • dp[j]dp[j]dp[j] 通常表示容量为 jjj 时的最大价值。

  3. 确定循环 (Iteration Order)：

    • 外层： 物品 iii (或组 kkk)。
    • 内层： 容量 jjj。

  4. 确定 jjj 的方向 (Key Pitfall)：

    • 倒序 ( jjj 从 V→w[i]V \to w[i]V→w[i] )： 0/1 背包、多重背包（二进制拆分）、分组背包。
    • 正序 ( jjj 从 w[i]→Vw[i] \to Vw[i]→V )： 完全背包。

  5. 初始化 (Initialize)：

    • 求最大值：dp[0]=0dp[0] = 0dp[0]=0，其余 dp[1..V]=−INFdp[1..V] = -INFdp[1..V]=−INF (确保从合法状态转移)。
    • 求方案数：dp[0]=1dp[0] = 1dp[0]=1，其余为 0。

三、 总结分析 (Analysis & Comparison)

1. 复杂度分析 (Complexity)

维度 复杂度 (N=物品, V=容量, C=数量) 说明
0/1 背包 O(NV)O(NV)O(NV) 基础
完全背包 O(NV)O(NV)O(NV) 基础
多重背包 (二进制) O(NVlog⁡C)O(NV \log C)O(NVlogC) 常用，易于实现
多重背包 (单调队列) O(NV)O(NV)O(NV) 最优，较难实现
分组背包 O(N1V+N2V+...)=O(NV)O(N_1 V + N_2 V + ...) = O(NV)O(N1​V+N2​V+...)=O(NV) NNN 为所有物品总数

2. 核心注意点 (Key Pitfalls)

  1. jjj 的循环方向 (灵魂)：

    • 倒序 (0/1)： 保证 dp[j−w]dp[j-w]dp[j−w] 是上一行 ( i−1i-1i−1 ) 的状态。
    • 正序 (完全)： 保证 dp[j−w]dp[j-w]dp[j−w] 是当前行 ( iii ) 的状态。
    • 这是区分 0/1 背包和完全背包的唯一代码区别。

  2. 循环嵌套顺序 (关键)：

    • 99% 的背包问题都是物品在外层 (for i...)，容量在内层 (for j...)。
    • （特例：完全背包可以交换顺序，0/1 背包绝对不行）。

  3. 初始化 (边界)：

    • 如果题目要求“恰好装满”，dp[0]=0dp[0]=0dp[0]=0，dp[1…V]=−INFdp[1 \dots V] = -INFdp[1…V]=−INF。
    • 如果题目要求“不超过容量”，dp[0..V]=0dp[0..V] = 0dp[0..V]=0。

  4. 多重背包转化： 二进制拆分（1, 2, 4, ...）是将多重背包问题转化为 0/1 背包问题。
