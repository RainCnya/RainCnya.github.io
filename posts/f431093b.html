(旧)算法学习笔记：冒泡排序（Sort-Bubble）

冒泡排序 (含“逆序对”分析)

第一版冒泡排序。

一、 什么是排序？

通俗地讲，排序就是把一串杂乱无章的数字，按照从小到大（或从大到小）的顺序重新排列。

例如，把数列 [5, 4, 2, 1, 3] 排序成升序（从小到大），结果就是 [1, 2, 3, 4, 5]。

(要学习排序，你需要 数组、for 循环和 if-else 分支语句的基础知识)

二、 冒泡排序 (Bubble Sort)

1. 核心思想

最简单的一种排序方法叫做冒泡排序。

我们的目标是让数列变成升序，即第一个数字最小，最后一个数字最大。

冒泡的核心思想很简单：在一排数字中，从头到尾依次比较相邻的两个数，如果顺序不对就交换它们。

2. 模拟过程

让我们用 [5, 4, 3, 2, 1] 这个例子来看具体操作。

  • 第一轮： (目标是把最大的数“冒泡”到最后)

    • 对比第 1 和第 2 个数：5 > 4，交换！ 8-> [4, 5, 3, 2, 1]
    • 对比第 2 和第 3 个数：5 > 3，交换！ 10 -> [4, 3, 5, 2, 1]
    • 对比第 3 和第 4 个数：5 > 2，交换！ 11 -> [4, 3, 2, 5, 1]
    • 对比第 4 和第 5 个数：5 > 1，交换！ 12-> [4, 3, 2, 1, 5]
    • 第一轮后，最大的 5 已经到了正确位置 14。

  • 第二轮： (目标是把第二大的数冒泡到倒数第二)

    • 我们对 [4, 3, 2, 1] 部分重复操作。
    • … 交换后 … -> [3, 2, 1, 4, 5]

  • 第三轮：

    • 我们对 [3, 2, 1] 部分重复操作。
    • … 交换后 … -> [2, 1, 3, 4, 5]。

  • 第四轮：

    • 我们对 [2, 1] 部分重复操作。
    • … 交换后 … -> [1, 2, 3, 4, 5]。

现在，整个数组完全有序了。对于一个有 n 个数字的数组，我们总共需要进行 n − 1 轮这样的冒泡。

三、 复杂度与代码实现

1. 复杂度分析

  • 时间复杂度： O(N2)

    • 我们需要 N − 1 轮冒泡。
    • 在第 i 轮中，我们需要比较 N − 1 − i 次。
    • 总比较次数为 $(N-1) + (N-2) + ... + 1 = \frac{N(N-1)}{2}$，因此时间复杂度为 O(N2)。

  • 空间复杂度： O(1)

    • 交换 tmp 变量是常数级别，我们是原地排序。

四、 深入分析：排序的本质 (逆序对)

1. 什么是“逆序对”？

这里我们需要引入一个概念——逆序对 (Inversion Pair)

  • 定义： 当 i < j 却出现了 A[i] > A[j] 的情况，我们就说 (A[i], A[j]) 构成了一个“逆序对” 。

  • “逆序对”的数量，是衡量一个数组“有多乱”的指标。

    • 完全有序的数组，逆序对为 0 。
    • 完全倒序的数组，逆序对数量最多。

  • 举例： 数组 A = [2, 3, 1]

    • i = 0, j = 1: (2 < 3)，不是逆序对。
    • i = 0, j = 2: (2 > 1)，是逆序对 (2, 1)。
    • i = 1, j = 2: (3 > 1)，是逆序对 (3, 1)。
    • 总逆序对数量为 2。

2. 冒泡排序的本质

所有排序算法的最终目标，就是通过一系列操作，将数组的 逆序对数量降为 0 。

回头看冒泡排序，它的操作非常纯粹：只交换 相邻且顺序错误 的元素。

当我们交换 A[j] 和 A[j + 1] (因为 A[j] > A[j + 1]) 时:

  • 我们精确地修正了 (A[j], A[j + 1]) 这一个逆序对。
  • 这次交换不会影响其他任何元素的相对顺序，因此也不会产生任何新的逆序对。

结论：冒泡排序中，每执行一次交换，数组的逆序对总数就精确地减少 1。“交换的本质，就是消除逆序对。”

因此，一个含有 k 个逆序对的数组，用冒泡排序使之完全有序，不多不少，正好需要 k 次交换。
