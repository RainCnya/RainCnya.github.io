算法学习笔记：ST 表 (Sparse Table)

算法学习笔记：ST 表 (Sparse Table)

一、 初步认知

  • 命名： ST 表 (Sparse Table)

  • 领域： 数据结构 / 静态区间查询 (RMQ)

  • 定义： 一种基于倍增 (Binary Lifting) 思想的静态数据结构。它通过 O(Nlog⁡N)O(N \log N)O(NlogN) 的预处理，实现 O(1)O(1)O(1) 的极速区间查询。

  • 典型设问： “给定一个长度为 NNN 的数组，进行 MMM 次询问，每次询问区间 [L,R][L, R][L,R] 内的最大值（或最小值、GCD）。”（注意：数组必须是静态的，不支持修改）。

  • 可扩展性：

    • 运算类型： 仅支持 幂等性 (Idempotent) 运算，即满足 x op x=xx \ op \ x = xx op x=x 的运算，如 max, min, gcd, lcm, bitwise OR/AND。不支持 sum (求和)。

    • 维度扩展： 可扩展为 2D ST 表，用于处理矩阵内的区域最值查询。

二、 主题渗透

1. 基本思路

  • 倍增预处理： 定义状态 st[i][j] 表示从位置 i 开始，长度为 2j2^j2j 的区间（即 [i,i+2j−1][i, i + 2^j - 1][i,i+2j−1]）内的最值。

  • 状态转移： 一个长度为 2j2^j2j 的区间可以拆分为两个长度为 2j−12^{j-1}2j−1 的子区间。

    • 左半部分：st[i][j-1]

    • 右半部分：st[i + (1 << (j-1))][j-1]

    • 方程：st[i][j] = max(st[i][j-1], st[i + (1 << (j-1))][j-1])

2. 核心技巧 (O(1) 查询)

  • 重叠覆盖： 对于任意查询区间 [L,R][L, R][L,R]，其长度为 lenlenlen。我们找到一个最大的 kkk 使得 2k≤len2^k \le len2k≤len。

  • 拼凑： 用两个长度为 2k2^k2k 的区间来覆盖 [L,R][L, R][L,R]：

    1. 以 LLL 开头的区间：[L,L+2k−1][L, L + 2^k - 1][L,L+2k−1]

    2. 以 RRR 结尾的区间：[R−2k+1,R][R - 2^k + 1, R][R−2k+1,R]

  • 原理： 这两个区间可能会重叠，但对于 max/gcd 等幂等运算，重叠不影响结果（重复计算同一个数没关系）。这与线段树不同（线段树通过不重叠区间的加法求和）。

模拟 (查询区间 $[3, 9]$ 的最大值)

  • 区间长度 len=9−3+1=7len = 9 - 3 + 1 = 7len=9−3+1=7。

  • 最大的 2k≤72^k \le 72k≤7 是 22=42^2 = 422=4 (k=2k=2k=2)。

  • 覆盖策略：

    • 左区间 (Start at 3): [3,3+4−1]=[3,6][3, 3+4-1] = [3, 6][3,3+4−1]=[3,6] →\rightarrow→ 查 st[3][2]

    • 右区间 (End at 9): [9−4+1,9]=[6,9][9-4+1, 9] = [6, 9][9−4+1,9]=[6,9] →\rightarrow→ 查 st[6][2]

  • 结果： max(st[3][2], st[6][2])。

  • 注意： 元素 6 被重复计算了，但这不影响最大值。

三、 核心思想

  • 倍增 (Doubling)：利用 222 的幂次快速扩张区间。

  • 幂等性覆盖 (Idempotent Coverage)：利用 max(x, x) = x 的特性，允许区间重叠以换取 O(1)O(1)O(1) 的查询速度。

2. 部署细节

  • Log 预处理： 虽然 C++ 提供了 log2() 或 __lg() 函数，但在频繁查询（如 10710^7107 次）下，预处理 lg[] 数组效率最高且最稳妥。

  • 循环顺序： 构建 ST 表时，必须先枚举指数 j，再枚举起点 i。因为长区间的状态依赖于短区间，必须先把所有短区间的层都算好。

  • 空间消耗： st 数组的大小是 N×log⁡NN \times \log NN×logN。对于 N=106N=10^6N=106，int st[1000005][21] 大约占用 80MB 内存，需注意题目空间限制。

  • 扩展应用：

    • GCD 查询： 将 max 换成 __gcd。

    • 最值位置： 如果题目要求返回最大值的下标而不是数值，可以将 st 数组存 index，比较时通过 a[index] 比较。

五、 维度概览

维度 复杂度 说明
时间 (建表) O(Nlog⁡N)O(N \log N)O(NlogN) 填满 N×log⁡NN \times \log NN×logN 的表格。
时间 (查询) O(1)O(1)O(1) 无论区间多长，只需要操作 2 次。
空间 O(Nlog⁡N)O(N \log N)O(NlogN) 需要较大的内存空间。
限制 静态 / 幂等 不可修改数值；仅适用于 Max/Min/GCD 等。
