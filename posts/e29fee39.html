一、 题目大意

给定一个 a × b 的矩阵。

要求我们找到一个 n × n 的子矩阵（正方形区域），

使得这个区域内所有元素的最大值与最小值的差值最小。输出这个最小的差值。

二、 思路分析与关键点

1. 暴力（Naive Solution）

最直接的思路是：

  1. 枚举所有 n × n 正方形的左上角 (i, j)。可能的 i 范围是 [1, a − n + 1]，j 的范围是 [1, b − n + 1]。
  2. 对于每一个左上角 (i, j)，再用两层循环 O(n2) 遍历这个 n × n 的区域，找到该区域内的最大值 maxv 和最小值 minv。
  3. 计算 maxv − minv，并更新全局答案 ans = min (ans, maxv − minv)。
  • 复杂度： 共有 O(a ⋅ b) 个子矩阵，每次查找耗时 O(n2)。总时间复杂度为 O(a ⋅ b ⋅ n2)。
  • 结论： 根据数据规模，1000 × 1000 × 1002 ≈ 1010，必定超时 (TLE)。

2. 核心思路：单调队列优化 (二维滑动窗口)

注意到，暴力的瓶颈在于 O(n2) 地重复查找 n × n 区域内的最值。

这本质上是一个 二维的滑动窗口 的 最值 (RMQ) 问题。

我们可以将二维问题降维成两个一维问题，使用单调队列进行两次预处理：

  1. 第一次 (行处理)： 预处理出所有 1 × n 窗口（即长度为 n 的行）的最值。
  2. 第二次 (列处理)： 利用第一次的结果，处理所有 n × 1 窗口（即高度为 n 的列）的最值。
  • 复杂度对比：
    • 暴力 O(a ⋅ b ⋅ n2) TLE
    • 单调队列 O(a ⋅ b) AC
  • 算法基础原理请参考：单调队列学习笔记

关键步骤：

  1. 预处理行（O(a ⋅ b)）：

    • 我们定义 rmin[i][j] 表示在第 i 行中，从第 j 列开始，连续 n 个数（即窗口 [j, j + n − 1]）中的最小值。rmax[i][j] 同理。
    • 我们遍历每一行 i ∈ [1, a]。
    • 在第 i 行上，使用两个单调队列进行一次长度为 n 的滑动窗口操作。
    • 这样我们可以在 O(b) 的时间内处理完第 i 行，并将结果存入 rmin[i][...] 和 rmax[i][...]。
  • 示例：对 a × b 矩阵的每一行 i 进行滑动窗口。

  1. 预处理列（O(a ⋅ b)）：

    • 经过步骤 1，rmin 矩阵的第 j 列，代表了所有以 j 为起点的 1 × n 窗口的最小值集合。
    • 我们定义 amin[i][j] 表示以 (i, j) 为左上角的 n × n 区域中的全局最小值。amax[i][j] 同理。
    • amin[i][j] 就等于 rmin[i][j], rmin[i+1][j], ..., rmin[i+n-1][j] 里的最小值。（amax 同理）
    • 我们遍历每一列 j ∈ [1, b − n + 1]（注意是处理过的列）。
    • 在第 j 列上，对 rmin[...][j] 和 rmax[...][j] 数组，再次使用单调队列进行一次长度为 n 的滑动窗口操作。
    • 将结果存入 amin[i][j] 和 amax[i][j]。

  2. 计算核心（O(a ⋅ b)）：

    • 经过两次预处理，amin[i][j] 和 amax[i][j] 已经分别存储了以 (i, j) 为左上角的 n × n 区域的最小值和最大值。
    • 我们只需遍历所有可能的左上角 (i, j)，其中 i ∈ [1, a − n + 1], j ∈ [1, b − n + 1]。
    • 计算 amax[i][j] - amin[i][j]，并更新全局答案 ans。

3. 关键点与总结

  • 时间复杂度：

    • 步骤 1 (行处理) O(a ⋅ b)，
    • 步骤 2 (列处理) O(a ⋅ (b − n + 1)) ≈ O(a ⋅ b)，
    • 步骤 3 (统计答案) O(a ⋅ b)。
    • 总时间复杂度 O(a ⋅ b)。

  • 空间复杂度： O(a ⋅ b)，用于存储 rmin, rmax, amin, amax 数组。

  • 核心逻辑： 本题的核心是将二维 RMQ 问题分解为两次一维滑动窗口。第一次按行滑动，第二次利用第一次的结果按列滑动。

因为不需要进行动态修改，此题使用ST表也能高效求解。
