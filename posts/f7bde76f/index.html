[Leaf] [ABC445F] Exactly K Steps 2

F - Exactly K Steps 2

1. 题面梗概

中译中：给一个 NNN 个点的完全有向图，每条边 (i,j)(i, j)(i,j) 都有代价 Ci,jC_{i,j}Ci,j​。给定 KKK，对于每个起点 sss，求出恰好经过 KKK 条边并回到 sss 的最小总代价。

  • 1≤N≤1001 \le N \le 1001≤N≤100
  • 1≤K≤1091 \le K \le 10^91≤K≤109
  • 0≤Ci,j≤109 (1≤i≤N,1≤j≤N)0 \le C_{i,j} \le 10^9\ (1 \le i \le N,1 \le j \le N)0≤Ci,j​≤109 (1≤i≤N,1≤j≤N)

2. 逻辑推导

考虑 DP，设 dp[k][i][j]dp[k][i][j]dp[k][i][j] 表示恰好经过 kkk 条边从 iii 到 jjj 的最小代价。

那么很容易能写出一个转移方程。

dp[k+1][i][j]=min⁡1≤m≤N{dp[k][i][m]+Cm,j}dp[k+1][i][j] = \min_{1 \le m \le N} \{ dp[k][i][m] + C_{m,j} \} dp[k+1][i][j]=1≤m≤Nmin​{dp[k][i][m]+Cm,j​}

为了走 k+1k + 1k+1 步到 jjj，我可以先走 kkk 步到某个中间点 mmm，然后再花 Cm,jC_{m,j}Cm,j​ 的代价走最后一步。

但是！k≤109k \leq 10^9k≤109，直接状态转移，绝对会TLE，我们需要优化。

2.1 优化

对于这个数量级的规模，最简单想到的一种优化就是 二分 或者 倍增 这种带 log⁡\loglog 的复杂度。

我们再拆解一下这个转移方程：

dp[a+b][i][j]=min⁡1≤p≤n(dp[a][i][p]+dp[b][p][j])dp[a + b][i][j] = \min_{1 \leq p \leq n}(dp[a][i][p] + dp[b][p][j]) dp[a+b][i][j]=1≤p≤nmin​(dp[a][i][p]+dp[b][p][j])

有思路了吗？没有？我们再对比一下下面这个柿子：

C[i][j]=∑(A[i][p]⋅B[p][j])C[i][j] = \sum( A[i][p] \cdot B[p][j]) C[i][j]=∑(A[i][p]⋅B[p][j])

是不是惊人的相似，这里解释一下，下面这个是 矩阵乘法，而前面那个状态转移其实是满足 结合律 的。

我们只需要把 求和 替换为 取最小值，乘法 换成 加法，这个问题就可以用矩阵快速幂优化。

插入一句，这玩意学名好像叫 (min,+)(min, +)(min,+) 卷积。

2.2 快速幂

既然这种运算符合 结合律 的，我们就可以定义一个初始代价矩阵 MMM，其中 Mi,j=Ci,jM_{i,j} = C_{i, j}Mi,j​=Ci,j​，经过 KKK 条边的最短路径矩阵就是 MKM^{K}MK。

单位矩阵：在这个 (min,+)(min, +)(min,+) 运算下，单位矩阵 III 满足 Ii,i=0I_{i,i} = 0Ii,i​=0，其余元素为 ∞\infty∞。

2.3 广义单位矩阵

这里插入一节拓展知识。

在标准的算术矩阵中，单位矩阵 III 的对角线是 111（乘法单位元），其余是 000（加法单位元）。

但在 (min⁡,+)(\min, +)(min,+) 代数系中：

  • 乘法 →\to→ 加法：其单位元是 000（因为 x+0=xx + 0 = xx+0=x）。

  • 加法 →\to→ 最小值：其单位元是 ∞\infty∞（因为 min⁡(x,∞)=x\min(x, \infty) = xmin(x,∞)=x）。

因此，(min⁡,+)(\min, +)(min,+) 意义下的单位矩阵 III 必须满足：

  • 对角线 Ii,i=0I_{i,i} = 0Ii,i​=0：代表走 000 步的代价为 000。

  • 其余 Ii,j=∞I_{i,j} = \inftyIi,j​=∞：代表走 000 步无法到达其他位置。

但是这里我偷懒了，我直接把矩阵初始化为第一步的情况，就避免了 走 0 步 的边界处理。

2.4 结论

通过矩阵快速幂，我们将复杂度从 O(K⋅N2)O(K \cdot N^2)O(K⋅N2) 降低到了 O(log⁡K⋅N3)O(\log K \cdot N^3)O(logK⋅N3)。
