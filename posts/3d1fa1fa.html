算法学习笔记：区间动态规划

算法学习笔记：区间动态规划

一、 算法图谱

  • 命名： 区间动态规划 (Interval DP)

  • 领域： 动态规划 (DP) / 经典模型

  • 定义： 状态由区间端点 iii 和 jjj 定义（dp[i][j]dp[i][j]dp[i][j]），表示区间 [i,j][i, j][i,j] 内的最优解。通过枚举分割点，将大区间拆分为两个小区间进行转移。

  • 典型设问： “有 NNN 堆石子排成一排，每次只能合并相邻的两堆，求将所有石子合并为一堆的最小/最大代价。”（石子合并问题）

二、 核心推演

1. 基本思路

  • 状态定义： dp[i][j]dp[i][j]dp[i][j] 表示将第 iii 堆到第 jjj 堆石子合并成一堆的最小代价。

  • 决策（分割）： 既然最后要变成一堆，那么它最后一步一定是某两堆（比如“左边一部分”和“右边一部分”）合并而来的。

  • 转移方程：

    dp[i][j]=min⁡i≤k<j{dp[i][k]+dp[k+1][j]}+cost(i,j)dp[i][j] = \min_{i \le k < j} \{ dp[i][k] + dp[k+1][j] \} + \text{cost}(i, j) dp[i][j]=i≤k<jmin​{dp[i][k]+dp[k+1][j]}+cost(i,j)

    • 其中 kkk 是分割点，cost(i,j)\text{cost}(i, j)cost(i,j) 是合并这两部分的代价（通常是区间元素之和，可用前缀和 O(1)O(1)O(1) 计算）。

2. 优化思路

  • 四边形不等式优化 (Knuth Optimization)：

    • 对于某些特定的代价函数（如石子合并），最佳分割点 kkk 的位置具有单调性。

    • 利用 s[i][j−1]≤s[i][j]≤s[i+1][j]s[i][j-1] \le s[i][j] \le s[i+1][j]s[i][j−1]≤s[i][j]≤s[i+1][j] 的性质，可以将枚举 kkk 的复杂度从 O(N)O(N)O(N) 降为 O(1)O(1)O(1) 级别。

    • 效果： 总时间复杂度从 O(N3)O(N^3)O(N3) 降为 O(N2)O(N^2)O(N2)。

3. 核心思想

  • “由小及大”：先计算短区间的解，再利用短区间合并出长区间的解。因此，循环的第一维必须是“区间长度”。

4. 演示

▶
模拟 (区间从小到大)

  • 输入： [3, 5, 2] (前缀和 s=[0, 3, 8, 10])

  • 初始化： dp[i][i] = 0 (合并自己不需要代价)

  • 过程 (按长度 len 枚举)：

    • len=2：

      • dp[1][2]=dp[1][1]+dp[2][2]+(3+5)=0+0+8=8dp[1][2] = dp[1][1] + dp[2][2] + (3+5) = 0+0+8 = \mathbf{8}dp[1][2]=dp[1][1]+dp[2][2]+(3+5)=0+0+8=8

      • dp[2][3]=dp[2][2]+dp[3][3]+(5+2)=0+0+7=7dp[2][3] = dp[2][2] + dp[3][3] + (5+2) = 0+0+7 = \mathbf{7}dp[2][3]=dp[2][2]+dp[3][3]+(5+2)=0+0+7=7

    • len=3：

      • 计算 dp[1][3]dp[1][3]dp[1][3]，区间 [1,3][1, 3][1,3] 和为 10。

      • 尝试 k=1k=1k=1 (切在1后): dp[1][1]+dp[2][3]+10=0+7+10=17dp[1][1] + dp[2][3] + 10 = 0 + 7 + 10 = \mathbf{17}dp[1][1]+dp[2][3]+10=0+7+10=17

      • 尝试 k=2k=2k=2 (切在2后): dp[1][2]+dp[3][3]+10=8+0+10=18dp[1][2] + dp[3][3] + 10 = 8 + 0 + 10 = 18dp[1][2]+dp[3][3]+10=8+0+10=18

      • 取 min: 17

  • 结果： 最小代价为 17。 (先合 5+2=7, 再合 3+7=17)

三、 实战部署

1. 代码模板

▶
区间 DP (O(N^3) 通用模板)

2. 部署细节

  • 循环顺序 (绝对关键)：

    • 错误写法： for(i=1..n) for(j=i..n)。这样算 dp[1][5]dp[1][5]dp[1][5] 时可能用到还没算的 dp[2][5]dp[2][5]dp[2][5]。

    • 正确写法： for(len=2..n) 在最外层。保证计算长区间时，其包含的所有短区间都已计算完毕。

  • 分割点 kkk： 范围是 [i, j-1]。注意不能取到 j，否则右半部分 [j+1, j] 为空。

  • 环形区间 DP： 如果题目是“项链”（首尾相接），通常做法是将数组复制一份接在后面 (a[1...n, 1...n])，长度变为 2N2N2N，最后答案是 min⁡(dp[i][i+n−1])\min(dp[i][i+n-1])min(dp[i][i+n−1])。

四、 维度概览

维度 复杂度 说明
时间 O(N3)O(N^3)O(N3) 三重循环。
空间 O(N2)O(N^2)O(N2) 二维数组存储区间状态。
特性 长度驱动 必须先算小区间，再算大区间。
