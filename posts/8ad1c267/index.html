Tarjan SCC 与缩点 (Strongly Connected Components)

1. 生态位 (Ecological Niche)

  • 定义: 在有向图中，找到最大子图，使得子图中任意两点 u,vu, vu,v 互相可达。
  • 解决痛点:
    • 化环为链: 图论问题中最棘手的是“环” (Cycle)。Tarjan 将每个强连通分量 (SCC) 视为一个点，将有向有环图转化为 DAG (有向无环图)。
    • 传递性分析: 如 P2812 中“软件共享”，环内的学校可以互传，不仅等价，还能共享资源。
  • 竞品对比:
    • vs Kosaraju: Kosaraju 需要两次 DFS，代码略长；Tarjan 仅需一次 DFS，常数更优，且更易扩展到割点/桥的求解。

2. 逻辑支点 (The Logical Pivot)

2.1 追溯值 (The Low-Link Value)

Tarjan 的灵魂在于两个时间戳的博弈：

  • dfn[u] (Timeline): 物理时间。节点 uuu 被 DFS 首次访问的次序。
  • low[u] (Capability): 回溯能力。节点 uuu 通过有向边（包含返祖边）能回溯到的、且仍在栈中的最小 dfn 值。

2.2 根的判定 (Root of SCC)

  • 判定: 当 DFS 回溯时，若 dfn[u] == low[u]，说明 uuu 无法回到比自己更早的祖先。
  • 结论: uuu 是当前 SCC 的根 (Root)。此时栈中 uuu 及其上方的所有节点构成一个完整的 SCC。

2.3 DAG 补全定理 (DAG Completion Theorem)

P2812 揭示了 DAG 连通性的两个重要结论（缩点后）：

  1. 源点覆盖 (Task A): 至少选几个点才能覆盖全图？
    • 结论: DAG 中 入度为 0 的点数 (ins)。
    • Logic: 非源点一定被别人连，只有源点必须手动触发。
  2. 强连通补边 (Task B): 至少加几条边使图强连通？
    • 结论: max⁡(ins,outs)\max(ins, outs)max(ins,outs)。
    • Logic: 我们需要把所有汇点 (Out=0) 连回源点 (In=0)。理想情况下 1 对 1 连接，剩下的多余部分串联即可。
    • 特判: 若 SCC 只有 1 个，答案为 0。

3. 实战部署 (Implementation)

  • Airy Code Style v3.0:
    • 注意 stack 的使用与 in_stk 标记的配合。
    • 缩点建图部分从 O(N2)O(N^2)O(N2) 优化为 O(M)O(M)O(M) 遍历原边集。

Run Tarjan    for( int i = 1; i <= n; ++ i )    {        if( ! dfn[i] ) tarjan( i );    }    // 2. Build DAG (Shrink Points)    // 遍历所有边 (u -> v)，若不在同一 SCC，则 scc[u] -> scc[v]    for( int u = 1; u <= n; ++ u )    {        for( int v : adj[u] )        {            int a = scc_id[u];            int b = scc_id[v];                        if( a != b )            {                outd[a] ++;                ind[b] ++;            }        }    }    // 3. Calculate Answers    int ins = 0, outs = 0;    for( int i = 1; i <= scc_cnt; ++ i )    {        if( ind[i] == 0 ) ins ++;        if( outd[i] == 0 ) outs ++;    }    // Task A: 覆盖全图需选多少源点    cout << ins << '\n';    // Task B: 强连通需加多少边    // Corner Case: 本身就是强连通图 (1个SCC)    if( scc_cnt == 1 )     {        cout << 0 << '\n';    }    else     {        cout << max( ins, outs ) << '\n';    }}int main( ){    ios::sync_with_stdio(0); cin.tie(0); cout.tie(0);    solve( );    return 0;}

4. 知识粘附 (Adhesion)

  • 前置知识: [DFS] (时间戳概念)。

  • 变体模型:

    • 割点/桥 (Cut Vertex/Bridge): low[v] >= dfn[u] (判定割点) vs low[v] > dfn[u] (判定桥)。逻辑相似但没有栈。

    • 2-SAT 问题: 利用 SCC 判定逻辑变量 xxx 和 ¬x\neg x¬x 是否在同一分量中。

  • 母题索引:

    • [Luogu P2812/P2821]: 校园网络 (DAG 补边定理)。

    • [Luogu P2341]: 受欢迎的牛 (缩点 + 出度唯一性)。

Architect’s Checklist:

  1. Low Logic: 检查了 low[u] 更新时 dfn[v] 与 low[v] 的区别（返祖边只能用 dfn，虽然在求 SCC 时混用有时能过，但在求割点时是致命错误，建议严格区分）。
  2. Corner Case: 检查了 scc_cnt == 1 的特判。
