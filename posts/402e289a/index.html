[Leaf] [ZJ2021B] Restore Atlantis

ZJ2021B - Restore Atlantis

1. 题面梗概

中译中: 有 nnn 个矩形，编号为 1,2,⋯ ,n1, 2,\cdots, n1,2,⋯,n。有 qqq 次查询，每次删除编号在 [si,ti][s_i,t_i][si​,ti​] 范围内的矩形。求剩下的矩形总面积。

约束条件：n,q≤105n,q \leq 10^5n,q≤105， x,y≤2000x,y \leq 2 000x,y≤2000

2. 逻辑推导

2.1 第一步推理

这题从各种意义上都不好切入啊。

我们尝试从 x,yx,yx,y 的范围切入，因为最大只有 200020002000，所以不用考虑复杂的矩形切割，我们只需要从 1∗11*11∗1 的格子出发即可。

那么对于一次询问，这个格子满足什么条件才计入统计呢？

对于一个格子 Sx,yS_{x,y}Sx,y​，设覆盖它的矩形集合为 Rx,yR_{x,y}Rx,y​。

在去除 [si,ti][s_i,t_i][si​,ti​] 区间的矩形后，若 s≤min(Rx,y)s \leq min(R_{x,y})s≤min(Rx,y​) 且 max(Rx,y)≤tmax(R_{x,y}) \leq tmax(Rx,y​)≤t，这个格子就是我们要从总面积中删除的部分。

2.2 第二步推理

由上述推导得知：我们只需要算出每个格子的 minid 和 maxid。

处理阶段：

  • 采用扫描线思维，按 xxx 轴扫描，线段树维护 yyy 轴这一列的 min/max。
  • 线段树节点：这里挂两个优先队列，大根堆和小根堆，分别处理 min/max。
  • 每一列处理完后，DFS 跑一遍线段树，把这一列的 min/max 记录下来。

统计阶段：

  • 现在我们有 2000×20002000 \times 20002000×2000 个点，每个点有两个属性 (minid,maxid)(minid, maxid)(minid,maxid)。
  • 接着查询 [s,t][s,t][s,t] 就转化为：有多少点满足 s≤minid≤ns \leq minid \leq ns≤minid≤n 且 1≤maxid≤t1 \leq maxid \leq t1≤maxid≤t。
  • 这就是一个典型的“二维偏序”问题。

2.3 第三步推理

计算阶段：

  • 如果我们要在线查询，那么复杂度 O(Q×G2)O( Q \times G^2)O(Q×G2)，这个复杂度是我们不能接受的。
  • 所以我们考虑借助树状数组离线处理。

离线处理：

  • 既然 maxid≤tmaxid \leq tmaxid≤t，那么我们就按 ttt 升序排序所有询问。
  • 凡是 maxid≤tmaxid \leq tmaxid≤t 的那些格子，都已经被删掉了，我们把这些格子的 minidminidminid 投入树状数组。
  • 那么树状数组里的，就都是那些“上限已经被 ttt 封死了的格子”。
  • 我们只需要查询，还有多少格子 minid≥sminid \geq sminid≥s？
  • 这就是个简单的后缀和查询：cnt = query(n) - query(s - 1);。

最后，通过总面积减去每次查询完的删除面积，统计答案，按顺序输出。

4. 复盘

  • 复杂度:

    • 预处理网格：O(G2+Nlog⁡N)O(G^2 + N \log N)O(G2+NlogN)。
    • 离线查询：O(G2+Qlog⁡N)O(G^2 + Q \log N)O(G2+QlogN)。

  • 碎碎念:

    • 由于直接统计面积太复杂了，所以我们采用补集法，删去不满足条件的格子。

    • 离线查询的精髓在于：通过排序把一个限制维度变成时间线，然后再处理另一个维度，这里我们借助树状数组维护。
