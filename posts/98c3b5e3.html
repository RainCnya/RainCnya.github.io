P2149 Elaxia的路线

Luogu-P2149 [SDOI2009] Elaxia的路线

题目简述

难度：提高+ | 题解格式：V6.3

给定一个无向图和两对起点终点 (x1,y1),(x2,y2)(x_1, y_1), (x_2, y_2)(x1​,y1​),(x2​,y2​)。求这两对点之间的任意一条最短路径中，最长的公共路径长度（公共路径必须是连续的边）。

  • 数据范围：N≤1500,M≤3×105N \le 1500, M \le 3 \times 10^5N≤1500,M≤3×105。

算法笔记可以参考：

  • 算法学习笔记：最短路图 (Shortest Path DAG) - RainCnya

解题思路

1. 初步认知

  • 直观想法：分别求出一条最短路然后比对？不行，最短路可能不唯一，公共部分可能存在于某两条特定的最短路之间。
  • 瓶颈分析：我们需要描述“所有最短路”的集合。这指向了 最短路图 (Shortest Path DAG) 的概念。

2. 主题渗透 (正解)

算法：最短路图 + 记忆化搜索 (O(Mlog⁡N+N)O(M \log N + N)O(MlogN+N))

  • 基本概念：
    判定一条边 u→wvu \xrightarrow{w} vuw​v 是否属于 S→TS \to TS→T 的最短路集合，充要条件是：

    dist(S,u)+w+dist(v,T)=dist(S,T)dist(S, u) + w + dist(v, T) = dist(S, T) dist(S,u)+w+dist(v,T)=dist(S,T)

    我们需要分别以 x1,y1,x2,y2x_1, y_1, x_2, y_2x1​,y1​,x2​,y2​ 为源点跑 4 次 Dijkstra，获取所有点到这 4 个端点的距离。
  • 状态定义：
    dfs(u)dfs(u)dfs(u) 表示从点 uuu 开始，沿着 x1→y1x_1 \to y_1x1​→y1​ 的最短路方向，能延伸的最长公共路径长度。
  • 转移逻辑：
    枚举 uuu 的邻居 vvv。
    1. 第一重筛选：边 (u,v)(u, v)(u,v) 必须在 x1→y1x_1 \to y_1x1​→y1​ 的最短路 DAG 上。
    2. 第二重筛选：边 (u,v)(u, v)(u,v) 必须在 x2→y2x_2 \to y_2x2​→y2​ 的最短路 DAG 上。注意公共路径可能是同向 (u→vu \to vu→v) 或 反向 (v→uv \to uv→u)，需分类讨论。
      若满足上述条件：

    dfs(u)=max⁡(dfs(u),w+dfs(v))dfs(u) = \max(dfs(u), w + dfs(v)) dfs(u)=max(dfs(u),w+dfs(v))

i <= n; ++ i ) dist[i] = inf;    dist[st] = 0;    priority_queue< PII, vector<PII>, greater<PII> > pq;    pq.push({ 0, st });    while( !pq.empty( ) )    {        auto [d, u] = pq.top( );        pq.pop( );        if( d > dist[u] ) continue;        for( auto [v, w] : adj[u] )        {            if( dist[u] + w < dist[v] )            {                dist[v] = dist[u] + w;                pq.push({ dist[v], v });            }        }    }}// 记忆化搜索求 DAG 最长路// dir = 0: 假设两人在公共路径上反向// dir = 1: 假设两人在公共路径上同向int dfs( int u, bool dir ){    if( memo[u] != -1 ) return memo[u];    int len = 0;    for( auto& [v, w] : adj[u] )    {        // 核心判定 1: 必须在 Elaxia (X1 -> Y1) 的最短路图上        // 判定式: dist(X1, u) + w + dist(v, Y1) == dist(X1, Y1)        if( (ll)d1[u] + w + d2[v] != d1[Y1] ) continue;        bool ok = 0;        if( dir )         {            // 同向: 必须也在 w** (X2 -> Y2) 的最短路图上 (u -> v)            ok = ( (ll)d3[u] + w + d4[v] == d3[Y2] );        }        else         {            // 反向: 必须也在 w** (X2 -> Y2) 的最短路图上 (v -> u)            ok = ( (ll)d3[v] + w + d4[u] == d3[Y2] );        }        if( ok ) len = max( len, w + dfs( v, dir ) );    }    return memo[u] = len;}int main( ){    ios::sync_with_stdio(0); cin.tie(0); cout.tie(0);    cin >> n >> m;    cin >> X1 >> Y1 >> X2 >> Y2;    dijkstra( Y1, d2 );    dijkstra( X2, d3 );    dijkstra( Y2, d4 );    int ans = 0;    // 2. 分类讨论两种相遇情况 (同向/反向) 并取最大值    memset( memo, -1, sizeof( memo ) );    for( int i = 1; i <= n; ++ i )        ans = max( ans, dfs( i , 0 ) ); 四、 结语

本题是 最短路图 (Shortest Path DAG) 的经典应用。

解决此类问题的通用范式是：

  1. 拓扑结构化：利用 dist(S,u)+w+dist(v,T)==dist(S,T)dist(S, u) + w + dist(v, T) == dist(S, T)dist(S,u)+w+dist(v,T)==dist(S,T) 将满足条件的边提取出来，这就构成了一个 DAG。

  2. DAG 上的 DP：在提取出的 DAG 上进行拓扑排序或记忆化搜索，解决最长链、计数等问题。
