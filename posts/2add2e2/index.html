1. P4391 Radio Transmission 无线传输

题意

给定一个字符串 S1S_{1}S1​，它是由某个字符串 S2S_{2}S2​ 不断自我连接形成的。但是字符串 S2S_{2}S2​ 是不确定的，要求其最短值。

举个例子：cabcabca 可以视为 cab 的不断连接 (cab,cab,ca)b

思路

如果一个字符串 S1S_{1}S1​ 是由 S2S_{2}S2​ 重复拼接而成的子串，那么 S1S_{1}S1​ 一定具有 周期性。

这里有一个结论：对于一个长度为 LLL 的字符串，如果它有一个长度为 xxx 的 Border，那么 L−xL - xL−x 就是该字符串的一个周期。

Border: 字符串 SSS 的一个子串，既是 SSS 的真前缀，也是 SSS 的真后缀。
周期：对于所有满足条件的 iii，都有 S[i]=S[i+k]S[i] = S[i+k]S[i]=S[i+k]，则 kkk 是 SSS 的周期。

如下图，把前后缀分开，令它们上下一一相等，此时红色段加上后缀就是原字符串。

接着可以推出，由于上下对应，0=1,2=3,4=5,6=7,8=90=1,2=3,4=5,6=7,8=90=1,2=3,4=5,6=7,8=9；
由于公共前后缀，1=2,3=4,5=6,7=8,9=101=2,3=4,5=6,7=8,9=101=2,3=4,5=6,7=8,9=10，因此，红色段 000 即循环子串。

题目要求最短周期子串，即 k=n−max(border)k = n - max(border)k=n−max(border)，问题就转化为了求最长 BorderBorderBorder，而最长 BorderBorderBorder 正好就是 KMP 算法中的 Next 数组，至此逻辑完整。

2. P3435 OKR-Periods of Words

题意

如果 QQQ 是 AAA 的真前缀，并且 AAA 是 QQQQQQ 的前缀，那么 QQQ 是 AAA 的周期，要求给定字符串 SSS 的所有前缀的最大周期长度之和。

字符串 AAA 的最大周期是其最长的周期，例如：ababab 的最大周期是 abab。

思路

参考上题思路，因为 QQQ 是 AAA 的真前缀，不妨记 A=QS′A = QS'A=QS′，而 AAA 是 QQQQQQ 的前缀，则 QS′QS'QS′ 是 QQQQQQ 的前缀，也就是说 S′S'S′ 是 QQQ 的前缀。

因为 S′S'S′ 本来是 AAA 的后缀，又是 AAA 的前缀（ S′S'S′ 是 QQQ 的前缀，QQQ 是 AAA 的前缀 ），所以，S′S'S′ 是 AAA 的一个 Border。

要使 QQQ 的长度最长，也就是 Border S′S'S′ 的长度最短。因此，对于长度为 iii 的前缀，其 最大周期长度 = i−min(Border)i - min(Border)i−min(Border)。

由于在 KMP 算法中，求得的 Next 是最长的 Border，我们没法直接套用，需要再进行修改。根据 Next 数组的性质，一个字符串的所有 Border 长度构成一个链式结构：

Border(i)={nxt[i],nxt[nxt[i]],nxt[nxt[nxt[i]]]… }Border(i) = \{ nxt[i], nxt[nxt[i]], nxt[nxt[nxt[i]]] \dots \} Border(i)={nxt[i],nxt[nxt[i]],nxt[nxt[nxt[i]]]…}

我们要找的就是这个序列中最后一个大于 000 的数，举个例子，babababa 的前缀 babab ( 长度为 5 )，nxt[5] = 3, nxt[3] = 1, nxt[1] = 0，所以最长周期长度 = 5−1=45 - 1 = 45−1=4。

由于可能会出现不断跳跃的情况，我们可以采用并查集的路径压缩思路来进行优化。

3. P3121 Censoring G

题意

给定一个字符串 SSS，要求删除其中所有的子串 TTT。（注：删除后可能会形成新的子串）

思路

模式串匹配，自然能想到 KMP 匹配，但是发现是删除操作，所以需要针对删除操作修改 KMP。

由于匹配成功等于这个串删除了，都是在序列的右端进行操作，所以想到了 栈 这种数据结构，但是这样就会遇到一个新问题，如何处理删除后产生的新串，也就是 jjj 指针应该如何移动呢？

如果按照朴素的 KMP 来说，匹配成功后 jjj 要转移到 nxt[j]nxt[j]nxt[j] 处，问题是原串变了，这个就不一定成立（样例比较良心，告诉你这样是过不去的）。发现对于一个没有匹配成功的字符，它对应的目标字符串的位置是不会变的，所以就可以开一个辅助数组来记录这个对应位置。

4. P2375 动物园

题意

在 KMP 算法的 next 数组基础上，求出字符串的 num 数组，定义为：对于字符串 SSS 的前 iii 个字符构成的子串，既是它的后缀又同时是它的前缀，并且该后缀与该前缀不重叠，这种字符串的数量记为 num[i]。

举个例子：若 SSS 为 aaaaa，则 num[4] = 2，因为长度为 3 的 aaa 前后缀重叠了，就不满足条件。

最后输出 ∏(num[i]+1) mod (109+7)\prod( num[i] + 1 ) \bmod (10^9+7)∏(num[i]+1)mod(109+7)。

思路

前面推导过，对于一个前缀 iii 的 next[i]next[i]next[i]，它长这样：

Border(i)={nxt[i],nxt[nxt[i]],nxt[nxt[nxt[i]]]… }Border(i) = \{ nxt[i], nxt[nxt[i]], nxt[nxt[nxt[i]]] \dots \} Border(i)={nxt[i],nxt[nxt[i]],nxt[nxt[nxt[i]]]…}

那么，为了统计 numnumnum 数组，我们只需要在预处理 nextnextnext 数组的同时把这个数量递推一下就行，这样就得到了题目下弱化的 numnumnum 数组：可以重叠的公共前后缀数量。

接着我们只需要顺着第二个限制条件去重即可，首先 next[i]<inext[i] < inext[i]<i，也就是说，如果一个递归了 kkk 次的 nextnextnext，如果比原前缀 iii 的长度的一半要小，那么这个 nextnextnext 递推出来的答案 ansansans 就是本题下 iii 的真正 numnumnum 了。

5. P3426 SZA-Template

题意

给定一个字符串 SSS，要求找出其中的最短印章（模版） TTT 来覆盖原串。

例子如下：印章 TTT 是 ababbaba，长度为 8。

思路

注意到印章 TTT 一定是 SSS 的前后缀，不然没法印到开头和结尾，所以 TTT 一定是原串 SSS 的一个 Border。虽然印章一定是 Border，但是 Border 不一定是印章，比如 abababa 的 Border 有 aba 和 a，但是 a 显然不行。

那么我们就需要判断：对于长度为 iii 的前缀 S[1…i]S[1\dots i]S[1…i]，它的最短印章长度 f[i]f[i]f[i] 是多少？这里可以采用 DP 的思想，首先考虑 nxt[i]=0nxt[i] = 0nxt[i]=0 的情况，此时 f[i]=if[i] = if[i]=i，因为没有更短的子串能覆盖它。

为什么采用 DP 的思想？

  1. 最优子结构：一个大串的印章，如果能缩短，那个缩短后的印章必然也是它某个 Border 的印章。
  2. 重叠子问题：在判定 f[i]f[i]f[i] 时，其实是在利用已经算好的 f[nxt[i]]f[nxt[i]]f[nxt[i]]。
  3. 无后效性：当我们确定了 f[i]f[i]f[i] 的值以后，只需要关心这个长度以及它最后出现的位置，并不需要知道前缀 iii 中的印章轨迹。只要 f[i]f[i]f[i] 算出来了，它对后续 f[k](k>i)f[k] (k > i)f[k](k>i) 的贡献就是确定的。

然后考虑能否使用更短的印章，如果能，那么这个印章肯定也能覆盖更短的 Border，记 L=nxt[i]L = nxt[i]L=nxt[i]，那么也就是 f[i]=f[L]f[i] = f[L]f[i]=f[L]。什么情况下可以覆盖到呢？

模拟后发现，f[L]f[L]f[L] 能覆盖 S[1…i]S[1\dots i]S[1…i] 的充要条件是：在 [i−L,i−1][i-L,i-1][i−L,i−1] 之间，曾经出现过一个位置 jjj，使得 f[j]=f[L]f[j] = f[L]f[j]=f[L]。

直观上的理解就是：f[L]f[L]f[L] 已经能覆盖前缀 S[1…L]S[1\dots L]S[1…L]。如果在靠近结尾的地方，f[L]f[L]f[L] 再次出现并覆盖了某个前缀 S[1…j]S[1\dots j]S[1…j]，那么这两次 “盖印” 就把中间的空隙填充了。
