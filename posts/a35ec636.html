算法学习笔记：[图论/最短路] - Floyd

算法学习笔记：图论/最短路 - Floyd

一、 初步认知

领域：图论 (Graph Theory)
子领域：多源最短路径 (APSP)
典型设问：N≤500N \le 500N≤500，求任意两点间的最短距离；或者涉及连通性传递、中转点限制的问题。

Floyd-Warshall 是解决任意两点间最短路径 (All-Pairs Shortest Path) 的经典动态规划算法。虽然 O(N3)O(N^3)O(N3) 的复杂度限制了其在大规模图上的应用，但其代码极简且支持负权边（无负环），在小图、传递闭包及动态加点场景下具有不可替代的地位。

二、 主题渗透

1. The Struggle (模拟推演)

核心冲突：为什么要三层循环？为什么 k 必须在最外层？
Floyd 的本质是 DP。状态 dp[k][i][j]dp[k][i][j]dp[k][i][j] 表示“只允许经过节点 1…k1 \dots k1…k 作为中转点”时，iii 到 jjj 的最短路径。
如果我们把 k 放在内层，意味着我们试图用“经过任意中转点”的状态来更新自己，这违背了 DP 的阶段性（无后效性）。

▶
模拟推演 (k循环的物理意义)

场景：1→21 \to 21→2 (w=5), 1→31 \to 31→3 (w=1), 3→23 \to 23→2 (w=1)。
目标：求 1→21 \to 21→2。

错误的循环顺序 (k 在内层)：

  • i=1,j=2i=1, j=2i=1,j=2：尝试所有 kkk。此时 dist[1][3]dist[1][3]dist[1][3] 和 dist[3][2]dist[3][2]dist[3][2] 可能还没被更新成最优（还是 INF 或初始边权）。
  • 结果：可能错过 1→3→21 \to 3 \to 21→3→2 这条路。

正确的循环顺序 (k 在外层)：

  • 阶段 k=1 (中转点 {1})：无变化。
  • 阶段 k=2 (中转点 {1,2})：无变化。
  • 阶段 k=3 (中转点 {1,2,3})：
    • 扫描到 dist[1][2]dist[1][2]dist[1][2]。尝试用 333 松弛。
    • dist[1][2]=min⁡(5,dist[1][3]+dist[3][2])=min⁡(5,1+1)=2dist[1][2] = \min(5, dist[1][3] + dist[3][2]) = \min(5, 1+1) = 2dist[1][2]=min(5,dist[1][3]+dist[3][2])=min(5,1+1)=2。
    • 成功更新。

2. Hack 思维 (Generator 视角)

  • 循环顺序 Hack：这是 Floyd 最容易写错的地方。如果你写了 for i for j for k，在构造一个链式图（如 1→2→3→⋯→N1 \to 2 \to 3 \to \dots \to N1→2→3→⋯→N）时，你的算法将无法传递连通性，只能计算直接邻居。
  • INF 溢出 Hack：在 dist[i][k]+dist[k][j]dist[i][k] + dist[k][j]dist[i][k]+dist[k][j] 时，如果两个都是 0x3f3f3f3f，相加会超过 int 范围变成负数。
    • 应对：必须判断 dist[i][k] != inf && dist[k][j] != inf。

三、 核心思想

"阶段性动态规划 (Phase-based DP)"

将“允许经过的中转点集合”作为 DP 的阶段。
从“只允许经过 1”推广到“只允许经过 1, 2”，最终推广到“允许经过 1…N”。
方程：dp[k][i][j]=min⁡(dp[k−1][i][j],dp[k−1][i][k]+dp[k−1][k][j])dp[k][i][j] = \min(dp[k-1][i][j], dp[k-1][i][k] + dp[k-1][k][j])dp[k][i][j]=min(dp[k−1][i][j],dp[k−1][i][k]+dp[k−1][k][j])。
空间优化后即为：dist[i][j]=min⁡(dist[i][j],dist[i][k]+dist[k][j])dist[i][j] = \min(dist[i][j], dist[i][k] + dist[k][j])dist[i][j]=min(dist[i][j],dist[i][k]+dist[k][j])。

四、 实战部署

1. 代码模板

▶
Floyd Template 
#include<bits/stdc++.h>using namespace std;using ll = long long;const int maxn = 505;const int inf = 0x3f3f3f3f;int dist[maxn][maxn];int n, m;void floyd( ){    // 标准 Floyd：k 必须在最外层    for( int k = 1; k <= n; ++ k )    {        for( int i = 1; i <= n; ++ i )        {            // 剪枝优化：如果 i 到 k 不通，则无法通过 k 中转            if( dist[i][k] == inf ) continue;            for( int j = 1; j <= n; ++ j )            {                if( dist[k][j] == inf ) continue;                if( dist[i][k] + dist[k][j] < dist[i][j] )                {                    dist[i][j] = dist[i][k] + dist[k][j];                }            }        }    }}void init( ){    for( int i = 1; i <= n; ++ i )    {        for( int j = 1; j <= n; ++ j )        {            if( i == j ) dist[i][j] = 0;            else dist[i][j] = inf;        }    }}

2. 部署细节

  • 初始化：对角线为 0，其余为 INF。读入边时注意重边（取 min）。

  • 传递闭包：如果只关心连通性（bool），方程改为 dp[i][j] |= (dp[i][k] & dp[k][j])。

  • 最小环：在最外层 k 刚开始，内部 i, j 还没用到 k 此时 dist[i][j] 表示“不经过 k 的最短路”。此时 dist[i][j] + w[j][k] + w[k][i] 可构成一个经过 k 的最小环。

  • 动态加点：(如 P1119) 若节点按顺序解锁，只需执行对应 k 的那一层循环，无需重跑整个算法。

五、 维度概览

维度 描述
时间复杂度 O(N3)O(N^3)O(N3) (严丝合缝，无法优化)
空间复杂度 O(N2)O(N^2)O(N2) (邻接矩阵)
核心考点 动态加点 (P1119)、传递闭包、最小环 (P6175)。
