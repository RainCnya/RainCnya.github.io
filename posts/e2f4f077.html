Luogu-P1119 灾后重建

Luogu-P1119 灾后重建

题目简述

难度：提高 | 题解格式：V6.3

给出 NNN 个村庄和 MMM 条双向道路。村庄 iii 在时刻 tit_iti​ 重建完成（tit_iti​ 非降）。有 QQQ 个询问 (x,y,T)(x, y, T)(x,y,T)，求在时刻 TTT 时，村庄 xxx 和 yyy 之间的最短路径长度。若不可达或点未重建，输出 −1-1−1。

  • 数据范围：N≤200,Q≤50000,tiN \le 200, Q \le 50000, t_iN≤200,Q≤50000,ti​ 非降，询问时间 TTT 非降。

算法笔记可以参考：

  • 算法学习笔记：Floyd 的 DP 本质与阶段性更新 - RainCnya

解题思路

1. 初步认知

  • 直观想法：对于每个询问，只保留满足 tu≤Tt_u \le Ttu​≤T 的点，跑一遍 Dijkstra 或 BFS。
  • 瓶颈分析：单次最短路 O(Mlog⁡N)O(M \log N)O(MlogN) 或 O(N2)O(N^2)O(N2)。总复杂度 O(Q⋅N2)O(Q \cdot N^2)O(Q⋅N2)，在 Q=50000Q=50000Q=50000 时会超时。我们需要利用 N≤200N \le 200N≤200 这个极小的数据范围以及询问时间 TTT 单调递增的特性。

2. 主题渗透 (正解)

算法：Floyd (动态加点) (O(N3+Q)O(N^3 + Q)O(N3+Q))

  • 基本概念：回归 Floyd 算法的 DP 本质。
  • 状态定义：
    dp[k][i][j]dp[k][i][j]dp[k][i][j] 表示只允许经过前 kkk 个已重建的节点作为中转点时，iii 到 jjj 的最短路径。
    注意：这里的“前 kkk 个”是指按重建时间排序后的前 kkk 个点。
  • 转移逻辑：
    标准 Floyd 的最外层循环 k 正是代表“阶段”。
    由于询问的时间 TTT 是非降的，我们不需要每次重跑 Floyd。维护一个指针 k，当询问时间推进到 TTT 时，将所有满足 tnow≤Tt_{now} \le Ttnow​≤T 的新节点加入 Floyd 的外层循环进行松弛。

    dist[i][j]=min⁡(dist[i][j],dist[i][k]+dist[k][j])dist[i][j] = \min(dist[i][j], dist[i][k] + dist[k][j]) dist[i][j]=min(dist[i][j],dist[i][k]+dist[k][j])

    这样均摊复杂度仅为 O(N3)O(N^3)O(N3)。

AC 代码

▶
AcCode 
#include<bits/stdc++.h>using namespace std;const int maxn = 200 + 50;const int maxm = 4e4 + 50;const int inf = 1e9;int ts[maxn];int dist[maxn][maxn];int n, m, q;int main( ){    ios::sync_with_stdio(0);    cin.tie(0); cout.tie(0);    cin >> n >> m;    // 初始化邻接矩阵    for( int i = 1; i <= n; ++ i )    {        for( int j = 1; j <= n; ++ j )        {            if( i == j ) dist[i][j] = 0;            else dist[i][j] = inf;         }    }    for( int i = 1; i <= n; ++ i )    {        cin >> ts[i];    }    for( int i = 1; i <= m; ++ i )    {        int u, v, w;        cin >> u >> v >> w;        u ++, v ++; // 1-based index        dist[u][v] = dist[v][u] = w;    }    int k = 1; // 这里的 k 指向当前“待处理”的节点编号（按输入顺序即时间顺序）    cin >> q;    while( q -- )    {        int x, y, t;        cin >> x >> y >> t;        x ++, y ++; // 1-based index        // 动态更新 Floyd：将所有在当前时刻 t 之前（或恰好）重建的点加入松弛集合        while( k <= n && ts[k] <= t )        {            for( int i = 1; i <= n; ++ i )            {                for( int j = 1; j <= n; ++ j )                {                    // 仅当路径存在时才更新，避免 overflow                    if( dist[i][k] != inf && dist[k][j] != inf )                    {                        dist[i][j] = min( dist[i][j], dist[i][k] + dist[k][j] );                    }                }            }            k ++;        }        // 易错点：除了不可达，还要判断起终点本身是否已重建        if( ts[x] > t || ts[y] > t || dist[x][y] == inf )            cout << -1 << '\n';        else            cout << dist[x][y] << '\n';     }    return 0;}

四、 结语

本题深刻揭示了 Floyd 算法中最外层循环 k 的物理意义。Floyd 不仅仅是一个静态求全源最短路的黑盒，更是一个支持动态加点（增加中转点）的在线算法。掌握这一点对于理解图论中的 DP 思想至关重要。
