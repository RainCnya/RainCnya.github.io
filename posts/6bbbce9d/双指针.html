算法学习笔记：双指针 (Two Pointers) -V3.0

一、 初步认知 (Macro View)

1. 叫什么 (Identity)

  • 中文名： 双指针
  • 英文名： Two Pointers
  • 别名/相关名词： 尺取法

2. 什么地位 (Status)

  • 所属领域： 基础算法 / 优化技巧
  • 核心地位： 处理有序序列上的区间或配对问题。是能够将 O(N2)O(N^2)O(N2) 的暴力两重循环优化到 O(N)O(N)O(N) 的关键技巧。

3. 有什么用处 (Utility)

  • 解决的核心问题： 在序列上高效查找满足某种条件的区间、子序列或配对，前提是该条件具备单调性。
  • 典型应用场景：
    • 查找数组中和为 KKK 的两个数（对撞指针）。
    • 查找符合 A[r]−A[l]≤KA[r] - A[l] \le KA[r]−A[l]≤K 的最长区间（同向尺取法）。
    • 快速排序和归并排序中的分区/合并操作。

二、 主题渗透 (Theme Osmosis)

1. 核心原理 (The Eureka Moment)

  • 底层思想： 降维 + 单调性。
  • 关键推导：
    • 降维： 暴力法需要枚举 LLL 和 RRR 两个变量，是 O(N2)O(N^2)O(N2)。双指针通过利用序列的单调性，让 LLL 和 RRR 的移动具有单向性。
    • 核心机制： 当 RRR 向前移动一步时，LLL 只会向前移动，而永远不会后退。
      • 这样，LLL 和 RRR 在整个过程中，总共只会遍历序列一次（最多 2N2N2N 次移动），从而将时间复杂度降为 O(N)O(N)O(N)。

2. 详细实现过程 (Step-by-Step)

  1. 预处理 (Pre-condition)： 几乎所有双指针问题都要求序列有序，因此通常第一步是 O(Nlog⁡N)O(N \log N)O(NlogN) 的排序。
  2. 指针初始化 (Initialization)：
    • 尺取法： LLL 和 RRR 都从起始位置开始，如 L=1,R=1L=1, R=1L=1,R=1。
    • 对撞指针： L=1,R=NL=1, R=NL=1,R=N。
  3. 核心循环 (Iteration)： 以尺取法为例：
    • RRR 指针主动向前推进（外层循环）。
    • 在 RRR 的位置， LLL 指针被动地根据条件（如 A[R]−A[L]>KA[R] - A[L] > KA[R]−A[L]>K）向前推进（内层 while 循环），直到满足条件。
  4. 答案维护 (Maintenance)： 在每次 RRR 移动或 LLL 移动后，维护当前最优的区间长度或配对结果。

三、 总结分析 (Analysis & Comparison)

1. 复杂度分析 (Complexity)

维度 复杂度 说明
预处理时间 O(Nlog⁡N)O(N \log N)O(NlogN) 主要消耗在对数组进行排序（必要时）
双指针过程 O(N)O(N)O(N) 两个指针总共的移动次数不超过 2N2N2N
总时间 O(Nlog⁡N)O(N \log N)O(NlogN) 排序为瓶颈

2. 核心注意点 (Key Pitfalls)

  • 前提条件： 算法的 O(N)O(N)O(N) 优化严格依赖于序列的单调性以及所求问题的单调性。若序列无序，双指针无法使用。
  • 指针关系： 必须确保 LLL 和 RRR 指针的移动逻辑能够保证它们永不后退，这是 O(N)O(N)O(N) 复杂度的保障。
  • 边界维护： 在同向指针中，维护区间长度通常是 R - L + 1；在对撞指针中，需要注意 LLL 和 RRR 最终是否重合或错开。

四、 代码模板 (Code Template)

{% fold info @尺取法 %}

int a[maxn];
// 使用 1-based 索引
int main()
{
    int n, k;
    cin >> n >> k;
    for( int i = 1; i <= n; ++ i )
        cin >> a[i];

    // 1. 预处理：排序保证数列有序
    sort(a + 1, a + n + 1);

    int l = 1;
    int max_len = 0; // 记录最长有效区间

    // 2. 核心逻辑：R 主动推进，L 被动收缩
    for(int r = 1; r <= n; ++r)
    {
        // L 被动收缩：当条件不满足时，L 向前移动
        // 这里的条件是：区间 [L, R] 范围太大 (a[r] - a[l] > k)
        while(a[r] - a[l] > k && l <= r)
            l++;

        // 3. 维护答案：此时 [L, R] 是以 R 结尾的最长有效区间
        max_len = max(max_len, r - l + 1);
    }

    // cout << max_len << '\n';
    return 0;
}
