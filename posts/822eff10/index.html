[Note] 树图DP

[L4] 树形/图形状态 动态规划

顾名思义，是以树或图为模型进行的动态规划。由于树和图本身具有明确的结构关系，DP 的状态转移往往紧扣其拓扑性质。

一、核心特征

1. 树形 DP

  • 天然子结构：树的父子关系构成了天然的层级。通常通过 DFS（后序遍历） 实现，先递归处理子节点，再利用子节点的 DP 值回溯推导父节点。

  • 状态转移：状态通常包含 f[u][… ]f[u][\dots]f[u][…]，表示以 uuu 为根的子树内的某种最优解。

  • 树上背包优化：在合并子树 vvv 到 uuu 时，限制循环上限为当前已处理节点数 sz[u]sz[u]sz[u]。本质是枚举每对点在 LCA 处合并，总复杂度为 O(N2)O(N^2)O(N2) 或 O(N⋅M)O(N \cdot M)O(N⋅M)。

2. 图形 DP

  • 拓扑序：图上的 DP 必须在 DAG 上进行，或者通过 SCC 缩点 转化为 DAG，以保证转移的无环性。

  • 层级划分：某些图 DP 通过 “层” 的概念（如时间步 ttt、倍增步 2k2^k2k）将图转化为线性/分层结构。

二、模型分类

树形DP

换根DP

树形 DP 中的换根 DP 问题又被称为二次扫描，通常不会指定根结点，并且根结点的变化会对一些值，例如子结点深度和、点权和等产生影响。

  • 相关例题：

    • P2986 Great Cow Gathering G：换根DP，树上重心。

树上背包

在树结构上分配“容量”（如点数、边数），子树之间构成典型的分组背包关系。

  • 典型特征：f[u][j]f[u][j]f[u][j] 表示 uuu 子树内选 jjj 个单位。转移时枚举分配给子树的容量 kkk。

  • 相关例题：

    • P1352 没有上司的舞会：子节点状态推导父节点状态。

    • P2015 二叉苹果树：树上背包基础，边权转点权的技巧。

    • P2014 选课：有依赖的背包，多叉树上的容量分配。

    • P1273 有线电视网：树上背包，复杂度优化。

图形DP

DAG 上的 DP

在拓扑序基础上结合概率、期望或路径计数。

  • 相关例题：

    • P4316 绿豆蛙的归宿：DAG 上的期望 DP，反向建图。

    • P2656 采蘑菇：强连通分量 (SCC) 缩点 + DAG 最长路。

    • P7077 函数调用：利用拓扑序反向计算执行权重的贡献统计。

路径上的 DP

此类模型中，状态和路径有关，例如说两点之间可达性 / 最短路，从 uuu 走 kkk 步到 vvv 的方案数。

由于路径的分层不是特别明显，一般以路径长度或者点的顺序，或者题目的特殊条件来构造 DP 状态。

  • 相关例题：

    • P1613 跑路：利用倍增思想 (2k2^k2k) 重新建图，再跑最短路。

    • P6772 美食家：图上的状态转移 / 矩阵加速优化。

    • P3953 逛公园：最短路 + kkk 维度扩展，记忆化搜 + 计数DP。
