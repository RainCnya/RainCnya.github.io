区间 DP 进阶：从费用提前到逆向分治

[专题] 区间 DP 的多重面孔：从费用提前到逆向分治

1. 前言：不止是“石子合并”

提起区间 DP，很多人的第一反应公式就是：

dp[i][j]=max⁡k(dp[i][k]+dp[k+1][j])+costdp[i][j] = \max_k (dp[i][k] + dp[k+1][j]) + cost dp[i][j]=kmax​(dp[i][k]+dp[k+1][j])+cost

这是最经典的“石子合并”模型。它的核心逻辑是：大区间由两个小区间合并而来。

但在更复杂的题目中，区间 DP 的难点往往不在于“枚举断点 kkk”，而在于：

  1. 合并的代价不是静态的，而是随着时间/操作累积的。
  2. 合并的规则受到边界状态（如颜色、形状）的严格限制。
  3. 合并的方向可能需要逆向思考，从“被消除”的元素切入。

本专题将通过三道极具代表性的题目，带你领略区间 DP 的变幻莫测。

2. 进阶：三种变体策略

变体一：时间维度——费用提前计算

场景：当操作需要消耗时间，且未处理的元素会持续产生代价时。

在 P1220 关路灯 中，老张关灯的过程是一个区间扩大的过程。
难点在于：移动需要时间，而移动过程中，所有未关的灯都在耗电。
如果我们只记录“已关掉的灯的耗电”，状态就需要包含“当前时间”，导致维度爆炸。

思维转换（费用提前计算）：

与其计算“过去耗了多少电”，不如计算“现在的移动会造成未来多少电的浪费”。
当我们从 iii 走到 jjj 花费 ttt 秒时，这 ttt 秒不仅影响现在，还意味着剩下的所有灯都要多亮 ttt 秒。
因此，状态只需记录：dp[i][j][0/1]dp[i][j][0/1]dp[i][j][0/1]（关掉 [i,j][i,j][i,j] 区间，人停在左/右端点）。
转移时，将移动时间 ×\times× 剩余总功率累加进代价。

变体二：状态维度——边界状态记录

场景：当区间合并受到端点状态（如颜色、匹配关系）的限制时。

CF149D Coloring Brackets 是处理括号序列染色的经典题。
括号序列本身就是天然的递归结构（区间），非常适合区间 DP。
但题目限制繁多：匹配括号必须染且只能染一个、相邻括号颜色不能相同。

思维转换（增加维度）：

仅仅知道子区间 [l,r][l, r][l,r] 的方案数是不够的，因为合并时我们需要检查 lll 和 l−1l-1l−1、rrr 和 r+1r+1r+1 是否颜色冲突。
因此，必须把边界颜色写入状态：
dp[l][r][color_l][color_r]dp[l][r][color\_l][color\_r]dp[l][r][color_l][color_r]。
在转移时（分为“嵌套”和“拼接”两种情况），枚举分割点的颜色，检查是否满足题目约束。

变体三：逆向思考——辅助 DP 与分治

场景：当区间内的元素是被“单独”消灭，而不是两两合并时。

P5851 Greedy Pie Eaters 的题意比较新颖：牛吃掉区间内的派。
如果我们正向思考“哪头牛先吃”，状态极其复杂。

思维转换（逆向分治）：

考虑区间 [i,j][i, j][i,j]，我们枚举区间内某一个派 kkk 是被哪头牛吃掉的。
这就把区间分成了 [i,k−1][i, k-1][i,k−1] 和 [k+1,j][k+1, j][k+1,j] 两个独立子问题。

难点在于：如何快速知道“能覆盖 kkk 且在 [i,j][i, j][i,j] 范围内的牛的最大体重”？
这里引入了一个辅助 DP 数组 g[i][j][k]g[i][j][k]g[i][j][k]，专门预处理这个最大值。
这种**“主 DP + 辅助 DP”**的双层结构，在复杂区间问题中非常有效。

3. 核心对比总结

题目 核心难点 状态设计的突破口 关键技巧
P1220 时间流逝导致代价累积 增加 [0/1][0/1][0/1] 维记录位置 费用提前计算 (消除时间维)
CF149D 相邻颜色冲突限制 增加 [cl][cr][cl][cr][cl][cr] 维记录颜色 记忆化搜索 (处理复杂结构)
P5851 覆盖与消除的顺序 引入 ggg 数组预处理覆盖值 逆向分治 (枚举最后被吃的点)

总结与建议：

  1. 什么时候需要增加维度？ 当合并两个区间时，仅凭子区间的“最优解数值”无法判断合并是否合法（如颜色冲突），或者无法计算合并代价（如位置距离）时，必须把关键的边界信息加入状态。
  2. 费用提前计算是解决“累积型代价”问题的神器，它能成功消掉一个维度（时间/轮数）。
  3. 区间 DP 不一定非要是循环：对于像 CF149D 这种结构复杂的题目（嵌套 vs 拼接），写成记忆化搜索往往比递推更清晰，也更不易出错。
