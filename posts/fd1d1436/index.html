[L2] 50_计算几何

核心哲学：计算几何不是在玩尺规作图，而是在用“不精准”的浮点数去模拟精准的物理世界。

它的核心逻辑只有两个：
一是如何优雅地处理精度误差，二是如何通过“扫描”或“旋转”把杂乱分布的点变成有序的线条。

1. 知识图谱与演化

  • 核心工具: 向量运算、极角排序、凸性观察、扫描线。

  • 演化路径:

    1. 点线面基础：最原始的坐标运算。学会判断点与线的相对位置，以及线段之间的交点判定。

    2. 凸性封装：凸包。识破一组点中只有“最外面那一圈”是有用的，通过剔除内部点来极大缩小搜索空间。

    3. 旋转与扫描：利用单调性。让尺子（扫描线）在平面上划过，或让卡尺（旋转卡壳）绕着凸包转动，把静态分布变成动态过程。

    4. 多维与切割：半平面交。将多次的区域切割操作合并，寻找所有限制条件下的共同重叠区域。

2. 决策矩阵

  • 涉及“点集最远距离”或“求外接形状”:

    • 优先求凸包。若要求直径，在凸包基础上利用 [[Note] 旋转卡壳] 寻找对踵点。

  • 涉及“矩形面积并”或“复杂覆盖统计”:

    • 采用 [[Note] 扫描线]。将二维平面切成条状，配合线段树动态维护。

  • 涉及“判断一个点是否在多个限制区域内”:

    • 采用 [[Note] 半平面交]。寻找所有半平面的共同交集空间。

  • 遇到浮点数报错或诡异的判题失败:

    • 检查精度控制逻辑（严禁直接用 == 判断浮点数）。

    • 若题目给定的是整数坐标且不涉及开方，强制全过程使用 long long 运算以彻底杀掉精度损耗。

3. 核心板块索引

3.1 几何基础

  • [L3] 51 几何基础: 计算几何的起点。

    • [[Note] 精度控制]: 核心是“不直信浮点数”，通过极小偏移量修正逻辑判定。

    • [[Note] 向量工具]: 利用叉积判断方向，利用点积判断投影。

3.2 凸包与旋转卡壳

  • [L3] 52 凸包与旋转卡壳: 提取点集的骨架。

    • [[Note] 安德鲁算法]: 通过两次扫描稳健地构建点集的最外围边界。

    • [[Note] 旋转卡壳]: 利用凸包单调性，像转动卡尺一样寻找几何特征。

3.3 扫描线与半平面

  • [L3] 53 扫描线与半平面: 解决区域覆盖与切割。

    • [[Note] 扫描线]: 将二维面积问题降维成一根动线在时间轴上的积分。

    • [[Note] 半平面交]: 排序后利用双端队列寻找多个切割区域的共同核心。

3.4 三维几何

  • [L3] 54 三维几何: 从平面跨越到立体空间。

    • [[Note] 空间向量基础]: 引入三维叉积（法向量）与空间平面方程。

    • [[Note] 三维凸包]: 利用面片翻转逻辑包裹空间点集。

    • [[Note] 最小外接球]: 空间随机增量法处理最远点对约束。

4. 依赖与规划

  • 前置依赖: 熟练掌握双精度浮点数特性、基础三角函数、以及线段树优化（用于扫描线）。

  • 学习路径:

    1. P1 (基础)：构建一套没有任何冗余的基础几何模板（点、线、叉积算子）。

    2. P2 (中坚)：掌握凸包与扫描线。这是区域赛几何题目的生存底线。

    3. P3 (顶尖)：攻克半平面交、闵可夫斯基和、以及动态凸包等复杂模型。
