05 计算几何 核心指南

05 计算几何 核心指南

核心哲学：计算几何不是在玩尺规作图，而是在用“不精准”的浮点数去模拟精准的物理世界。
它的核心逻辑只有两个：
一是如何优雅地处理那点儿讨人嫌的精度误差，
二是如何通过“扫描”或“旋转”把杂乱分布的点变成有序的线条。

1. 知识图谱与演化

  • 核心抓手: 向量运算、极角排序、凸性观察、扫描线。

  • 演化路径:

    1. 点线面基础：最原始的坐标运算。学会如何判断一个点在不在直线左边，以及两条线段怎么求交点。

    2. 凸性封装：凸包。发现一组点中只有“最外面那一圈”是有用的，通过扔掉内部点大幅简化搜索空间。

    3. 旋转与扫描：利用单调性。让尺子（扫描线）在平面上划过，或者让卡尺（旋转卡壳）绕着凸包转动，把静态的分布变成动态的过程。

    4. 多维与切割：半平面交。把无数次的切割操作合并，寻找最终留下的那个共同重叠的“核心区域”。

2. 决策矩阵

  • 涉及“点集最远距离”或“求外接形状”: 优先求凸包。如果需要求直径，在凸包基础上利用旋转卡壳寻找对踵点。

  • 涉及“矩形面积并”或“复杂覆盖统计”: 采用扫描线。将二维平面切成一根根竖线，配合线段树动态维护覆盖长度。

  • 涉及“判断一个点是否在很多个限制区域内”: 采用半平面交。寻找所有半平面的共同交集。

  • 遇到浮点数报错或诡异的 WA:

    • 检查 eps 精度控制（不要直接用 == 判断）。

    • 如果题目给的是整数坐标且没有开方运算，尝试全过程使用 long long 避免精度损耗。

3. 核心板块索引

3.1 几何基础

  • [L3] 01 几何基础: 计算几何的敲门砖。

  • [Note] 精度控制: 核心是“不要直接相信浮点数”，通过一个极小的偏移量来修正判定。

  • [Note] 向量工具: 利用叉积判断点在向量的哪一侧，利用点积判断投影位置。

3.2 凸包与旋转卡壳

  • [L3] 02 凸包与旋转卡壳: 提取点集的骨架。

  • [Note] 安德鲁算法 (Andrew)：通过两次扫描（上凸壳+下凸壳）稳健地构建出最外围。

  • [Note] 旋转卡壳: 核心是“利用凸包的单调性”，像转动卡尺一样寻找最远点对。

3.3 扫描线与半平面

  • [L3] 03 扫描线与半平面: 解决区域覆盖与切割。

  • [Note] 扫描线: 核心是“把二维降成一维”，把平面的面积问题变成一根动线在时间轴上的积分。

  • [Note] 半平面交: 核心是“排序后维护双端队列”，寻找所有切割区域共同围成的凸多边形。

3.4 三维几何

  • [L3] 04 三维几何: 从平面跨越到立体空间。

  • [Note] 三维凸包: 通过不断“面片翻转”来包裹空间点集。

4. 依赖与规划

  • 前置依赖: 必须掌握基本的三角函数、double 类型精度特性、以及基础的线段树原理（用于扫描线优化）。

  • 学习路径:

    1. P1 (基础)：写出一套稳健、简洁且没有任何冗余的几何模板（点、线、叉积）。

    2. P2 (中坚)：掌握凸包和扫描线。这是区域赛几何题目的“及格线”。

    3. P3 (顶尖)：攻克半平面交、闵可夫斯基和、以及动态凸包。
