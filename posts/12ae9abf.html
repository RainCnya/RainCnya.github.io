Luogu-P2519 problem a

Luogu-P2519 [HAOI2011] problem a

一、 题目分析

1. 题目大意

nnn 个人考试，每个人 (i)(i)(i) 给出两个数：aia_iai​ (成绩比我高的人数) 和 bib_ibi​ (成绩比我低的人数)。

求最少有几个人没有说真话。

2. 元信息与数据规模

二、 解题思路

1. 朴素解法 (O(N^2) 2D-LIS)

一个常见的思路是将其看作二维偏序。

  • 如果两个人 iii 和 jjj 都说了真话，且 iii 的分数 < jjj 的分数。
  • 那么 iii 的 aia_iai​（比 iii 高）必然 >j> j>j 的 aja_jaj​。
  • 且 iii 的 bib_ibi​（比 iii 低）必然 <j< j<j 的 bjb_jbj​。
  • 我们可以求一个 aia_iai​ 递减、bib_ibi​ 递增的最长子序列 (二维 LIS )。这可以用 O(N2)O(N^2)O(N2) DP 解决。
  • 但对于本题数据规模来说，这种算法必定 TLE。

2. 正解：区间 DP O(N+KlogK)O(N + K log K)O(N+KlogK)

我们将 “人” 转化为 “排名区间” 。

  1. 转化

    • 过滤： 如果一个人说 aia_iai​ 人比他高，bib_ibi​ 人比他低，那么 a[i]+b[i]<na[i] + b[i] < na[i]+b[i]<n 必须成立。
    • 所以如果 a[i]+b[i]>=na[i] + b[i] >= na[i]+b[i]>=n，这个人必定在说谎。直接计数。
    • 转化： 对于一个“可能”说真话的人 (ai,bi)(a_i, b_i)(ai​,bi​)：
      • a[i] 人排名在他之前，b[i] 人排名在他之后。
      • 这意味着，他自己的排名必须在 [a[i] + 1, n - b[i]] 这个闭区间内。
      • 我们定义 l = a[i] + 1，r = n - b[i]。

  2. 压缩

    • 我们有很多 (a,b)(a, b)(a,b) 相同的人，他们都要求一个 [l, r] 内的排名。
    • 所以我们对每一个 (a, b) 的区间进行查询，记作 cnt。
    • 而这个区间的“容量”是 s=n−a−bs = n - a - bs=n−a−b（即最多 sss 个人可以同分）。
    • 因此，这一组 (a, b) 最多能贡献的“真话人数”为 v = min(cnt, s)。

  3. 区间 DP

    • 经过前面两步的转换，我们得到了下面这个等价的问题：
    • 我们有 KKK ( K≤NK \le NK≤N ) 个“任务”，每个任务 iii 是一个三元组 (l, r, v)，表示：“在 [l, r] 排名区间内，我们可以安插 v 个说真话的人”。
    • 定义状态： dp[i] = 只考虑 1…i1 \dots i1…i 这些排名，能安插的“真话人数”的最大值。
    • 数据结构： ed[r] 存储所有 结束 排名为 r 的任务 (l, v)。

  4. DP 递推 O(N+K)O(N+K)O(N+K)

    • 我们遍历 iii 从 1 到 nnn (代表 iii 这个排名)：
    • 1. 不更新 dp[i] = dp[i-1]。
    • 2. 更新 for( auto& [l, v] : ed[i] )
      • (遍历所有 结束 排名为 i 的任务 (l, v))
      • dp[i] = max( dp[i], dp[l - 1] + v )
      • 我们选择执行 (l, i, v) 这个任务。它占用了 [l, i] 区间，获得了 v 的价值。它必须与 dp[l - 1] (即 lll 之前的最优解) 相连接。

    DP

  5. 计算答案

    • dp[n] = 在 1…n1 \dots n1…n 的排名中，我们能安插的最大真话人数。
    • n - dp[n] = 最少说谎人数。

    注：n 是总人数，dp[n] 是最大说真话人数。n 中也包括了步骤 1 中被过滤掉的 a+b >= n 的人，他们是必定的说谎者，n - dp[n] 自动包含了他们。

复杂度分析：

  • 时间： O(N)O(N)O(N) (读入) + O(Klog⁡K)O(K \log K)O(KlogK) (排序 KKK 个合法元素) + O(K)O(K)O(K) (压缩分组) + O(N+K)O(N+K)O(N+K) (DP 递推)。
  • 总时间： O(N+Klog⁡K)O(N + K \log K)O(N+KlogK) ( K≤NK \le NK≤N )。
  • 空间： O(N+K)O(N+K)O(N+K) (存储 ed 和 dp 数组)。

四、 结语

这题的 二维- LIS 思路很明显，但如果对其进行优化，实现起来比较麻烦。

所以我这里采用了一种问题转化思路 ( 人 →\to→ 区间 )。

从而将一个二维偏序问题，降维成了一个一维的、经典的带权区间调度问题。
