Luogu-P2829 大逃离

Luogu-P2829 大逃离

题目简述

难度：提高+ | 题解格式：V6.3

在一个 NNN 个点 MMM 条边的无向图中，求从起点 111 到终点 NNN 的严格次短路长度。
路径允许重复经过点和边。
限制：除起点和终点外，不能经过“直接连接的邻居数量（去重后）”小于 KKK 的节点。

  • 数据范围：N≤5000,M≤105,K≤NN \le 5000, M \le 10^5, K \le NN≤5000,M≤105,K≤N。

算法笔记可以参考：

  • 算法学习笔记：[图论/最短路] - Dijkstra - RainCnya

解题思路

1. 初步认知

  • 直观想法：先跑最短路，然后尝试删掉最短路上的边再跑？不对，次短路可能和最短路共享大部分边，甚至在某条边上反复横跳。
  • 瓶颈分析：单纯的搜索或删边法难以处理“重复经过”的情况。我们需要在 Dijkstra 的状态上做文章，记录更多的信息。

2. 主题渗透 (正解)

算法：拆点 Dijkstra (次短路) (O(Mlog⁡N)O(M \log N)O(MlogN))

  • 基本概念：
    传统的 Dijkstra 维护 dist[u]dist[u]dist[u] 表示到 uuu 的最短距离。为了求次短路，我们将状态扩展为二维。
    注意题目要求的限制条件：必须预处理每个点的度数（注意重边只算一次邻居），在 Dijkstra 扩展时跳过不满足度数限制的点。
  • 状态定义：
    dist[u][0]dist[u][0]dist[u][0]：起点到 uuu 的最短距离。
    dist[u][1]dist[u][1]dist[u][1]：起点到 uuu 的严格次短距离。
  • 转移逻辑：
    当通过 uuu 松弛邻居 vvv 时，新距离 d=dist[u][… ]+wd = dist[u][\dots] + wd=dist[u][…]+w：
    1. 若 d<dist[v][0]d < dist[v][0]d<dist[v][0]：发现了一条比当前最短路还短的路。
      • 关键操作：原最短路 dist[v][0]dist[v][0]dist[v][0] 沦为次短路 dist[v][1]dist[v][1]dist[v][1]。
      • 更新最短路 dist[v][0]=ddist[v][0] = ddist[v][0]=d。
    2. 若 dist[v][0]<d<dist[v][1]dist[v][0] < d < dist[v][1]dist[v][0]<d<dist[v][1]：发现了一条介于最短和次短之间的路。
      • 更新次短路 dist[v][1]=ddist[v][1] = ddist[v][1]=d。
    3. 若 d==dist[v][0]d == dist[v][0]d==dist[v][0]：题目要求严格次短，故忽略。

#include<bits/stdc++.h>using namespace std;using PII = pair<int, int>;const int maxn = 5e3 + 50;const int inf = 0x3f3f3f3f;// 使用邻接矩阵辅助去重计算度数 (N <= 5000 可行)bool mat[maxn][maxn];int deg[maxn];vector<PII> adj[maxn];int dist[maxn][2]; // [0]: 最短, [1]: 次短int n, m, k;void dijkstra( int st ){    memset( dist, 0x3f, sizeof( dist ) );    dist[1][0] = 0;    priority_queue< PII, vector<PII>, greater<PII> > pq;    pq.push({ 0, 1 });    while( !pq.empty( ) )    {        auto [d, u] = pq.top( );        pq.pop( );        // 剪枝：如果当前距离比已知的次短路还长，无意义        if( d > dist[u][1] ) continue;                for( auto [v, w] : adj[u] )        {            // 限制检查：非起终点且度数不够，跳过            if( v != 1 && v != n && deg[v] < k ) continue;                        int nd = d + w;            // 情况 1: 更新最短路            if( nd < dist[v][0] )            {                // 旧的最短路降级为次短路 (必须非 INF)                if( dist[v][0] != inf )                {                    dist[v][1] = dist[v][0];                    pq.push({ dist[v][1], v });                }                dist[v][0] = nd;                pq.push({ dist[v][0], v });            }            // 情况 2: 更新严格次短路            else if( dist[v][0] < nd && nd < dist[v][1] )            {                dist[v][1] = nd;                pq.push({ dist[v][1], v });            }        }    }   }int main( ){       ios::sync_with_stdio(0);    cin.tie(0); cout.tie(0);    cin >> n >> m >> k;    for( int i = 1; i <= m; ++ i )    {        int u, v, w;        cin >> u >> v >> w;        adj[u].push_back({ v, w });        adj[v].push_back({ u, w });                // 处理重边对度数的影响：只有第一次连接才增加度数        if( !mat[u][v] )        {            deg[u] ++, deg[v] ++;            mat[u][v] = mat[v][u] = 1;        }    }    dijkstra( 1 );    if( dist[n][1] == inf )    {        cout << -1 << '\n';    }    else    {        cout << dist[n][1] << '\n';    }    return 0;}

四、 结语

本题有两个易错点：

一是重边图的度数统计（题目定义“连接的地方”而非边数），

二是次短路更新时状态降级的逻辑（最短变次短）。

掌握 dist[u][k]dist[u][k]dist[u][k] 这种状态定义是解决 K 短路问题的基础。
