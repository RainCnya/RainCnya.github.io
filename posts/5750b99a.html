算法学习笔记：最短路图 (Shortest Path DAG)

一、 初步认知

领域：图论 (Graph Theory)
父节点：最短路 (Shortest Path)
典型设问：最短路计数、最短路上的最长链、必须经过/不经过某条边的最短路。

在求出单源最短路后，我们往往不仅仅满足于“一条”路径。最短路图是指原图中所有属于某条最短路（从 SSS 到 TTT，或从 SSS 到任意点）的边和点构成的子图。
若边权 >0>0>0，该子图必然是一个 DAG (有向无环图)。

二、 主题渗透

1. The Struggle (模拟推演)

核心判定式：
对于一条边 u→wvu \xrightarrow{w} vuw​v，若它在 S→TS \to TS→T 的最短路上，必须满足：

dist(S,u)+w+dist(v,T)=dist(S,T)dist(S, u) + w + dist(v, T) = dist(S, T) dist(S,u)+w+dist(v,T)=dist(S,T)

▶
模拟推演 (S=1, T=4)

假设图如下，求 1→41 \to 41→4 的最短路图。
Edges:
A: 1->2 (w=1)
B: 1->3 (w=2)
C: 2->4 (w=1)
D: 3->4 (w=0)
E: 1->4 (w=2)

  • dT[]d_T[]dT​[] (From 4 in rev): [2, 1, 0, 0]

Step 2: 边判定

  • Edge A (1->2, w=1): dS[1](0)+1+dT[2](1)=2==Ld_S[1](0) + 1 + d_T[2](1) = 2 == LdS​[1](0)+1+dT​[2](1)=2==L -> 保留
  • Edge B (1->3, w=2): dS[1](0)+2+dT[3](0)=2==Ld_S[1](0) + 2 + d_T[3](0) = 2 == LdS​[1](0)+2+dT​[3](0)=2==L -> 保留
  • Edge C (2->4, w=1): dS[2](1)+1+dT[4](0)=2==Ld_S[2](1) + 1 + d_T[4](0) = 2 == LdS​[2](1)+1+dT​[4](0)=2==L -> 保留
  • Edge D (3->4, w=0): dS[3](2)+0+dT[4](0)=2==Ld_S[3](2) + 0 + d_T[4](0) = 2 == LdS​[3](2)+0+dT​[4](0)=2==L -> 保留

2. Hack 思维 (Generator 视角)

  • 0 边权与环：如果图中存在 0 边权，最短路子图可能包含环（例如 u→vu \to vu→v 且 w=0w=0w=0，若 dist[u]=dist[v]dist[u]=dist[v]dist[u]=dist[v]，可能出现双向 0 边）。
    • 应对：在 DAG 上跑 DP 时，必须严格按照拓扑序，或者在判定时要求 dist[u]+w=dist[v]dist[u] + w = dist[v]dist[u]+w=dist[v] 且 w>0w > 0w>0（视题目要求是否允许 0 边环）。
  • 计数溢出：最短路计数问题中，路径条数可能指数级爆炸。
    • 应对：通常题目要求取模，或者数据范围很小。

三、 核心思想

"拓扑结构化"

将“最短路”这一数值属性，转化为 DAG 这一拓扑属性。
一旦转化为 DAG，所有在 DAG 上可用的算法（拓扑排序、DP、记忆化搜索）均可直接套用。

四、 实战部署

1. 通用模板结构

Shortest Path DAG Builder 
// 1. 预处理dijkstra(S, ds, adj);       // 正向跑dijkstra(T, dt, rev_adj);   // 反向跑 (在反图上)// 2. 隐式或显式建图// 显式建图 (需要拓扑排序时)vector<int> dag[maxn];int in_degree[maxn];for(int u = 1; u <= n; ++u) {    for(auto [v, w] : adj[u]) {        // 判定条件        if(ds[u] + w + dt[v] == ds[T]) {            dag[u].push_back(v);            in_degree[v]++;        }    }}// 3. 接下来可以在 dag 上跑拓扑排序 DP

2. 部署细节

  • 计数问题：在 DAG 上 dp[v]+=dp[u]dp[v] += dp[u]dp[v]+=dp[u]。

  • 最长链问题：在 DAG 上 dp[v]=max⁡(dp[v],dp[u]+w)dp[v] = \max(dp[v], dp[u] + w)dp[v]=max(dp[v],dp[u]+w)。

  • 点/边 重要性判定：如果某条边在所有最短路上（割边），移除它会导致最短路变长。

五、 维度概览

维度 描述
时间复杂度 O(Mlog⁡N)O(M \log N)O(MlogN) (Dijkstra) + O(N+M)O(N+M)O(N+M) (DAG 构建与 DP)。
空间复杂度 O(N+M)O(N+M)O(N+M)，通常需要反图和额外的 DAG 存储。
核心考点 计数、公共路径、网络流建图基础。
常见误区 忘记跑反向 Dijkstra；错误地认为最短路树包含所有最短路径（树只能包含部分，图才能包含所有）。
