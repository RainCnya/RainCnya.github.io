算法学习笔记：记忆化搜索

算法学习笔记：记忆化搜索

一、 初步认知

  • 命名： 记忆化搜索 (Memoization Search)

  • 领域： 搜索 / 动态规划 (DP)

  • 定义： 一种结合了深度优先搜索 (DFS) 与 动态规划 思想的算法策略。通过记录（缓存）已经计算过的子问题结果，避免在递归过程中重复计算相同的状态，从而将指数级的时间复杂度降低为多项式级。

  • 典型设问： 求“方案数”、“最大值/最小值”或“是否存在路径”，且状态之间存在大量重叠子问题（如：0/1 背包、斐波那契数列、矩阵最长路径）。

二、 主题渗透

1. 基本思路

  • 朴素 DFS： 直接按照题意进行递归模拟。每遇到一个状态就继续向下递归，不管这个状态之前是否已经计算过。

  • 状态树膨胀： 在朴素 DFS 中，许多相同的子树会被重复遍历，导致计算量呈指数爆炸（例如斐波那契数列 fib(n)=fib(n−1)+fib(n−2)fib(n) = fib(n-1) + fib(n-2)fib(n)=fib(n−1)+fib(n−2)）。

2. 优化思路

  • 引入记事本（缓存）： 创建一个数组（通常命名为 mem 或 f），其维度与状态维度一致。

  • 查表： 每次进入递归函数时，首先检查当前状态是否已经在 mem 中有记录。

    • 若有： 直接返回记录的值（剪枝）。

    • 若无： 继续向下递归计算，并在函数返回前将结果存入 mem。

3. 核心思想

  • “自顶向下 (Top-Down) 的动态规划”：保留了搜索的逻辑直观性，同时利用缓存消除了重复计算。

2. 部署细节

  • 初始化： 在 main 函数中调用 dfs 之前，必须初始化 mem 数组为一个不可能出现的特殊值（通常是 -1 或 0，视题目具体情况而定）。常用 memset(mem, -1, sizeof(mem));。

  • 状态定义： 确保 mem 数组的维度能完全覆盖所有可能的状态组合。如果状态过于稀疏或维度太高（例如字符串或复杂状压），可以考虑用 std::map<State, int> 代替数组，虽然会增加一个 logloglog 的时间常数。

  • 引用传递： 如果状态是比较大的结构（不是本例中的 int），记得在函数参数中使用引用传递以减少拷贝开销。

  • 1-based 索引： 循环或递归边界通常设为 i > n 时终止。
