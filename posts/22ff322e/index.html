题意

给定一个 01 串 sss 和一个整数 kkk，每次操作可以指定 iii，然后将 iii 和 (i+k)(modn)(i+k) \pmod n(i+k)(modn) 两个位置的值反转（0→1,1→00 \to 1, 1 \to 00→1,1→0）。询问是否能通过若干次操作使得这个串变成一个回文串。

1003 Livehouse

题意

给定 nnn 天，安排 mmm 场演唱会，任意两场在区间天数上要求不能相互包含。求在所有的非空合法的区间集合中，需要的乐队数目不超过 kkk 的集合总数。

n≤500,∑n≤2000n \leq 500, \sum n \leq 2000n≤500,∑n≤2000

思路

这题 ...... 我乍一眼看完没思路，原题的题面更难理解，这看似是一个区间覆盖问题，但关键在于这个 “两两没有包含关系” 的约束中。

不包含意味着什么？假设现在有两个区间 [li,ri][l_{i}, r_{i}][li​,ri​] 和 [lj,rj][l_{j},r_{j}][lj​,rj​]，如果满足不包含关系，且固定 li<ljl_{i} < l_{j}li​<lj​，那么一定有 ri<rjr_{i} < r_{j}ri​<rj​。这意味着，如果把所有区间按左端点 lll 排序，它的右端点 rrr 也一定是 严格单调递增 的。

由此我们得到了两个性质：1. 先开始的区间必须先结束 ( 类队列 )。2. 覆盖某一点的最大区间数就是所需的最小乐队数。

既然区间是按顺序开始，按顺序结束的，不妨按照时间顺序来遍历，每天只有下面四种操作：

  1. 开始一个新区间 ( 左端点 lll 定在今天 )。
  2. 结束一个旧区间 ( 右端点 rrr 定在今天 )。
  3. 开始并结束 ( 今天产生了一个 [i,i][i, i][i,i] 的长度为 1 的区间 )。
  4. 摆烂，什么都不做。

计数类问题，多阶段，自然想到用 DP 来计数。

定义状态 dp[i][j]dp[i][j]dp[i][j] 表示当前处理到第 iii 天，有 jjj 个已经开始但未结束的区间数量。

由于第二个性质，jjj 的最大值就是所需的乐队数，题目要求乐队数 ≤k\leq k≤k，所以在转移过程中，只要保证 j≤kj \leq kj≤k 即可。

这题看起来像是一个 动态二维字符串匹配的问题，加上 n,q=1e4n, q = 1e4n,q=1e4 的规模，很容易会想到用 KMP，AC 自动机，甚至更复杂的数据结构上考虑，于是赛场上的我想到这，投降了喵。

但是，但是 m≤20m \leq 20m≤20，也就是说每一行的字符串长度最大只有 20，那它的子串最多有多少个呢？答案是 (20×21)/2=210(20 \times 21) / 2 = 210(20×21)/2=210 个，既然子串可能性就这么多，为什么不直接暴力枚举呢？

于是就想到了用字符串哈希表来维护这个匹配问题。接着考虑哈希表映射构造，对于一个绝对对齐的两个串，它们的 内容 和 列位置 一定是一样的，所以我们还需要把 位置 也一起写到哈希表里面。

这里用 unordered_map<ull, int> cnt 来存，unordered_map 本身就是基于哈希表，ull 是 64 位无符号，这里用作自然溢出。( 如果看不懂，移步字符串哈希 )

然后是算法的具体实现步骤，首先我们需要预处理每一行的子串，然后都存起来。

接着是操作 1 修改：由于子串很少，所以我们暴力把移动前的至多 210 个子串都删掉，更新这一行移动之后，再把至多 210 个子串加进去。

最后是操作 2 查询：枚举当前第 xxx 行的所有子串，如果某个子串的 Key 在 cnt 中出现的次数 ≥2\geq 2≥2，就说明存在一个与之对应，说明匹配成功。
