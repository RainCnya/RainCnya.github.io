Luogu-P5664 / TZOJ-6281Emiya 家今天的饭

题目简述

难度：提高+(38832) | 题解格式：V6.2

Emiya 需要用 nnn 种烹饪方法和 mmm 种主要食材做菜。要求：

  1. 至少做一道菜。
  2. 每道菜的烹饪方法互不相同。
  3. 每种主要食材出现的次数至多为总菜数的一半（即 ≤⌊k/2⌋\le \lfloor k/2 \rfloor≤⌊k/2⌋）。

求满足条件的方案数，对 998244353998244353998244353 取模。

  • 数据范围：n≤100n \le 100n≤100，m≤2000m \le 2000m≤2000，ai,j<998244353a_{i,j} < 998244353ai,j​<998244353。

解题思路

1. 初步思考

  • 直观想法：直接 DP，记录每种食材出现的次数。但 mmm 高达 200020002000，状态无法压缩，直接记录所有食材次数不可行。
  • 瓶颈分析：限制条件“每种食材不过半”非常麻烦，因为食材之间会相互制约。

2. 核心转化 (正解)

算法：计数 DP + 容斥原理 (O(n2⋅m)O(n^2 \cdot m)O(n2⋅m))

  • 大概思路：
    正难则反，计算总方案数容易，计算不合法的方案数也比较容易。
    所以大胆采用容斥原理，计算总方案数，再减去不合法的方案数，就可以得到答案了。

    答案=总方案数−∑不合法方案数(col)\text{答案} = \text{总方案数} - \sum \text{不合法方案数}(\text{col}) 答案=总方案数−∑不合法方案数(col)

  • 状态定义：
    既然题目中出现了这么多状态，那就不得不用 DP 了。
    我们需要枚举每一种食材 colcolcol 作为那个“违规食材”。

    注意到限制条件：每种主要食材的出现次数至多为总菜数的一半。
    所以定义 dp[i][j]dp[i][j]dp[i][j]：前 iii 行中，(违规食材 colcolcol 的数量) - (其他食材的数量) = jjj 的方案数。
    为了处理负数下标，实际存储时 jjj 需要加上偏移量 nnn。

  • 转移逻辑：
    对于第 iii 行，有三种选择：

    1. 不做菜：差值 jjj 不变。方案数 111。
    2. 做违规食材 colcolcol：差值 j+1j+1j+1。方案数 ai,cola_{i,col}ai,col​。
    3. 做其他食材：差值 j−1j-1j−1。方案数 Si−ai,colS_i - a_{i,col}Si​−ai,col​ （SiS_iSi​ 为第 iii 行的总方案数）。

    最后统计 j>nj > nj>n （即差值 >0>0>0）的状态和，即为非法方案数。

#include<bits/stdc++.h>using namespace std;const int mod = 998244353;const int maxn = 105;const int maxm = 2005;int a[maxn][maxm];long long sum[maxn];         // 每一行的总方案数long long dp[maxn][maxn<<1]; // dp[i][j] j带偏移量 nint n, m;int main( ){     ios::sync_with_stdio(0);    cin.tie(0); cout.tie(0);    cin >> n >> m;    for( int i = 1; i <= n; ++ i )    {        for( int j = 1; j <= m; ++ j )        {            cin >> a[i][j];            sum[i] = ( sum[i] + a[i][j] ) % mod;        }    }	// 计算总方案数量。    long long total = 1;    for( int i = 1; i <= n; ++ i )    {        total = ( total * ( sum[i] + 1 ) ) % mod;    }        // 减去全都不做的情况    total = ( total - 1 + mod ) % mod;	// 计算非法方案数    long long res = 0;        // 枚举每一列作为违规食材    for( int c = 1; c <= m; ++ c )    {        memset( dp, 0, sizeof( dp ) );        dp[0][n] = 1;         // 初始状态：差值为0，坐标偏移 n                for( int i = 1; i <= n; ++ i )        {            for( int j = n - i; j <= n + i; ++ j )            {                // 情况1: 第 i 行不做菜 -> 差值不变                dp[i][j] = dp[i-1][j];                // 情况2: 做食材 c -> 差值 +1                if( j - 1 >= 0 )                    dp[i][j] = ( dp[i][j] + dp[i-1][j-1] * a[i][c] ) % mod;                                // 情况3: 做其他食材 -> 差值 -1                // 方案数为 (行总数 - 当前食材数)                if( j + 1 <= 2 * n )                    dp[i][j] = ( dp[i][j] + dp[i-1][j+1] * ( sum[i] - a[i][c] + mod ) % mod ) % mod;            }        }        // 统计当前列导致的非法方案        for( int j = n + 1; j <= 2 * n; ++ j )        {            res = ( res + dp[n][j] ) % mod;        }    }    long long ans = ( total - res + mod ) % mod;    cout << ans << '\n';    return 0;}

四、 结语

本题的核心在于利用容斥原理。

将复杂的全局限制（所有食材 ≤k/2\le k/2≤k/2）转化为单调的局部限制（即存在一个食材 >k/2> k/2>k/2）。

具体代码实现中，如果遇到负数状态，可以添加偏移量 nnn ，来进行处理。

注：这是一个通用技巧。
