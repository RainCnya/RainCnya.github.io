算法学习路线图：图论体系

算法学习路线图：图论体系

一、 初步认知

  • 命名： 图论 (Graph Theory)
  • 领域： 算法与数据结构 / 核心模型
  • 定义： 研究 点 (Vertex) 与 边 (Edge) 之间关系的数学模型。
  • 核心地位： 算法竞赛的核心，是解决所有关系问题的数学模型。它将现实世界的连接、依赖、路径问题抽象化，是解决复杂系统问题的基础语言。

二、 主题渗透

1. 核心原理

  • 底层逻辑： 抽象 (Abstraction) + 遍历 (Traversal)。
  • 基本元素： G=(V,E)G = (V, E)G=(V,E)。
    • VVV (Vertex)： 实体（人、地、任务）。
    • EEE (Edge)： 关系（友谊、道路、依赖）。
  • 核心操作： 所有的图论算法本质上都在做两件事：
    1. 如何走？ (DFS, BFS, 拓扑序)
    2. 维护什么？ (距离 dist, 连通性 fa, 流量 flow)

2. 体系分支

  • 基础篇：
    • 存图： 邻接矩阵 vs 邻接表（链式前向星）。
    • 遍历： DFS (深度优先), BFS (广度优先)。
  • 核心模型：
    • 最短路 (Shortest Path)： Dijkstra, SPFA, Floyd。
    • 生成树 (MST)： Kruskal, Prim。
    • 连通性 (Connectivity)： 并查集, Tarjan (强连通/双连通分量)。
  • 树论专题：
    • LCA (最近公共祖先)： 倍增, Tarjan。
    • 树上问题： 树形 DP, 树链剖分, 树的直径/重心。
  • 进阶篇：
    • 拓扑排序： 解决依赖关系。
    • 二分图： 染色判定, 最大匹配 (匈牙利)。
    • 网络流： 最大流 (Dinic), 最小割, 费用流。

3. 核心思想

  • 将“关系”具象化为“边”，通过特定的遍历顺序，计算出点与点之间的性质（如距离、连通性）。

4. 演示 (图论解题流)

▶
图论解题三步走
  1. 建模：
    • 谁是点？谁是边？边有方向吗？有权值吗？
  2. 选型：
    • 求最短距离？ →\to→ Dijkstra / BFS
    • 求连通块？ →\to→ DFS / 并查集
    • 求依赖顺序？ →\to→ 拓扑排序
  3. 实现： * 建图 (邻接表) →\to→ 跑算法 →\to→ 处理边界。

2. 部署细节

  • 空间开销：
    • head 数组大小为 点数 NNN。
    • edge, next_edge, val 数组大小为 边数 MMM。
    • 易错点： 无向图的双向边需要开 2M2M2M 的空间！
  • 初始化： head 数组通常初始化为 -1（或 0，取决于 idx 从 0 还是 1 开始）。
  • 重边与自环： 邻接表天然支持重边和自环，但在某些算法（如最短路）中可能需要特殊处理。

四、 维度概览

维度 复杂度 说明
存图空间 O(N+M)O(N+M)O(N+M) 邻接表是最通用的方式。
遍历时间 O(N+M)O(N+M)O(N+M) 每个点和每条边访问一次。
特性 稀疏图友好 适合绝大多数竞赛题目 (M≪N2M \ll N^2M≪N2)。
