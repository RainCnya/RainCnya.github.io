算法学习路线图：图论 (Graph Theory Roadmap)

算法学习路线图：图论 (Graph Theory Roadmap) -V3.2

一、 初步认知 (Macro View)

1. 叫什么 (Identity)

  • 中文名： 图论
  • 英文名： Graph Theory

2. 什么地位 (Status)

  • 所属领域： 算法与数据结构
  • 核心地位： 算法竞赛的绝对核心，是所有 “关系” 问题的数学模型。它将现实世界的连接、依赖、路径问题抽象为“点”和“边”，是解决复杂系统问题的基础语言。

3. 有什么用处 (Utility)

  • 解决的核心问题： 解决连通性、路径、环路、依赖、匹配和网络流等问题。
  • 典型应用场景：
    • 社交网络（好友关系）。
    • 地图导航（最短路径）。
    • 任务调度（拓扑排序）。
    • 网络路由（最大流量）。

二、 主题渗透 (Theme Osmosis)

1. 核心原理 (The Eureka Moment)

  • 底层思想： 抽象 + 遍历。
  • 关键推导：
    • 万物皆图： 现实世界中的“实体”可以抽象为节点 (Vertex)，“实体”之间的“关系”可以抽象为边 (Edge)。
    • G = (V, E)： 一个图 (G) 由一个点集 (V) 和一个边集 (E) 定义。
    • 核心操作： 图论算法的本质，几乎都是围绕着**“如何遍历这张图”（如 DFS, BFS）以及“遍历时维护什么信息”**（如 dist[] 数组）来展开的。

2. 模拟演示 (Visualization)

阶段一：基础与遍历 (The Basics & Traversal)

  • 目标： 理解图的本质，学会如何存储和搜索图。
  • 1. 图的存储 (Storage)：
    • 邻接矩阵 (Adjacency Matrix)： O(V2)O(V^2)O(V2) 空间。适用于稠密图。
    • 邻接表 (Adjacency List)： O(V+E)O(V+E)O(V+E) 空间。最常用。
  • 2. 图的遍历 (Traversal)：
    • 深度优先搜索 (DFS)： 探索“一条路走到底”。
    • 广度优先搜索 (BFS)： 探索“一层一层向外扩”。

阶段二：经典核心模型 (Classic Models)

  • 目标： 解决图论中最常见的两类问题：最短路径和最小生成树。
  • 1. 最短路径 (Shortest Path)：
    • 单源 (无负权)： Dijkstra 算法（朴素 O(V2)O(V^2)O(V2) / 堆优化 O(Elog⁡V)O(E \log V)O(ElogV)）。
    • 单源 (有负权)： Bellman-Ford 算法 (O(VE)O(VE)O(VE)), SPFA 算法（通常 O(kE)O(kE)O(kE)，最坏 O(VE)O(VE)O(VE)）。
    • 多源 (任意权)： Floyd-Warshall 算法 (O(V3)O(V^3)O(V3))。
  • 2. 最小生成树 (MST)：
    • Kruskal 算法： O(Elog⁡E)O(E \log E)O(ElogE)（基于边贪心）。
    • Prim 算法： O(V2)O(V^2)O(V2) 或 O(Elog⁡V)O(E \log V)O(ElogV)（基于点贪心）。

阶段三：树论专题 (Tree Theory)

  • 目标： 深入研究“树”这一特殊的无环连通图。
  • 1. 基础应用：
    • 并查集 (Union-Find)： 维护连通性（Kruskal 的基础）。
    • 最近公共祖先 (LCA)： 倍增法 / Tarjan (离线)。
  • 2. 进阶应用：
    • 树上 DP、树的直径/重心、树链剖分。

阶段四：高级应用与专题 (Advanced Topics)

  • 目标： 解决更复杂的图模型。
  • 1. 连通性与依赖：
    • 拓扑排序 (Topological Sort)： 解决有向无环图 (DAG) 的依赖/顺序问题。
    • 欧拉路径 / 欧拉回路： 一笔画问题。
    • 强连通分量 (SCC)： Tarjan 算法（缩点）。
  • 2. 匹配与流：
    • 二分图 (Bipartite Graph)： 判定（染色法）、最大匹配（匈牙利算法 / 网络流）。
    • 网络流 (Network Flow)： 最大流 / 最小割 (Dinic, EK)、费用流。

3. 详细实现过程 (Step-by-Step)

这是解决任何图论问题的通用思考流程：

  1. 建模 (Modeling)：
    • 明确：什么是点 (Vertex)？ （例如：城市、人、任务）
    • 明确：什么是边 (Edge)？ （例如：道路、关系、依赖）
    • 明确：图是有向/无向？带权/无权？
  2. 存储 (Storage)：
    • 根据点数 VVV 和边数 EEE 选择存储方式。
    • 99% 的情况选择邻接表（使用 vector 或手写链表）。
  3. 算法 (Algorithm Selection)：
    • 根据问题选择合适的算法模型（例如：求最短路 →\to→ Dijkstra；求连通性 →\to→ DFS/BFS/并查集；求依赖 →\to→ 拓扑排序）。
  4. 实现 (Implementation)：
    • 运行所选算法，注意边界、重边、自环等问题。

三、 总结分析 (Analysis & Comparison)

1. 复杂度分析 (Complexity)

维度 复杂度 (V=点数, E=边数) 说明
遍历 (邻接表) O(V+E)O(V+E)O(V+E) DFS / BFS / 拓扑排序 / Tarjan
最短路 (Dijkstra) O(Elog⁡V)O(E \log V)O(ElogV) 或 O(V2)O(V^2)O(V2) 堆优化 O(Elog⁡V)O(E \log V)O(ElogV)，朴素 O(V2)O(V^2)O(V2)
最短路 (Floyd) O(V3)O(V^3)O(V3) 多源最短路
最小生成树 O(Elog⁡E)O(E \log E)O(ElogE) 或 O(Elog⁡V)O(E \log V)O(ElogV) Kruskal ( Elog⁡EE \log EElogE ) 或 Prim ( Elog⁡VE \log VElogV )

2. 核心注意点 (Key Pitfalls)

  1. 点数 (V) vs 边数 (E)： 估算复杂度时，maxn 到底该设为点数还是边数？邻接表（链式前向星）的 head 数组大小是 VVV，edge 数组和 idx 大小是 EEE。
  2. 1-based vs 0-based： 竞赛中常用 1-based 索引，for(i=1; i<=n; ++i)，确保 head 数组等初始化正确。
  3. 重边和自环： 在某些问题中（如 Dijkstra），重边需要取权值最小的；自环通常需要忽略。
  4. 有向 vs 无向： 如果是无向图，使用邻接表存边时，必须 add(u, v) 和 add(v, u) 两次。
  5. 初始化： memset 初始化 dist 数组为 0x3f（无穷大）时，注意检查 long long 溢出。
