算法学习笔记：0/1 背包

一、 初步认知

  • 命名： 0/1 背包 (0/1 Knapsack Problem)

  • 领域： 动态规划 (DP) / 背包模型

  • 定义： 给定 NNN 件物品和一个容量为 VVV 的背包。每件物品只有 1 件，可以选择放或不放。

  • 典型设问： “给定背包容量 VVV，物品 iii 的重量为 w[i]w[i]w[i]，价值为 v[i]v[i]v[i]，求在不超过容量的前提下能获得的最大价值。”

二、 主题渗透

1. 基本思路

  • 状态定义： dp[i][j] 表示从前 iii 个物品中选，总体积不超过 jjj 时的最大价值。

  • 决策： 对于第 iii 个物品，只有两种选择：

    • 不选： 价值等于前 i−1i-1i−1 个物品在容量 jjj 下的价值 →dp[i−1][j]\to dp[i-1][j]→dp[i−1][j]。

    • 选： 价值等于前 i−1i-1i−1 个物品在容量 j−w[i]j-w[i]j−w[i] 下的价值 + 当前物品价值 v[i]v[i]v[i] →\to→ dp[i−1][j−w[i]]+v[i]dp[i-1][j-w[i]] + v[i]dp[i−1][j−w[i]]+v[i]。

  • 转移方程： dp[i][j]=max⁡(dp[i−1][j], dp[i−1][j−w[i]]+v[i])dp[i][j] = \max(dp[i-1][j], \ dp[i-1][j-w[i]] + v[i])dp[i][j]=max(dp[i−1][j], dp[i−1][j−w[i]]+v[i])

2. 优化思路

  • 空间压缩 (滚动数组 Rolling Array)：

    • 观察发现，dp[i][...] 只依赖于 dp[i-1][...]（上一层）。

    • 我们可以去掉第一维，只用 dp[j] 来保存状态。

    • 为了保证计算 dp[j] 时，用到的 dp[j-w[i]] 是上一层（即还没放入物品 iii）的数据，内层循环必须从大到小遍历。

3. 核心思想

  • 利用“倒序循环”来消除后效性，确保每个物品在每个容量下至多被选择一次。

4. 演示

{% fold info @模拟 (1D 数组倒序更新) %}

  • 物品： w=2,v=5w=2, v=5w=2,v=5。 当前状态 dp： [0, 0, 3, 3, 6, 6] (容量 0~5)

  • 更新过程 (倒序)：

    • j=5: dp[5] = max(dp[5], dp[5-2] + 5) = max(6, 3+5) = 8

    • j=4: dp[4] = max(dp[4], dp[4-2] + 5) = max(6, 3+5) = 8

    • j=3: dp[3] = max(dp[3], dp[3-2] + 5) = max(3, 0+5) = 5

    • j=2: dp[2] = max(dp[2], dp[2-2] + 5) = max(3, 0+5) = 5

  • 结果： [0, 0, 5, 5, 8, 8]。 (正确：物品只被加了一次)

    {% endfold %}

三、 实战部署

1. 代码模板

假设 w[i] 和 v[i] 分别表示第 iii 个物品的 重量 和 价值。

{% fold info @0/1 背包 2D %}


for( int i = 1; i <= n; ++ i )
{
	for( int j = m; j >= w[i]; -- j )
	{
		dp[i][j] = max( dp[i-1][j], dp[i-1][j - w[i] ] + v[i] );
	}
}

{% endfold %}

{% fold info @0/1 背包 1D %}

for(int i = 1; i <= n; ++i) 
{
    for(int j = m; j >= w[i]; --j) 
    {
        dp[j] = max(dp[j], dp[j - w[i]] + v[i]);
    }
}

{% endfold %}

2. 部署细节

  • 循环方向： 内层循环 j 必须从 mmm 到 w[i]w[i]w[i] 倒序。如果正序，会变成完全背包（物品被重复放置）。

  • 循环下界： j >= w[i]。因为当背包容量小于当前物品重量时，装不下，状态继承 dp[j] 不变，无需遍历。

  • 初始化：

    • 求“最大价值”（不强求装满）：memset(dp, 0)。

    • 求“恰好装满”的最大价值：dp[0] = 0，其余 dp[1..m] = -INF。
