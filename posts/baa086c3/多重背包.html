算法学习笔记：多重背包

一、 算法图谱

  • 命名： 多重背包 (Bounded Knapsack Problem)
  • 领域： 动态规划 (DP) / 背包模型
  • 定义： 给定 NNN 种物品和一个容量为 VVV 的背包。第 iii 种物品最多有 m[i]m[i]m[i] 件可用。
  • 典型设问： “给定背包容量 VVV，物品 iii 的重量 w[i]w[i]w[i]，价值 v[i]v[i]v[i]，数量限制 m[i]m[i]m[i]，求最大价值。”

二、 核心推演

1. 基本思路

  • 策略是取 kkk 件物品 iii，其中 0≤k≤m[i]0 \le k \le m[i]0≤k≤m[i]。
  • 朴素方程：

    dp[j]=max⁡0≤k≤m[i](dp[j−k⋅w[i]]+k⋅v[i])dp[j] = \max_{0 \le k \le m[i]} (dp[j - k \cdot w[i]] + k \cdot v[i]) dp[j]=0≤k≤m[i]max​(dp[j−k⋅w[i]]+k⋅v[i])

  • 瓶颈： 等同于三重循环，复杂度 O(V⋅∑m[i])O(V \cdot \sum m[i])O(V⋅∑m[i])，当 m[i]m[i]m[i] 较大时极易超时。

2. 优化思路一：二进制拆分 (Binary Splitting)

  • 原理： 任何正整数 SSS 都可以分解为 1,2,4,…,2p1, 2, 4, \dots, 2^p1,2,4,…,2p 及余数 RRR 的和。
  • 转化： 将第 iii 种物品的 m[i]m[i]m[i] 件拆分成 log⁡m[i]\log m[i]logm[i] 个新的 0/1 物品。
  • 效果： 复杂度降为 O(V⋅∑log⁡m[i])O(V \cdot \sum \log m[i])O(V⋅∑logm[i])。这是竞赛中最常用的解法。

3. 优化思路二：单调队列 (Monotonic Queue)

  • 原理： 观察转移方程，发现 dp[j]dp[j]dp[j] 的更新只依赖于 j,j−w,j−2w…j, j-w, j-2w \dotsj,j−w,j−2w… 这些模 www 同余的位置。
  • 转化： 将状态按 j mod wj \bmod wjmodw 分组。对于每一组，转移方程实际上是在求一个**“固定长度滑动窗口”**内的最大值（需减去偏移量 t⋅vt \cdot vt⋅v）。
  • 效果： 复杂度降为 O(N⋅V)O(N \cdot V)O(N⋅V)，与物品数量 m[i]m[i]m[i] 无关，是理论最优解。

三、 实战部署

1. 代码模板 (二进制优化 - 推荐)

{% fold info @多重背包 (二进制优化) %}

// 1. 拆分过程
vector<int> new_w, new_v;
for(int i = 1; i <= n; ++i)
{
    int count = m[i];
    for(int k = 1; k <= count; k <<= 1)
    {
        new_w.push_back(k * w[i]);
        new_v.push_back(k * v[i]);
        count -= k;
    }
    if (count > 0) // 处理余数
    {
        new_w.push_back(count * w[i]);
        new_v.push_back(count * v[i]);
    }
}

// 2. 0/1 背包过程 (倒序)
for(int i = 0; i < new_w.size(); ++i)
{
    for(int j = m; j >= new_w[i]; --j)
    {
        dp[j] = max(dp[j], dp[j - new_w[i]] + new_v[i]);
    }
}

{% endfold %}

2. 代码模板 (单调队列优化 - 进阶)

{% fold info @多重背包 (单调队列优化) %}

// dp[j]: 当前容量 j 下的最大价值
// g[j]:  上一轮 dp 数组的备份
// q[]:   单调队列，存储的是拆分后的份数下标 p
int dp[MAXV], g[MAXV], q[MAXV];
int n, m; // n: 物品数量, m: 背包容量

for( int i = 1; i <= n; ++ i )
{
	int w, v, s;
	cin >> w >> v >> s; // 读入重量、价值、数量限制

	// 1. 备份 DP 数组
	// 必须用 g 数组保存上一层状态
	memcpy(g, dp, sizeof(dp));

	// 2. 枚举余数 u (将 dp 数组分为 w 组)
	// 每一组的索引为: u, u+w, u+2w, ...
	for( int u = 0; u < w; ++ u )
	{
		int head = 0, tail = -1;

		// 对应公式：g[u + k*w] - k*v
		auto val = [&]( int k ) {
			return g[u + k * w] - k * v;
		};

		// 3. 枚举份数 p (滑动窗口推进)
		// 实际容量 j = u + p * w
		for( int p = 0; u + p * w <= m; ++ p )
		{
			// A. 维护单调递减队列
			while( head <= tail && val(q[tail]) <= val(p) )
				tail--;
			q[++tail] = p;

			if( q[head] < p - s )
				head++;

			// B. 更新 DP 值
			// 当前最大价值 = 最优净值 + 当前份数补偿 (p * v)
			// 对应公式：dp[j] = (g[best_j] - best_k * v) + p * v
			dp[u + p * w] = val( q[head] ) + p * v;
		}
	}
}

{% endfold %}

3. 部署细节

  • 方案选择： 一般情况下（m[i]≤1000m[i] \le 1000m[i]≤1000），二进制优化够了，代码简单不易错。

    • 只有在 m[i]m[i]m[i] 极大（如 10510^5105）且 VVV 较小（如 4⋅1044 \cdot 10^44⋅104）时，才需使用单调队列。

  • 单调队列细节：

    • 必须备份 dp 数组（使用 memcpy），因为滑动窗口是从左向右更新，会覆盖需要用到的“上一轮”数据。
    • 队列中存的是下标 p（份数），但比较的是 val - p*v。
