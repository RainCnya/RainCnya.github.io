算法学习笔记：完全背包

一、 初步认知

  • 命名： 完全背包 (Unbounded Knapsack Problem)

  • 领域： 动态规划 (DP) / 背包模型

  • 定义： 给定 NNN 种物品和一个容量为 VVV 的背包。每种物品有无限件可用。

  • 典型设问： “每种物品都有无限件，求在不超过背包容量 VVV 的前提下，能获得的最大价值。”

二、 主题渗透

1. 基本思路

  • 这个问题非常类似于 0/1 背包，所不同的是每种物品都有无限件可用。所以相对应的，它的状态转移不止 "取" 和 “不取”。

  • 而是有 取 0 件，取 1 件，取 2 件 …… 取 kkk 件（其中 k⋅w[i]≤jk \cdot w[i] \le jk⋅w[i]≤j）。

  • 按照 0/1 背包的思想，很容易能得到以下朴素状态转移方程：

    dp[i][j]=max⁡0≤k⋅w[i]≤j(dp[i−1][j−k⋅w[i]]+k⋅v[i])dp[i][j] = \max_{0 \le k \cdot w[i] \le j}( dp[i-1][j - k \cdot w[i]] + k \cdot v[i] ) dp[i][j]=0≤k⋅w[i]≤jmax​(dp[i−1][j−k⋅w[i]]+k⋅v[i])

  • 这和 0/1 背包一样有 O(NV)O(NV)O(NV) 个状态需要处理，但求解每个状态 dp[i][j] 的时间取决于 kkk 的枚举次数 (O(V/w[i])O(V/w[i])O(V/w[i]))，所以总复杂度为 O(V⋅∑Vw[i])O(V \cdot \sum \frac{V}{w[i]})O(V⋅∑w[i]V​)。

  • 这个复杂度较高，等同于将其转化为多重背包求解。

2. 优化思路

  • 数学推导 (O(1)O(1)O(1) 转移)：

    • 观察 dp[i][j]dp[i][j]dp[i][j] 的展开式：

      dp[i][j]=max⁡(dp[i−1][j],dp[i−1][j−w]+v,dp[i−1][j−2w]+2v,… )dp[i][j] = \max(dp[i-1][j], dp[i-1][j-w]+v, dp[i-1][j-2w]+2v, \dots) dp[i][j]=max(dp[i−1][j],dp[i−1][j−w]+v,dp[i−1][j−2w]+2v,…)

    • 观察 dp[i][j−w]dp[i][j-w]dp[i][j−w] 的展开式：

      dp[i][j−w]=max⁡(dp[i−1][j−w],  dp[i−1][j−2w]+v,… )dp[i][j-w] = \max( dp[i-1][j-w], \ \ dp[i-1][j-2w]+v, \dots) dp[i][j−w]=max(dp[i−1][j−w],  dp[i−1][j−2w]+v,…)

    • 显然发现： dp[i][j]dp[i][j]dp[i][j] 的后半部分（除了第一项），恰好等于 dp[i][j−w]+vdp[i][j-w] + vdp[i][j−w]+v。

    • 结论： 我们不需要枚举 kkk，只需利用当前层 ( iii ) 刚才计算出的结果。

    • 最终方程： dp[i][j]=max⁡(dp[i−1][j], dp[i][j−w[i]]+v[i])dp[i][j] = \max(dp[i-1][j], \ dp[i][j-w[i]] + v[i])dp[i][j]=max(dp[i−1][j], dp[i][j−w[i]]+v[i])。

3. 核心思想

  • “正序遍历” 的可行性在于它可以用当前层已更新状态转移，正好就表示可以多次选取物品。

4. 演示

{% fold info @模拟 %}

  • 物品： w=2,v=5w=2, v=5w=2,v=5。 状态 dp： 全 0。

  • 正序更新 (jjj 从 2 到 4)：

    • j=2: dp[2] = max(dp[2], dp[2-2] + 5) = 5 (取1个)

    • j=3: dp[3] = max(dp[3], dp[3-2] + 5) = 5

    • j=4: dp[4] = max(dp[4], dp[4-2] + 5)

      • 此时 dp[4-2] 即 dp[2] 已经是 5。

      • dp[4] = 5 + 5 = 10 (取2个)

  • 结果： 通过正序，我们在计算 dp[4] 时自动利用了 dp[2] (已选1个) 的结果，实现了“多选”。

{% endfold %}

三、 实战部署

1. 代码模板 (朴素解法)

{% fold info @完全背包1 %}

// 朴素思路：枚举 k (类似于多重背包)
for( int i = 1; i <= n; ++ i ) 
{
    for( int j = m; j >= 0; -- j ) // 倒序
    {
        for( int k = 0; k * w[i] <= j; ++ k )
        {
            dp[j] = max(dp[j], dp[j - k * w[i]] + k * v[i]);
        }
    }
}

{% endfold %}

2. 代码模板 (正序优化)

{% fold info @完全背包2 %}

for(int i = 1; i <= n; ++i) 
{
    for(int j = w[i]; j <= m; ++j) 
    {
        dp[j] = max(dp[j], dp[j - w[i]] + v[i]);
    }
}

{% endfold %}

3. 部署细节

  • 正序优化：

    • 循环方向： 内层循环 j 必须从 w[i]w[i]w[i] 到 mmm 正序。

    • 物理意义： dp[j-w[i]] 是当前层 (iii) 已经考虑过放入物品 iii 的状态。再加一个 v[i]v[i]v[i]，就意味着“再多拿一个”。

    • 对比： 0/1 背包倒序是为了避开本层状态；完全背包正序是为了利用本层状态。

4. 变体：可行性背包

当题目不要求最大价值，只要求“能否凑出容量 VVV”时（例如货币系统），完全背包的思想依然适用。

  • 差异： dp 数组存储 bool 值（可行性）。
  • 优化： 往往结合排序和贪心，去除可被小面额凑出的冗余物品。
  • 题解：Luogu-P5020 货币系统
