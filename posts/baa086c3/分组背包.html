算法学习笔记：分组背包

一、 初步认知

  • 命名： 分组背包 (Grouped Knapsack Problem)

  • 领域： 动态规划 (DP) / 背包模型

  • 定义： 给定 NNN 组物品和一个容量为 VVV 的背包。每组包含若干物品，同一组内的物品互斥（最多只能选 1 个）。

  • 典型设问： “有 KKK 组物品，每组有若干个，求在不超过容量前提下，从每组至多选一个物品能获得的最大价值。”

二、 主题渗透

1. 基本思路

  • 这是一个 0/1 背包的变种。

  • 我们可以把“每一组”看作一个“广义的物品”。

  • 对于第 kkk 组，决策从简单的“选/不选”变成了 “不选，或者选组内的某一个”。

  • 状态转移：

    dp[j]=max⁡(dp[j], dp[j−w[i1]]+v[i1], dp[j−w[i2]]+v[i2],… )dp[j] = \max(dp[j], \ dp[j - w[i_1]] + v[i_1], \ dp[j - w[i_2]] + v[i_2], \dots) dp[j]=max(dp[j], dp[j−w[i1​]]+v[i1​], dp[j−w[i2​]]+v[i2​],…)

    (其中 i1,i2…i_1, i_2 \dotsi1​,i2​… 是第 kkk 组内的物品)

2. 优化思路

  • 空间压缩与循环顺序：

    • 依然可以使用一维数组 dp[j]。

    • 关键难点： 如何保证在考虑组内物品 B 时，用到的状态不是刚刚选了组内物品 A 的状态？（否则就变成了每组选多个）。

    • 解决方案： 调整循环层级。

      1. 最外层： 枚举 组 ( kkk ) （阶段）。

      2. 中间层： 枚举 容量 ( jjj )，必须 倒序 （保证引用的是上一组的状态）。

      3. 最内层： 枚举 组内物品 ( iii ) （决策）。

3. 核心思想

  • 将容量循环设置在物品循环外，结合倒序更新，可以保证同一组物品只能竞争上一个状态，从而达成互斥。

三、 实战部署

1. 代码模板

{% fold info @分组背包 %}

// 假设 group[k] 存储了第 k 组所有物品的索引
for( int k = 1; k <= total_groups; ++ k )
{
    for( int j = m; j >= 0; -- j )
    {
        for( int i : group[k] )
        {
            if( j >= w[i] )
            {
                // 尝试该组内的所有选项，取最优
                dp[j] = max(dp[j], dp[j - w[i]] + v[i]);
            }
        }
    }
}
// 最终答案 dp[m]

{% endfold %}

2. 部署细节

  • 三层循环顺序 (绝对关键)：

    • 正确： 组 →\to→ 容量 →\to→ 组内物品。

    • 错误： 组 →\to→ 组内物品 →\to→ 容量。（这会让组内物品变成 0/1 背包叠加，即每组可选多个）。

  • 初始化： 与 0/1 背包一致。

  • 数据存储： 通常用 vector<int> group[maxn] 或邻接表来存储分组信息。

3. 建模技巧：离散化决策

有些题目看起来物品选择是连续的（例如：投入任意数量的资源），但实际上收益产生变化的临界点是有限的。

  • 技巧： 将这些有限的“临界点”视为组内的不同物品，从而将连续问题转化为分组背包。
  • 题解：Luogu-P5322 排兵布阵
