Luogu-P1439 两个排列的最长公共子序列

一、 题目大意

给定两个 1→n1 \to n1→n 的排列 A 和 B。
求 A 和 B 的最长公共子序列 (LCS) 的长度。

示例：

二、 思路分析与关键点

1. 暴力（Naive Solution）- O(N2)O(N^2)O(N2) DP

最经典的 LCS 解法是 O(N2)O(N^2)O(N2) 的动态规划。

定义 dp[i][j]dp[i][j]dp[i][j] 表示 AAA 序列的前 iii 个元素和 BBB 序列的前 jjj 个元素的最长公共子序列长度。

状态转移：

  1. 如果 A[i]==B[j]A[i] == B[j]A[i]==B[j]，则 dp[i][j]=dp[i−1][j−1]+1dp[i][j] = dp[i-1][j-1] + 1dp[i][j]=dp[i−1][j−1]+1
  2. 如果 A[i]≠B[j]A[i] \ne B[j]A[i]=B[j]，则 dp[i][j]=max⁡(dp[i−1][j],dp[i][j−1])dp[i][j] = \max(dp[i-1][j], dp[i][j-1])dp[i][j]=max(dp[i−1][j],dp[i][j−1])
  • 时间复杂度： O(N2)O(N^2)O(N2)。
  • 结论： N=105N=10^5N=105，O(N2)≈1010O(N^2) \approx 10^{10}O(N2)≈1010，T了！

2. 核心思路：LCS →\to→ LIS 转化 (O(N log N))

不难想到，印象中同类型的线性 DP 还有一个 LIS ，而 LIS 有一种二分优化可以达到 O(Nlog⁡N)O(N \log N)O(NlogN) 的时间复杂度，完全足以满足这题的规模。

于是我们回顾题目，发现有一个关键特性：A 和 B 都是 1→n1 \to n1→n 的排列。

这意味着 1→n1 \to n1→n 中的每个数字在 A 和 B 中恰好只出现一次。

大体思路

  1. LCS 的定义：

    一个公共子序列 C = (c1,c2,…,ckc_1, c_2, \dots, c_kc1​,c2​,…,ck​) 必须满足：

    • 在 A 中：posA(c1)<posA(c2)<⋯<posA(ck)pos_A(c_1) < pos_A(c_2) < \dots < pos_A(c_k)posA​(c1​)<posA​(c2​)<⋯<posA​(ck​)
    • 在 B 中：posB(c1)<posB(c2)<⋯<posB(ck)pos_B(c_1) < pos_B(c_2) < \dots < pos_B(c_k)posB​(c1​)<posB​(c2​)<⋯<posB​(ck​)

  2. 问题转化 (基于 A 序列)：

    我们可以 "拉直" A 序列。我们创建一个映射 mp，mp[val] = idx，表示数值 val 在 A 序列中的位置是 idx。

    ▶
    Example

    示例映射表 (A: 3 2 1 4 5)

    A[i] (值)32145
    i (位置)12345
    mp[A[i]]mp[3]=1mp[2]=2mp[1]=3mp[4]=4mp[5]=5

    现在，我们遍历 B 序列，并构造一个新的序列 CCC：
    C[i] = mp[ B[i] ] (B[i] 在 A 中的位置)

    CCC 序列构造表 (B: 1 2 3 4 5)

    i12345
    B[i] (值)12345
    C[i] = mp[B[i]]mp[1]=3mp[2]=2mp[3]=1mp[4]=4mp[5]=5
    CCC 序列32145

    得到 C=[3,2,1,4,5]C = [3, 2, 1, 4, 5]C=[3,2,1,4,5]

  3. LCS →\to→ LIS：

    • 我们在 A， B 序列中找一个 LCS (c1,c2,…,ckc_1, c_2, \dots, c_kc1​,c2​,…,ck​)。
    • 它们在 B 中的下标 iii 是递增的。
    • 它们在 A 中的位置 jjj 也必须是递增的。
    • 这等价于：在 CCC 序列中，找到一个子序列，它们的下标递增，且值递增（因为它们在 A 中的位置 mp 是递增的）。
    • 这不就是 LIS 的定义吗

    C=[3,2,1,4,5]C = [3, 2, 1, 4, 5]C=[3,2,1,4,5]

    LIS: (1,4,5)(1, 4, 5)(1,4,5) 或 (2,4,5)(2, 4, 5)(2,4,5) 或 (3,4,5)(3, 4, 5)(3,4,5)。
    长度为 3。

  • 复杂度对比：

    • LCS O(N2)O(N^2)O(N2) TLE
    • LIS O(Nlog⁡N)O(N \log N)O(NlogN) AC

  • 算法基础原理请参考：个人学习笔记 LIS 详解

3. 关键点与总结

  • 时间复杂度： O(N)O(N)O(N) (预处理) + O(Nlog⁡N)O(N \log N)O(NlogN) (LIS) →\to→ 总时间 O(Nlog⁡N)O(N \log N)O(NlogN)。
  • 空间复杂度： O(N)O(N)O(N) (存储 mp, dp 等数组)。
  • 核心逻辑： 核心是利用排列这一特性，将二维的 LCS 问题，通过 "位置映射" 的方式，转化为一维的 LIS 问题。
