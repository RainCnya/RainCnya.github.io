算法学习路线图：动态规划 (Dynamic Programming Roadmap)

算法学习路线图：动态规划 (Dynamic Programming Roadmap) -V3.0

一、 初步认知 (Macro View)

1. 叫什么 (Identity)

  • 中文名： 动态规划
  • 英文名： Dynamic Programming (简称 DP)

2. 什么地位 (Status)

  • 所属领域： 算法设计与优化

  • 核心地位： 算法竞赛和工程优化的 “核心思想”。

    • 它是一种思想，是解决 “多阶段决策最优化问题” 的通用范式，是连接搜索、递归、贪心与最优解的桥梁。

3. 有什么用处 (Utility)

  • 解决的核心问题： 动态规划是一种通过把原问题分解为相对简单的子问题的方式求解复杂问题的方法。

  • 典型应用场景：

    • 最值问题： 求最大/最小值（如：最长公共子序列、0/1 背包）
    • 计数问题： 求有多少种方案（如：爬楼梯）
    • 可行性问题： 判定是否可行（如：能否凑成某个金额）

二、 主题渗透 (Theme Osmosis)

1. 核心原理 (The Eureka Moment)

  • 底层思想： 记忆化搜索（Memoization） + 最优子结构 （Optimal Substructure）。

  • 关键推导（DP 成立的两大特征）：

    • 1. 最优子结构 (Optimal Substructure)： 原问题的最优解，可以由其子问题的最优解推导得出。
    • 2. 无后效性 (without aftereffect)： 已经求解的子问题，不会再受到后续决策的影响。
    • 3. 重叠子问题 (Overlapping Subproblems)： 在递归/推导过程中，大量相同的子问题被反复计算。

  • DP 的本质： 通过“填表”（递推）或“记忆化”（递归）的方式，确保每个子问题只被计算一次，从而将指数级的（朴素搜索）复杂度，优化为多项式级。

  • DPDPDP = 递归 + 记忆化 = 递推

2. 学习路径与领域划分 (Visualization)

▶
DP 学习路线图
  • 路径： 线性 →\to→ 背包、区间、树、图 →\to→ 状压、数位。

阶段一：基础 DP (The Foundation)

  • 目标： 理解 DP 的核心三要素：状态定义、状态转移方程、初始化。

  • 子领域：

    • 1. 线性 DP (Linear DP)： 状态由一维或二维线性序列决定。
      • 典型： 最长上升子序列 (LIS), 最长公共子序列 (LCS), 最大子数组和。
    • 2. 简单背包 DP (Knapsack DP - Basic)： 学习如何处理“选择”与“容量”限制。
      • 典型： 0/1 背包。

阶段二：经典 DP 模型 (Classic Models)

  • 目标： 掌握最常见、最典型的 DP 建模方式。

  • 子领域：

    • 1. 背包 DP (Knapsack DP - Advanced)：
      • 典型： 完全背包、多重背包（及其二进制/单调队列优化）。
    • 2. 区间 DP (Interval DP)：
      • 特征： 状态 dp[i][j] 通常表示区间 [i, j] 的最优解，由小区间合并而来。
      • 典型： 石子合并、矩阵链乘。
    • 3. 树形 DP (Tree DP)：
      • 特征： 在树状结构上进行 DP，通常利用 DFS 实现状态的自底向上转移。
      • 典型： 没有上司的舞会、树的直径。

阶段三：高级 DP 技巧 (Advanced Techniques)

  • 目标： 解决状态空间更复杂的问题。

  • 子领域：

    • 1. 状态压缩 DP (Bitmask DP)：
      • 特征： 当某个维度的数据范围极小（如 N≤20N \le 20N≤20）时，使用一个整数的二进制位来表示该维度的状态。
      • 典型： 旅行商问题 (TSP)、蒙德里安的梦想。
    • 2. 数位 DP (Digit DP)：
      • 特征： 解决在给定区间 [L,R][L, R][L,R] 内，满足特定“数字”属性（如不含 49）的数的个数。
      • 典型： 统计 1∼N1 \sim N1∼N 中 1 出现的次数。

阶段四：DP 优化 (DP Optimization)

  • 目标： 当状态转移的复杂度过高（如 O(N2)O(N^2)O(N2) 转移导致 O(N3)O(N^3)O(N3) 总时间）时，进行降维处理。

  • 子领域：

    • 1. 单调队列优化 DP
    • 2. 斜率优化 (Convex Hull Trick, CHT) DP
    • 3. 矩阵快速幂优化 DP
    • 4. 数据结构优化 DP (如：树状数组/线段树)

3. 详细实现过程 (Step-by-Step)

这是解决任何 DP 问题的通用思考流程：

  1. 判断题型： 确认问题是否具有“最优子结构”、“无后效性”和“重叠子问题”的性质，判断是否为 DP 问题（区别于贪心、搜索）。
  2. 定义状态： 将原问题划分为若干阶段，每个阶段对应若干个子问题，提取这些子问题的特征（称之为状态）。
  3. 推导方程： 寻找每一个状态的可能决策，或者说是各状态间的相互转移方式（用数学的语言描述就是状态转移方程）。
  4. 求解答案： 按照顺序求解每一个阶段的问题。

如果用图论的思想理解，那就是在一个 DAG 上求最短(长)路的问题。

三、 总结分析 (Analysis & Comparison)

1. 复杂度分析 (Complexity)

维度 复杂度 说明
时间复杂度 O(状态总数×单次转移复杂度)O(\text{状态总数} \times \text{单次转移复杂度})O(状态总数×单次转移复杂度) 例如：线性 DP O(N×N)O(N \times N)O(N×N)；背包 O(N×V)O(N \times V)O(N×V)。
空间复杂度 O(状态总数)O(\text{状态总数})O(状态总数) 通常等于 DP 数组的大小。有时可用“滚动数组”优化掉一个维度。

2. 核心注意点 (Key Pitfalls)

  • 1.(状态定义)： DP 失败的第一原因：状态定义不清。dp[i]dp[i]dp[i] 的含义是“前 iii 个”还是“第 iii 个”。
  • 2.(初始化)： 边界（如 dp[0]dp[0]dp[0]）或非法状态（如求最小值时应初始化为 INT_MAX）设置错误，导致转移答案出错。
  • 3.(转移顺序)： 在 0/1 背包中 j 必须倒序；在区间 DP 中必须先算短区间。顺序错误会破坏“无后效性”。
  • 4.(1-based 索引)： 使用 1-based 索引时，处理 dp[i-1] 相关的转移要特别小心 i=1i=1i=1 时的边界。
  • 适用条件： 问题必须具有最优子结构和无后效性。
