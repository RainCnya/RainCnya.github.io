# [P2569 [SCOI2010] 股票交易 - 洛谷](https://www.luogu.com.cn/problem/P2569)

## 1. 题面梗概

**中译中**: 给出 $T$ 天的股票信息。每天你可以买入或卖出一定限额的股票。

限制：两次交易之间必须间隔 $W$ 天（即第 $i$ 天交易，下次最早 $i+W+1$ 天）。

求最后能赚到的最大收益。

## 2. 逻辑推导

遇到这种多状态的题目，直接 DP 起手。

**定义**：$f[i][j]$ 表示第 $i$ 天交易完，手里持有 $j$ 股的最大利润。

**转移**：买入 / 不操作 / 卖出

### 2.1 转移

这个状态转移太阴了。

1. 初始化：前 $W + 1$ 天内，可以直接买入。

$$
f[i][j] = -ap_{i} \cdot j \quad (j \leq as_{i})
$$
2. 不操作：直接继承前一天的状态

$$
f[i][j] = \max( f[i][j], f[i-1][j] )
$$

3. 买入优化：从 $i - W - 1$ 天优化，且手中股票增多。

$$
f[i][j] = \max_{j - as_{i} \leq k < j}(f[j - W - 1][k] - (j - k) \cdot ap_{i})
$$
- 变形：

$$
f[i][j] = \max_{j - as_{i} \leq k < j}(f[i - W - 1][k] + k \cdot ap_{i}) - j \cdot ap_{i}
$$

4. 卖出优化：从 $i - W - 1$ 天优化，且手中股票减少。

$$
f[i][j] = \max_{j < k \leq j + bs_{i}}(f[j - W - 1][k] - (j - k) \cdot ap_{i})
$$
- 变形：

$$
f[i][j] = \max_{j < k \leq j + bs_{i}}(f[i - W - 1][k] + k \cdot bp_{i}) - j \cdot bp_{i}
$$

### 2.2 滑动窗口 

发现又是移动区间维护最值，单调队列滑动窗口启动，维护两个单调队列维护买入和卖出即可。

### 2.3 结论

这道题是单调队列优化 DP 的集大成者，状态多、转移杂，需要极其清晰的逻辑剥离。

不过推完了应该还挺清晰的吧。

## 3. 代码实现

{% fold info @AcCode %}

```cpp
#include<bits/stdc++.h>
using namespace std;
using ll = long long;

const int maxn = 2000 + 50;
const int maxp = 2000 + 50;
const ll linf = 1e18;

int T, MaxP, W;
int ap[maxn], bp[maxn];
int as[maxn], bs[maxn];
ll dp[maxp][maxp];

int main( )
{	
    cin >> T >> MaxP >> W;
    for( int i = 1; i <= T; ++ i )
    {
        cin >> ap[i] >> bp[i];
        cin >> as[i] >> bs[i];
    }

    memset( dp, 128, sizeof( dp ) );
    for( int i = 0; i <= T; ++ i )
        dp[i][0] = 0;
    
    for( int i = 1; i <= T; ++ i )
    {
        for( int j = 0; j <= as[i]; ++ j )
            dp[i][j] = -1 * ap[i] * j;
        
        for( int j = 0; j <= MaxP; ++ j )
            dp[i][j] = max( dp[i][j], dp[i-1][j] );
        
        int from = i - W - 1;
        if( from < 0 ) continue;

        deque<int> q1;
        for( int j = 0; j <= MaxP; ++ j )
        {
            while( !q1.empty( ) && q1.front( ) < j - as[i] )
                q1.pop_front( );
            
            while( !q1.empty( ) && 
                dp[from][ q1.back( ) ] + q1.back( ) * ap[i] <= 
                dp[from][j] + j * ap[i] )
                q1.pop_back( );
            q1.push_back( j );

            if( !q1.empty( ) )
                dp[i][j] = max( dp[i][j], dp[from][ q1.front( ) ] + (q1.front( ) - j) * ap[i] );
        }

        deque<int> q2;
        for( int j = MaxP; j >= 0; -- j )
        {
            while( !q2.empty( ) && q2.front( ) > j + bs[i] )
                q2.pop_front( );

            while( !q2.empty( ) && 
                dp[from][ q2.back( ) ] + q2.back( ) * bp[i] <= 
                dp[from][j] + j * bp[i] )
                q2.pop_back( );
            q2.push_back( j );

            if( !q2.empty( ) )
                dp[i][j] = max( dp[i][j], dp[from][ q2.front( ) ] + (q2.front( ) - j) * bp[i] );
        }
    }

    ll ans = 0;
    for( int i = 0; i <= MaxP; ++ i )
        ans = max( ans, dp[T][i] );
    
    cout << ans << '\n';
    return 0;
}
```

## 4. 复盘

- **复杂度分析**：$O(T \cdot MaxP)$。
    
- **碎碎念**：这题的码量和下标变换确实很复杂。最容易写错的地方就是卖出的单调队列，因为买入是正着扫，卖出是倒着扫，队列需要维护的是“未来可能作为起点”的值。
