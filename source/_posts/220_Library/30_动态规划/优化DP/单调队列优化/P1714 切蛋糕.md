# [P1714 切蛋糕 - 洛谷](https://www.luogu.com.cn/problem/P1714)

## 1. 题面梗概

**中译中**: 给定一个长度为 $N$ 的序列，找出一个长度不超过 $M$ 的连续子段，使得该子段的元素之和最大。

## 2. 逻辑推导

求解最大子段和，还有一个限制条件，最简单的想法就是枚举左右端点，然后通过前缀和优化，总复杂度是 $O(N^2)$，TLE 了，我们需要想方法去优化。

### 2.1 转化

$S[i \dots j] = Sum[j] - Sum[i-1]$，这是最简单的转化，假设现在我们固定有节点 $j$，为了让这个子段和最大，问题就转化为了：在合法的左区间 $i \in [j - M + 1, j]$ 中，寻找一个最小的点。

### 2.2 滑动窗口

那其实就很简单了，当 $j$ 不断向右扫描时，合法的左端点 $i - 1$ 候选区间 $[j - M, j - 1]$ 也是一个不断向右移动的窗口。

那么我们只需要在这个长度为 $M$ 的滑动窗口里，动态维护一个最小的 $Sum$ 值即可。

而这则正好是**单调队列**的生态位。

### 2.3 结论

前缀和 + 滑动窗口 求区间极值的入门题。

## 3. 代码实现

{% fold info @AcCode %}

```cpp
#include <bits/stdc++.h>
using namespace std;

const int maxn = 5e5 + 50;
int s[maxn], n, m;

int main( ) 
{
    ios::sync_with_stdio( 0 );
    cin.tie( 0 );

    cin >> n >> m;
    for( int i = 1; i <= n; ++ i ) 
    {
        int x; cin >> x;
        s[i] = s[i - 1] + x;
    }

    deque< int > q;
    q.push_back( 0 ); // 入队 sum[0]

    int ans = -2e9;
    for( int i = 1; i <= n; ++ i ) 
    {
        // 维持窗口长度 M
        while( ! q.empty( ) && q.front( ) < i - m ) q.pop_front( );
        
        ans = max( ans, s[i] - s[ q.front( ) ] );

        // 入队新前缀和，保持递增以求最小值
        while( ! q.empty( ) && s[ q.back( ) ] >= s[i] ) q.pop_back( );
        q.push_back( i );
    }

    cout << ans << "\n";
    return 0;
}
```
