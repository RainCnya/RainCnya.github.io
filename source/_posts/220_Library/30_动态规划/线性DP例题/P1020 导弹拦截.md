# [P1020 [NOIP 1999 提高组] 导弹拦截 - 洛谷](https://www.luogu.com.cn/problem/P1020)

## 1. 题面梗概

**中译中**：

1. 给一串导弹高度，第一套系统能拦截一套**最长不上升**子序列。求最多截多少。
    
2. 问：要把所有导弹截完，最少需要准备多少套这样的系统？

> $1 \leq n \leq 10^5$

## 2. 逻辑推导

这是一道很经典的 “子序列提取” 模型，可以用线性状态表示：

用 $f_{i}$ 表示，只考虑前 $i$ 个数，并且以第 $i$ 个数为结尾的子序列的长度。

接着是状态的转移，只需要考虑把第 $i$ 个数接在某个已知序列后面，这个序列必然是以 $a_{1}$ 到 $a_{i-1}$ 中大于 $a_{i}$ 的某一个数结尾的，而且越长越好。

$$
f_{i} = \max_{j < i \text{且}a_{j} \geq a_{i}} \{f_{j}\} + 1
$$

不过基本的 $O( N^2 )$ 转移在本题数据下会 TLE，故代码部分采用的是 $O( N \log N)$ 的贪心做法。

### 2.1 Dilworth定理

> 最少要配备导弹拦截系统数量就是该序列的最长上升子序列长度。

**Dilworth 定理**：对于一个偏序集，其**最小链覆盖数 = 最长反链长度**。
    
在本题中，“不上升”的对立面（反链）就是“严格上升”。

因此，最少系统数 $\iff$ 该序列的最长上升子序列（LIS）长度。

### 2.2 总结

这里基于一个很显然的性质：最长子序列一定是由前面的某个最长子序列转移过来的，即 **最长子序列** 满足此题的最优子结构，所以即使对序列其他信息一无所知、只知道结尾和子序列长度的情况下也能放心转移。

像这类只需要考虑相邻事件 “新事件” 和 “上一个事件” 关系的问题，是 “子序列提取” 题的一类常见模型。

### 2.3 偏序

当然这道题也可以用 DAG 上最长路的模型理解。

注意到本题中，数字的 **前后关系** 和 **大小关系** 这两个维度同时影响 DAG 的形态，构成一种确定的偏序，故把这种模型称为 “二维偏序”。

## 3. 代码实现

{% fold info @AcCode %}

```cpp
#include<bits/stdc++.h>
using namespace std;

const int maxn = 1e5 + 50;

int a[maxn];
int n;

int main()
{	
    int t;
    while( cin >> t ) a[++ n] = t;

	vector< int > d1;
    for( int i = 1; i <= n; ++ i )
    {
        auto it = upper_bound( d1.begin( ), d1.end( ), a[i], greater< int >( ) );
        if( it == d1.end( ) ) d1.push_back( a[i] );
        else *it = a[i];
    }
    cout << d1.size( ) << '\n';

    vector< int > d2;
    for( int i = 1; i <= n; ++ i )
    {
        auto it = lower_bound( d2.begin( ), d2.end( ), a[i] );
        if( it == d2.end( ) ) d2.push_back( a[i] );
        else *it = a[i];
    }
    cout << d2.size( ) << '\n';

    return 0;
}
```
