# [P2034 选择数字 - 洛谷](https://www.luogu.com.cn/problem/P2034)

## 1. 题面梗概

**中译中**: 给出一串数字，让你选出一部分数字使和最大，但有个限制：你不能连续选超过 $K$ 个数字。

## 2. 逻辑推导

这道题的难点在于两个点，一个是连续，另一个是数量限制。

如果我们直接定义当前连着选了几个的话，状态空间会扩展到 $O(NK)$，这样会 TLE ，要求我们换个角度。

### 2.1 断点

既然不能连续选超过 $K$ 个，那就意味着，在任何长度为 $K + 1$ 的区间内，必须至少有一个数被放弃，这个被放弃的数就是 “断点”。

**定义**：

- $f[i]$ 表示第 $i$ 个数不选时，前 $i$ 个数最大收益。
- $g[i]$ 表示第 $i$ 个数选时，前 $i$ 个数最大收益。

**状态转移**：

- 不选 $i$：前一个位置 $i-1$ 选不选都行。

$$
f[i] = \max(f[i-1], g[i-1])
$$
- 要选 $i$：那它必须属于某一个连续段，那么我们需要找到前面的那个 **断点** $j$。

$$
f[i] = \max_{i-K \leq j < i}(f[j] + s[i] - s[j])
$$
- 接着我们把跟 $j$ 无关的提取出来：

$$
f[i] = s[i] + \max_{i-K \leq j < i}(f[j] - s[j])
$$
### 2.3 滑动窗口

接着动态维护这个最值就不用说了，单调队列模型秒了，注意细节即可，这里单调队列维护的这个值是 $f[j] - s[j]$。

## 3. 代码实现

{% fold info @AcCode %}

```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

const int maxn = 1e5 + 50;
ll a[maxn], s[maxn];
ll f[maxn], g[maxn];
int n, k;

int main( ) 
{
    ios::sync_with_stdio( 0 );
    cin.tie( 0 );

    cin >> n >> k;
    for( int i = 1; i <= n; ++ i ) 
    {
        cin >> a[i];
        s[i] = s[i - 1] + a[i];
    }

    deque< int > q;
    q.push_back( 0 ); // 初始断点

    for( int i = 1; i <= n; ++ i ) 
    {
        // 1. 维护滑窗范围 [i-K, i-1]
        while( ! q.empty( ) && q.front( ) < i - k ) q.pop_front( );

        // 2. 转移
        f[i] = max( f[i-1], g[i-1] );
        g[i] = s[i] + ( f[ q.front( ) ] - s[ q.front( ) ] );

        // 3. 入队: 维护 f[j][0] - s[j] 的递减队列
        ll val = f[i] - s[i];
        while( ! q.empty( ) && ( f[ q.back( ) ] - s[ q.back( ) ] ) <= val ) q.pop_back( );
        q.push_back( i );
    }

    cout << max( f[n][0], f[n][1] ) << "\n";
    return 0;
}
```

## 4. 复盘

- **复杂度分析**：$O(N)$。
        
- **碎碎念**：推导公式的时候，遇到复杂的情况，一定要把跟变量无关的量提取出来，这样就简单了。

- 这题其实还有个有趣的做法：求出所有数字总和，减去“每 $K+1$ 个数里至少选一个”的最小和，不过这就不在 DP 专题里了。
