# [P1613 跑路 - 洛谷](https://www.luogu.com.cn/problem/P1613)

## 1. 题面梗概

**中译中**：给定一个有向图，边权均为 1。你可以花费 1 秒走过长度恰好为 $2^k$（$k \in \mathbb{N}$）的路径。求从点 1 到点 $n$ 的最少秒数。

> $2 \le n \le 50, m \le 10^4$

## 2. 逻辑推导

由于特殊性质，直接求出的最短路不一定是最优解，因此要对原图进行加工。

设最优解路径长度为 $s$，可以得到最优解消耗的时间是在二进制下 $s$ 中 $1$ 的个数。

也就等价于把这条路径拆分成若干个 $2^k$ 的路径，每条路径消耗的时间是 $1$ 秒。

### 2.1 DP优化建图

设 $f[i][u][v]$ 表示是否存在 从 $u$ 到 $v$ 长度为 $2^i$ 的路径。

> 特别的 $f[0][u][v]$ 其实等价于原图。

由于任何一条 $2^i$ 的路径都能拆解为 $2$ 条 $2^{i-1}$ 长度的路径，因此只要枚举中转点 $t$。

$$
f[i][u][v] = \lor_{w} (f[i-1][u][w] \wedge f[i-1][w][v])
$$

> $\wedge$ 逻辑与，$\lor$ 逻辑或

```cpp
f[i][u][v] |= f[i-1][u][w] & f[i-1][w][v]
```

接着在新图上跑一遍最短路算法即可。

## 3. 代码实现

{% fold info @AcCode %}
```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

const int maxn = 50 + 5;
const ll inf = 1e18;

bool f[maxn][maxn][maxn];
// f[k][u][v] 表示 存在一条 长度为 2^k，从 u 到 v 的路径
ll dist[maxn][maxn];
int n, m;

int main( )
{
    ios::sync_with_stdio(0);
    cin.tie(0);
    
    // input
    cin >> n >> m;
    for( int i = 1; i <= m; ++ i )
    {
        int u, v;
        cin >> u >> v;
        f[0][u][v] = 1;
    }

    // DP 辅助建图
    for( int i = 1; i <= 30; ++ i )
        for( int k = 1; k <= n; ++ k )
            for( int u = 1; u <= n; ++ u )
                for( int v = 1; v <= n; ++ v )
                    f[i][u][v] |= ( f[i - 1][u][k] & f[i - 1][k][v] );

    // dist 初始化
    for( int i = 1; i <= n; ++ i )
        for( int j = 1; j <= n; ++ j )
            if( i == j ) dist[i][j] = ( i == 1 ? 0 : inf );

    for( int i = 0; i <= 30; ++ i )
        for( int u = 1; u <= n; ++ u )
            for( int v = 1; v <= n; ++ v )
                if( f[i][u][v] ) dist[u][v] = min( dist[u][v], 1ll );

    // floyd
    for( int k = 1; k <= n; ++ k )
        for( int u = 1; u <= n; ++ u )
            for( int v = 1; v <= n; ++ v )
                dist[u][v] = min( dist[u][v], dist[u][k] + dist[k][v] );

    cout << dist[1][n] << '\n';
    return 0;
}
```
