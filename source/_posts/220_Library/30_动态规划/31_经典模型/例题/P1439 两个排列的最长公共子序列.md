# [P1439 两个排列的最长公共子序列 - 洛谷](https://www.luogu.com.cn/problem/P1439)

## 1. 题面梗概

**中译中**：给定两个 $1 \sim N$ 的全排列 $A$ 和 $B$。求它们的最长公共子序列（LCS）长度。

> $1 \le N \le 10^5$

## 2. 逻辑推导

这不是 LCS 模型裸题吗，看我直接秒了。

$$
f[i][j] = \begin{cases}  
f[i-1][j-1] + 1 & (A_i = B_j)  \\
\max(f[i-1][j], f[i][j-1]) & (A_i \neq B_j)  \\
\end{cases}
$$

布豪，朴素解法复杂度是 $O( N^2 )$，这题过不了。

### 2.1 性质分析

对于两个排列的 LCS，可以使用一个技巧把它转化成 $LIS$，即将第二行的数字替换为 $\{1, 2, \dots, n\}$，同时也对第一行的相同数字做这一替换，下面结合样例说明。

> 排列是指 $1$ 到 $n$ 出现一次的长度为 $n$ 的序列。

```
input
3 1 4 2 5
4 1 3 2 5

change
3 2 1 4 5
1 2 3 4 5
```

可以看到第二行已经有序了，但由于只做了位置不变的字符替换，两序列的本质没有变，这样第一行任意一个子序列都可以是上下两行的公共子序列。

然后接下来就可以在这个新序列上跑一遍优化过的 LIS 模型，复杂度 $O( N \log N )$ 解决了。


## 3. AC 代码

{% fold info @AcCode %}

```cpp
#include<bits/stdc++.h>
using namespace std;

const int maxn = 1e5 + 5;

int n;
int a[maxn], b[maxn];
int mp[maxn], f[maxn];

int main()
{	
    cin >> n;
    for( int i = 1; i <= n; ++ i )
    {
        cin >> a[i];
        mp[ a[i] ] = i;
    }
    
    for( int j = 1; j <= n; ++ j )
    {
        cin >> b[j];
    }

    int len = 0;

    for( int i = 1; i <= n; ++ i )
    {
        int pos = lower_bound( f + 1, f + len + 1, mp[ b[i] ] ) - f;
        f[pos] = mp[ b[i] ];
        if( pos > len ) len = pos;
    }

    cout << len << '\n';
    return 0;
}
```

{% endfold %}

## 4. 复盘

- **复杂度分析**：$O(N \log N)$，瓶颈在 贪心 LIS 。
    
- **碎碎念**：这是一道伪装成 LCS 的 变种 LIS 问题，转化关键在 **排列**。
