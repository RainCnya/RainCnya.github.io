# [P2679 [NOIP 2015 提高组] 子串 - 洛谷](https://www.luogu.com.cn/problem/P2679)

## 1. 题面梗概

**中译中**：从字符串 $A$ 中取出 $k$ 个互不重叠的非空子串，按顺序拼成字符串 $B$。求方案数。

> $n \le 1000, m \le 200, k \le m$, 答案对 $10^9 + 7$ 取模。

## 2. 逻辑推导

从题目中出现的元素来说，这是一道 双序列 + 子段划分 的计数题。

既然要在字符串 $A$ 取出 $k$ 段，去匹配字符串 $B$，最好把子段数和两个序列的位置都放在状态的定义里。

### 2.1 状态

**定义**：$f[k][i][j]$ 表示字符串 $A$ 以 $i$ 字符结尾取出 $k$ 段后，在字符串 $B$ 恰好匹配到 $j$ 的方案数。

先考虑朴素的转移，我们发现大部分情况下 $f[k][i][j]$ 是等于 $0$ 的，除非 $A[i] = B[j]$。

那么问题来了，当 $A[i] = B[j]$ 的时候会发生什么呢？

- $A[i]$ 跟在上一段后：既然段数没变 ，那么就可以从 $(k, i - 1, j - 1)$ 转移过来。
- $A[i]$ 新起了一段：有点麻烦，能确定的是 $( k - 1, ?, j - 1 )$，不好说是从哪个 $i$ 转移的。

### 2.2 转移

因为状态已经是 $O( nmk )$ 数量级，转移必须控制在 $O(1)$ 的规模。为了方便第二种情况的转移，我们需要一个辅助数组来计算这个 **前缀和**。

定义： $g[k][i][j]$ 为 $\sum_{x=1}^{x \leq i}f[k][x][j]$，就是第二维度的 前缀和。

于是很简单能写出下面这段代码：

```cpp
for( int k = 1; k <= kk; ++ k )
{
	for( int i = 1; i <= n; ++ i )
	{
		for( int j = 1; j <= m; ++ j )
		{
			if( A[i] == B[j] ) f[k][i][j] = ( f[k][i-1][j-1] + g[k-1][i-1][j-1] ) % mod;
			else f[k][i][j] = 0;
			g[k][i][j] = ( f[k][i][j] + g[k][i-1][j] ) % mod;
		}
	}
}
```

### 2.3 滚动数组

注意到方程中 $i$ 的状态只依赖于 $i-1$，所以我们可以通过滚动数组把 $i$ 这一维度滚掉。

> 由于 $n = 1000, m = 200, k = 200$，三维数组 $1000 \times 200 \times 200 \times 4$ 字节约为 $160 MB$。 
> 在本题是爆内存了。（ 俗称 MLE ）

## 3. 代码实现

{% fold info @AcCode %}
```cpp
#include <bits/stdc++.h>
using namespace std;

using ll = long long;

const int maxn = 1e3 + 5;
const int maxm = 200 + 5;
const int mod = 1e9 + 7;

// f[k][i][j] 表示取 k 段且 A[i] 必须选，匹配 B[j]
int f[2][maxn][maxm];
// g[k][i][j] 表示取 k 段且 A[i] 可选，匹配 B[j]
int g[2][maxn][maxm];
int n, m, kk;
string A, B;

int main( )
{
    cin >> n >> m >> kk;
    cin >> A >> B;
    A = ' ' + A, B = ' ' + B;

    for( int i = 0; i <= n; ++ i )  g[0][i][0] = 1;

    int cur = 1;
    for( int k = 1; k <= kk; ++ k )
    {
        memset( f[cur], 0, sizeof( f[cur] ) );
        memset( g[cur], 0, sizeof( g[cur] ) );

        for( int i = 1; i <= n; ++ i )
        {
            for( int j = 1; j <= m; ++ j )
            {
                if( A[i] == B[j] ) f[cur][i][j] = ( f[cur][i-1][j-1] + g[cur^1][i-1][j-1] ) % mod;
                else f[cur][i][j] = 0;
                g[cur][i][j] = ( f[cur][i][j] + g[cur][i-1][j] ) % mod;
            }
        }
        cur ^= 1;
    }

    cout << g[cur^1][n][m] << '\n';
    return 0;
}
```
{% endfold %}

## 4. 复盘

- **复杂度分析**：$O( n \cdot m \cdot k )$。
    
- **碎碎念**：这个转移说难想不难，说简单也不简单，还得多练啊。
